## 1. 项目概览

**Radix Primitives** 是一个开源的、低层级(low-level)React UI 组件库,专注于 **可访问性(Accessibility)、可定制性(Customizability)与开发者体验(Developer Experience)**。它不是一套带样式的成品组件库(如 MUI),而是提供无样式、行为与可访问性完备的"原语"(primitives),用作设计系统的底层基座。

设计哲学(见 [philosophy.md](file:///workspace/philosophy.md))的五大原则:

1. **Accessible** — 遵循 WAI-ARIA,在主流浏览器与多种辅助技术(VoiceOver / JAWS / NVDA)上测试。
2. **Functional** — 功能完备:键盘交互、碰撞检测、焦点陷阱、动态尺寸、滚动锁定、原生回退等。
3. **Interoperable** — 开放 API,1-to-1 渲染(单组件只渲染单个 DOM 节点),ref 与事件透传如消费者预期。
4. **Composable** — 支持组合优先于配置;`asChild` + `Slot` 模式让组件可被任意元素替换。
5. **Customizable** — 默认零样式,通过 `data-state` 等 data 属性暴露状态供 CSS 选择,不绑定样式方案。

## 5. 核心抽象与设计模式

这些模式贯穿几乎所有组件,理解它们是理解整个仓库的钥匙。

### 5.1 Context Scope 机制(解决嵌套同名组件串扰)

文件:[packages/react/context/src/create-context.tsx](file:///workspace/packages/react/context/src/create-context.tsx)

当多个同名组件嵌套(如 Dialog 里套 Dialog),原生 `React.createContext` 会让内层误读外层。Radix 通过 `createContextScope` + `__scopeXxx` prop 链传递解决:

- `createContextScope(scopeName, deps)` 返回 `[createContext, createScope]`。
- `createContext` 创建带 scope 的 Provider/useContext,`useContext` 会从 `scope[scopeName][index]` 取正确的 Context 实例。
- `createScope()` 返回一个 hook,把 `__scopeXxx` 转成完整的 scope 对象。
- `composeContextScopes` 合并多个 scope(供衍生包嵌套基础包 scope)。

**典型用法**(见 [accordion.tsx](file:///workspace/packages/react/accordion/src/accordion.tsx#L24-L32)):

```tsx
const [Collection, useCollection, createCollectionScope] = createCollection(ACCORDION_NAME);
const [createAccordionContext, createAccordionScope] = createContextScope(ACCORDION_NAME, [
  createCollectionScope,
  createCollapsibleScope,
]);
const useCollapsibleScope = createCollapsibleScope();
```

衍生包(如 dropdown-menu)再嵌入基础包 scope:`createContextScope('DropdownMenu', [createMenuScope])`,并通过 `{...menuScope}` 透传。

### 5.2 asChild + Slot 模式

文件:[packages/react/slot/src/slot.tsx](file:///workspace/packages/react/slot/src/slot.tsx)、[packages/react/primitive/src/primitive.tsx](file:///workspace/packages/react/primitive/src/primitive.tsx)

`asChild` 让消费者用自己的元素替换默认 DOM 节点,同时保留组件的所有行为(props/ref/事件合并到该元素)。核心实现:

- `Primitive.div` 等内部判断 `asChild`,为 true 时用 `Slot` 渲染,否则直接渲染对应节点。
- `Slot`(`createSlot` 工厂)合并 props:`className` 拼接、`style` 浅合并、`onXxx` 事件处理器组合(子级先调用,父级后调用)、`ref` 合并(`useComposedRefs`)。
- `Slottable` 标记"插槽内容",支持 `<Slot><Slottable>{child}</Slottable>{rest}</Slot>` 这种把 `rest` 作为 child 的 children 注入 child 元素的场景。
- 兼容 React 19 的 ref 访问差异(`getElementRef` 分别处理 `element.props.ref` 与 `element.ref`)。
- 兼容 `React.lazy` 子元素(用 `use()` 解包 payload)。

### 5.3 受控/非受控统一

文件:[packages/react/use-controllable-state/src/use-controllable-state.tsx](file:///workspace/packages/react/use-controllable-state/src/use-controllable-state.tsx)

`useControllableState({ prop, defaultProp, onChange, caller })` 返回 `[value, setValue]`:

- `prop !== undefined` → 受控,直接用 `prop`,`setValue` 只触发 `onChange`。
- 否则非受控,内部 `useState`,`setValue` 更新内部状态并触发 `onChange`。
- 开发态监测 controlled↔uncontrolled 切换并告警(`caller` 用于提示来源组件名)。
- 用 `useInsertionEffect` 保持 `onChangeRef` 最新,避免闭包陈旧。

几乎所有有状态的组件(open / value / checked / currentTabStopId)都用它。

### 5.4 Collection(子项注册 + 有序访问)

文件:[packages/react/collection/src/collection-legacy.tsx](file:///workspace/packages/react/collection/src/collection-legacy.tsx)(默认)、[collection.tsx](file:///workspace/packages/react/collection/src/collection.tsx)(unstable 新版)

解决"父组件如何在不依赖 DOM 遍历的前提下,以稳定顺序访问一组子项"。

- `createCollection<ItemElement, ItemData>(name)` 返回 `[Collection, useCollection, createCollectionScope]`。
- 父级放 `Collection.Provider` + `Collection.Slot`;每个子项放 `Collection.ItemSlot` 注册 `{ ref, ...itemData }`。
- `useCollection(scope)` 返回 `getItems()`(legacy)或 itemMap(unstable)。
- **legacy 版**(当前默认):用 `querySelectorAll([data-radix-collection-item])` + DOM 顺序排序,惰性计算。
- **unstable 版**:基于 `OrderedDict`(继承 Map,额外维护 `#keys` 数组支持索引访问),用 `Node.compareDocumentPosition` 排序,及早维护。导出为 `unstable_createCollection`。

被 `roving-focus`、`select`、`slider`、`accordion`、`menu`、`menubar` 等广泛使用。

### 5.5 Roving Focus(漫游焦点)

文件:[packages/react/roving-focus/src/roving-focus-group.tsx](file:///workspace/packages/react/roving-focus/src/roving-focus-group.tsx)

WAI-ARIA 推荐的复合组件焦点模式:一组可聚焦元素中,只有"当前 tab stop"在 Tab 序列中(`tabIndex=0`),其余 `tabIndex=-1`,方向键在组内移动焦点。

- `RovingFocusGroup.Root`:`orientation` / `loop` / `currentTabStopId` / `onEntryFocus` / `dir`(via `useDirection`)。
- `RovingFocusGroup.Item`:`focusable` / `active` / `tabStopId`。
- RTL 处理:`getDirectionAwareKey` 在 rtl 下互换 ArrowLeft↔ArrowRight。
- 方向过滤:`getFocusIntent` 在 vertical 时忽略左右键。
- 键映射:ArrowLeft/Up→prev、ArrowRight/Down→next、Home/PageUp→first、End/PageDown→last。
- Shift+Tab 退出整组(`isTabbingBackOut`)。
- 内部自建一套 Collection。

被 `tabs`、`toggle-group`、`radio-group`、`toolbar`、`menu`(vertical)、`menubar`(horizontal)复用。

### 5.6 BubbleInput(原生表单桥接)

Checkbox / Radio / Switch / Slider / Select 等"按钮外观但需参与原生表单"的组件,都在 button 之外渲染一个 visually-hidden 的原生 `<input>`/`<select>`:

- 状态变化时取 `HTMLInputElement.prototype.checked`(或 value)描述符的 setter **原生调用**,再 dispatch `click`/`input`/`change` 事件,使外层 `<form>` 的原生校验与 change 事件正常工作。
- `hasConsumerStoppedPropagationRef` 协调事件只冒泡一次。
- `isFormControl = !!form || !!control.closest('form')`(默认 true 支持 SSR 无 JS)。

### 5.7 data-* 状态暴露

组件状态统一通过 data 属性暴露供 CSS 选择,这是"零样式但可主题化"的关键:

- `data-state`:`'open' | 'closed'` / `'checked' | 'unchecked' | 'indeterminate'` / `'on' | 'off'`
- `data-disabled`、`data-highlighted`、`data-orientation`、`data-side`、`data-align`
- `data-radix-*` 内部标识(如 `data-radix-collection-item`、`data-radix-menu-content`)

### 5.8 dispatchDiscreteCustomEvent

文件:[packages/react/primitive/src/primitive.tsx](file:///workspace/packages/react/primitive/src/primitive.tsx#L103-L105)

React 18+ 会批量所有事件。自定义事件在 `discrete` 事件(如 pointerdown)内派发时,React 无法推断其优先级,可能导致更新被意外批处理。`dispatchDiscreteCustomEvent` 用 `ReactDOM.flushSync` 强制立即 flush,保证自定义事件(如 `menu.itemSelect`、dismissable-layer 的 outside 事件)更新可预测。

## 6. 关键基础模块详解

### 6.1 Primitive

文件:[packages/react/primitive/src/primitive.tsx](file:///workspace/packages/react/primitive/src/primitive.tsx)

```ts
const NODES = ['a','button','div','form','h2','h3','img','input','label','li','nav','ol','p','select','span','svg','ul'] as const;
const Primitive = NODES.reduce((primitive, node) => {
  const Slot = createSlot(`Primitive.${node}`);
  const Node = React.forwardRef((props, forwardedRef) => {
    const { asChild, ...primitiveProps } = props;
    const Comp = asChild ? Slot : node;
    return <Comp {...primitiveProps} ref={forwardedRef} />;
  });
  return { ...primitive, [node]: Node };
}, {} as Primitives);
```

- 对 17 种常见 HTML 节点各生成一个 `Primitive.xxx` forwardRef 组件,统一支持 `asChild`。
- 扩展 `CSSProperties` 允许 `--radix-*` 自定义属性。
- 在 window 上打标记 `Symbol.for('radix-ui') = true`(用于运行时探测)。

### 6.2 Slot / Slottable

详见 5.2。关键导出:`Slot`(别名 `Root`)、`Slottable`、`createSlot`、`createSlottable`。`mergeProps` 处理 className 拼接、style 合并、`on[A-Z]` 事件处理器组合。

### 6.3 Presence(动画感知的挂载控制)

文件:[packages/react/presence/src/presence.tsx](file:///workspace/packages/react/presence/src/presence.tsx)

```tsx
const [state, send] = useStateMachine(initialState, {
  mounted:        { UNMOUNT: 'unmounted', ANIMATION_OUT: 'unmountSuspended' },
  unmountSuspended:{ MOUNT: 'mounted', ANIMATION_END: 'unmounted' },
  unmounted:      { MOUNT: 'mounted' },
});
```

- `present` 变 false 时不立即卸载,而是检测 `animationName` 是否变化以判断是否有退出动画;有则进入 `unmountSuspended` 等待 `animationend`/`animationcancel`,无则直接 `unmounted`。
- `useStableComposedRefs`:返回稳定 identity 的 callback ref(React 19 兼容,避免每次渲染重附 ref 导致死循环,见 issue #3664)。
- 监听 `animationstart`/`animationcancel`/`animationend`;退出动画期间临时设 `animationFillMode: 'forwards'` 防闪烁。

### 6.4 Portal

文件:[packages/react/portal/src/portal.tsx](file:///workspace/packages/react/portal/src/portal.tsx)

SSR 安全的 `ReactDOM.createPortal` 封装:`useLayoutEffect` 后才设 `mounted=true`,服务端不访问 `document.body` 返回 null;支持自定义 `container`。

### 6.5 DismissableLayer

文件:[packages/react/dismissable-layer/src/dismissable-layer.tsx](file:///workspace/packages/react/dismissable-layer/src/dismissable-layer.tsx)

弹层 dismiss 的核心。维护模块级 `DismissableLayerContext`(layers / layersWithOutsidePointerEventsDisabled / branches / dismissableSurfaces 四个 Set)。

- **Esc**:`useEscapeKeydown`,仅最高层响应。
- **外部 pointerdown**:`usePointerDownOutside`,先在 `pointerdown` 标记,等 `click` 再派发(容忍第三方 `stopPropagation` 与触屏 350ms 延迟);`deferPointerDownOutside` 控制是否等待。
- **外部 focus**:`useFocusOutside`,监听 `focusin`。
- **disableOutsidePointerEvents**:把 body `pointer-events` 设 none,仅当前层及以上启用(多层叠加正确还原)。
- **Branch**:`DismissableLayerBranch` 注册"分支"(如子菜单 anchor),外部交互落在分支上不 dismiss。
- **dismissableSurfaces**:`useDismissableLayerSurface` 注册"可关闭表面"(如 dialog overlay),即使其上 `stopPropagation` 仍能关闭(issue #3346)。

### 6.6 FocusScope

文件:[packages/react/focus-scope/src/focus-scope.tsx](file:///workspace/packages/react/focus-scope/src/focus-scope.tsx)

- `trapped`:true 时焦点无法逃出。靠 document `focusin`/`focusout` + `MutationObserver` 三重监听拉回 `lastFocusedElementRef`。
- `loop`:Tab 到末尾绕回首项。
- `onMountAutoFocus` / `onUnmountAutoFocus`:可阻止的自动焦点事件(CustomEvent,`cancelable`)。Mount 时聚焦首个可 tab 元素(过滤 `<a>`),Unmount 时延迟聚焦 `previouslyFocusedElement`。
- **focusScopesStack**:模块级单例栈,新 scope 入栈 pause 旧栈顶,出栈 resume 新栈顶。

### 6.7 FocusGuards

文件:[packages/react/focus-guards/src/focus-guards.tsx](file:///workspace/packages/react/focus-guards/src/focus-guards.tsx)

在 body 首尾各注入一个不可见 `<span data-radix-focus-guard tabIndex=0>`,引用计数共享单例。保证 Tab 到边界时 `focusin`/`focusout` 仍能被 FocusScope 捕获,防止 Tab 跳出文档。

### 6.8 Popper(定位引擎)

文件:[packages/react/popper/src/popper.tsx](file:///workspace/packages/react/popper/src/popper.tsx)

封装 `@floating-ui/react-dom`,无 `usePopper` hook,定位逻辑直接在 `PopperContent` 内调用 `useFloating`。

- `Popper`(Root):Provider,持有 `anchor` + `placementState`。
- `PopperAnchor`:DOM 锚点(callback ref,避免嵌套更新超限 issue #3858)或 `virtualRef` 虚拟锚点。
- `PopperContent`:`strategy: 'fixed'`,middleware 链 `offset → shift(limitShift by sticky) → flip → size → arrow → transformOrigin(自定义) → hide(hideWhenDetached)`。输出 CSS 变量 `--radix-popper-transform-origin` / `--radix-popper-available-width/height` / `--radix-popper-anchor-width/height`。未定位时 `translate(0, -200%)` 移出页面避免测量期闪烁。
- `PopperArrow`:基于 `@radix-ui/react-arrow`,外包 `<span>`(ResizeObserver 对 SVG 测量不准)。

### 6.10 composeEventHandlers

文件:[packages/core/primitive/src/primitive.tsx](file:///workspace/packages/core/primitive/src/primitive.tsx#L11-L23)

```ts
function composeEventHandlers(originalEventHandler, ourEventHandler, { checkForDefaultPrevented = true } = {}) {
  return function handleEvent(event) {
    originalEventHandler?.(event);                    // 消费者的 handler 先执行
    if (checkForDefaultPrevented === false || !event.defaultPrevented) {
      return ourEventHandler?.(event);                // 消费者可 preventDefault 阻止内部行为
    }
  };
}
```

让消费者能通过 `event.preventDefault()` 取消组件内部默认行为(如关闭、选中)。

## 7. 覆盖层(Overlay)系统

覆盖层系统是分层"积木式"架构,从底到顶:定位层(Popper)→ 基础能力层(Portal/Presence/DismissableLayer/FocusScope/FocusGuards)→ 中间组合层(Dialog/Popover/Tooltip/HoverCard)。

### 7.1 共享骨架模式

所有覆盖层组件遵循:`Root(Provider) → Trigger/Anchor → Portal(Presence 包裹) → Content`。

### 7.2 Dialog(强模态)

文件:[packages/react/dialog/src/dialog.tsx](file:///workspace/packages/dialog/src/dialog.tsx)

组合 `Portal` + `Presence` + `FocusScope` + `DismissableLayer` + `useFocusGuards` + `RemoveScroll`(react-remove-scroll)+ `hideOthers`(aria-hidden)。

- `DialogOverlay`:仅 `modal=true` 时渲染,`RemoveScroll allowPinchZoom shards={[contentRef]}`,通过 `useDismissableLayerSurface()` 注册为可关闭表面。
- `DialogContentModal`:`trapFocus={open}`、`disableOutsidePointerEvents={open}`、`hideOthers(content)`(比 `aria-modal` 兼容性好)、`onCloseAutoFocus` 焦点回 trigger、右键外部 `preventDefault`。
- `DialogContentNonModal`:`trapFocus=false`,用 `hasInteractedOutsideRef`/`hasPointerDownOutsideRef` 处理点击 trigger 不重开的边界。

### 7.3 Popover(可模态弹层)

文件:[packages/react/popover/src/popover.tsx](file:///workspace/packages/react/popover/src/popover.tsx)

基于 Popper,`modal` 默认 `false`。`hasCustomAnchor` 模式:有 `PopoverAnchor` 时 trigger 不再包 Anchor。Modal/NonModal 分流同 Dialog。CSS 变量重命名 `--radix-popper-*` → `--radix-popover-content-*` / `--radix-popover-trigger-*`。

### 7.4 Tooltip(瞬时提示)

文件:[packages/react/tooltip/src/tooltip.tsx](file:///workspace/packages/react/tooltip/src/tooltip.tsx)

- **延迟机制**:非状态机,用 `openTimerRef` + `wasOpenDelayedRef` + `useControllableState`。`handleDelayedOpen` setTimeout 延迟打开;`data-state` 区分 `'instant-open'` / `'delayed-open'` / `'closed'`。
- **TooltipProvider**:全局延迟协调,`delayDuration`(默认 700ms)、`skipDelayDuration`(默认 300ms,跨 trigger 跳过延迟窗口)。
- **Hoverable Content**(亮点):`TooltipContentHoverable` 用 **凸包(Convex Hull)多边形** 作为缓冲区,指针从 trigger 移向 content 时不误关。`getHull`(Andrew monotone chain)+ `isPointInPolygon`(射线法)。
- 无 FocusScope(不陷阱焦点);`DismissableLayer` 的 `onFocusOutside` preventDefault。

### 7.5 HoverCard(富内容卡片)

文件:[packages/react/hover-card/src/hover-card.tsx](file:///workspace/packages/react/hover-card/src/hover-card.tsx)

- 与 Tooltip 同源(都基于 Popper,非模态),但 **打开和关闭都延迟**(`openDelay` 默认 700,`closeDelay` 默认 300)。
- **文本选择支持**:`containSelection` state,按下时 `body.userSelect=none`、content 内 `userSelect=text`。
- **Tab 隔离**:effect 给 content 内所有 `tabIndex>=0` 设 `tabindex=-1`。
- trigger 是 `Primitive.a`(链接),`excludeTouch` 屏蔽 touch。
- 无 grace-area 凸包(用 closeDelay + content pointerEnter/Leave 替代)。

### 7.6 覆盖层对基础能力取舍矩阵

| 能力 | Dialog | Popover(模态) | Popover(非模态) | Tooltip | HoverCard |
|------|:----:|:----:|:----:|:----:|:----:|

| Portal + Presence | 是 | 是 | 是 | 是 | 是 |
| DismissableLayer | 是 | 是 | 是 | 是 | 是 |
| FocusScope(陷阱) | 是 | 是 | 否 | 否 | 否 |
| useFocusGuards | 是 | 是 | 是 | 否 | 否 |
| RemoveScroll | 是(Overlay) | 是 | 否 | 否 | 否 |
| hideOthers(aria-hide) | 是 | 是 | 否 | 否 | 否 |
| 延迟打开 | 否 | 否 | 否 | 是 | 是 |
| 延迟关闭 | 否 | 否 | 否 | 否 | 是 |

## 10. 表单系统

### 10.1 Form(校验编排中心)

文件:[packages/react/form/src/form.tsx](file:///workspace/packages/react/form/src/form.tsx)

`Form`(Primitive.form)包两层 context:`ValidationProvider` + `AriaDescriptionProvider`,内部 `useState` 维护四张按字段名索引的 map:

| Map | 内容 | 来源 |
|---|---|---|
| `validityMap` | 每字段 `ValidityState`(原生) | `onFieldValidityChange` |
| `customMatcherEntriesMap` | 每字段 `CustomMatcherEntry[]` | FormMessage(`FormCustomMessage`)注册 |
| `customErrorsMap` | 每字段 `Record<id, boolean>` | `onFieldCustomErrorsChange` |
| `messageIdsMap` | 每字段 `Set<string>`(FormMessage DOM id) | `FormMessageImpl` 注册 |

### 10.2 FormControl(校验执行核心)

`updateControlValidity(control)` 是异步五步流程:

1. 内置错误(`hasBuiltInError`,排除 valid/customError)→ 记录 validity 返回。
2. 收集 `new FormData(control.form)` 供跨字段比对。
3. 拆分同步/异步 matcher(`isAsyncCustomMatcherEntry` 检测 AsyncFunction/Promise)。
4. 跑同步 matcher → `control.setCustomValidity(...)` → 更新 map。
5. 无同步错误且有异步 matcher → `Promise.all` 跑异步。

监听原生 `change` 事件(非 React onChange,避免每次按键校验)触发校验;React `onChange` 调 `resetControlValidity`。渲染 `aria-describedby` 拼接所有 FormMessage id、`data-valid`/`data-invalid`。

### 10.3 FormMessage(三态)

- 无 `match`:无效时显示 `DEFAULT_INVALID_MESSAGE`。
- `match` 为字符串(如 `valueMissing`):`FormBuiltInMessage`,匹配 `DEFAULT_BUILT_IN_MESSAGES`。
- `match` 为函数:`FormCustomMessage`,生成 `customMatcherEntry`(useId 作 id)注册到 Form,显示条件为"无内置错误且 `customErrors[id]`"。

### 10.4 其它

- `FormField`:`{id(useId), name, serverInvalid}`,设 `data-valid`/`data-invalid`。
- `FormLabel`:包 `LabelPrimitive`,`htmlFor` 默认取 field id。
- `FormValidityState`:render-prop 暴露原始 `ValidityState`。
- `FormSubmit`:`type=submit` button。
- **ServerErrors**:通过 `FormField` 的 `serverInvalid` prop 注入,`onClearServerErrors` 在 submit/reset 时清除。

## 15. 版本与发布流程

### 15.1 Changesets

文件:[.changeset/config.json](file:///workspace/.changeset/config.json)、[changelog.cjs](file:///workspace/.changeset/changelog.cjs)

- `baseBranch: main`,`access: public`,`updateInternalDependencies: patch`(内部依赖变动只 bump patch)。
- `snapshot`:`useCalculatedVersion: true`,`prereleaseTemplate: 'rc.{timestamp}'`。
- `ignore: ["@repo/ssr-testing", "@repo/storybook"]`(内部应用不发布)。
- 自定义 changelog 生成器:`getReleaseLine`(首行作标题,多行作 body)、`getDependencyReleaseLine`(列出更新的依赖)、`getChangelogEntry`(支持 `### Overview` 排序)。
- 补丁:`patches/@changesets__apply-release-plan.patch`(通过 `pnpm-workspace.yaml` 的 `patchedDependencies` 应用)。

### 15.2 发布策略

文件:[release-process.md](file:///workspace/release-process.md)、[.github/workflows/publish.yml](file:///workspace/.github/workflows/publish.yml)

- **Tracking**:`pnpm changeset` 标记变更类型,提交 changeset 文件。
- **Stable release**:无长期 `stable` 分支,每次从 `main` 重建 `stable` → push 触发 Changeset action 创建 release PR → 合并触发发布 → 删 `stable`。发布后自动开 `stable → main` 的 backport PR 并启用 auto-merge。
- **Release candidates(snapshot)**:合并到 `main` 时 `detect-snapshot-changes` action 检测,有变更则 `publish-snapshot` action 自动发布到 npm `next` tag。
- 脚本:`pnpm bump:stable`(changeset version)、`pnpm bump:next`(snapshot)、`pnpm release:stable:ci`(changeset publish)、`pnpm release:next`(pnpm publish --tag next)。

### 15.3 提交规范

[Conventional Commits](https://www.conventionalcommits.org/),release-please 自动生成 changelog。破坏性变更用 `!` 后缀(如 `feat!:`)。

## 16. 代码规范与约定

文件:[AGENTS.md](file:///workspace/AGENTS.md)、[.oxlintrc.json](file:///workspace/.oxlintrc.json)

### 16.1 代码约定

- 控制流(if/for/while)与 switch case **优先用花括号**,而非单行。
- `never` 分支显式检查(`value satisfies never`)以增强穷尽性。
- 偏好清晰命名,而非缩写。
- 文件结构扁平,避免过早抽象。

### 16.2 Lint 规则要点

- 插件:`react` + `jsx-a11y` + `typescript`(TS 文件 override)。
- `react/rules-of-hooks: error`、`react/jsx-no-undef: error`、`react/no-render-return-value: error`、`react/style-prop-object: error`。
- `no-restricted-globals` 禁用一大批全局(避免隐式全局,如 `addEventListener`/`focus`/`open` 等需显式 `document.`/`window.`)。
- `no-var: off`(用 `prefer-const: warn` 替代)。
- `jsx-a11y` 覆盖 alt-text / aria-* / role-* / label-has-associated-control(controlComponents: ["Checkbox"])等。
- `ignorePatterns: ["dist/**", ".next/**"]`。

### 16.3 Pre-commit / PR checklist

- Pre-commit:`pnpm lint --fix` 修可自动修复项,再修剩余;`pnpm format`。
- PR:`pnpm format:check`、`pnpm lint`、`pnpm test`、`pnpm types:check` 全过;Conventional Commit 格式。

### 贯穿全局的设计约定

1. **Scoped Context**:`createContextScope(NAME, [deps])` + `__scopeXxx` 透传,解决同名组件多层嵌套 context 串扰。
2. **asChild + Slot**:外壳组件统一支持 `asChild`,通过 `react-slot`/`createSlot` 合并 props 到消费者元素。
3. **useControllableState**:受控/非受控统一,带 controlled↔uncontrolled 切换告警。
4. **data-* 状态暴露**:`data-state` / `data-disabled` / `data-orientation` / `data-side` / `data-highlighted`,样式与状态语义统一。
5. **BubbleInput 桥接**:button-based 控件挂隐藏原生 input/select,通过原生 prototype setter + dispatch 事件让原生 form 校验正常工作——"自定义外观但保留原生表单语义"。
6. **Collection 解耦 DOM**:子项自注册、父项按集合顺序访问,支持动态增删与虚拟化(legacy 用 querySelectorAll,unstable 用 OrderedDict + compareDocumentPosition)。
7. **dispatchDiscreteCustomEvent**:`flushSync` 保证自定义事件在 discrete 事件内更新可预测。
