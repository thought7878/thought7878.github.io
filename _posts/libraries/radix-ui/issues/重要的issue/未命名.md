# Bug 分析:

## 问题现象
在 Dialog 中嵌套使用 Select 组件时,打开 Select 下拉框后,点击 Dialog.Content(非 Select 下拉部分)会导致 **Dialog 意外关闭**。
 
复现步骤:
1. 创建一个 Dialog
2. 在 Dialog.Content 内放置 Select
3. 打开 Dialog → 打开 Select 下拉 → 点击 Dialog.Content
4. Dialog 被关闭(应该只关闭 Select)

## 根本原因

根据维护者 TylerGennaro 的分析,这是 **Radix UI 行为叠加**导致的:

- 当 Select 在 Dialog 中打开时,Radix 会给 Dialog.Content 添加 `pointer-events: none`
- 这意味着用户的所有点击实际上穿透到下层的 **DialogOverlay**
- Dialog 把对 overlay 的点击判定为"点击外部",于是触发关闭逻辑

换句话说:用户以为点的是 Dialog 内容区,实际点中的是 DialogOverlay,被 Dialog 误判为"外部点击"。

涉及的包版本:
- `@radix-ui/react-dialog: ^1.1.17`
- `@radix-ui/react-select: ^2.3.1`

## 解决方案

社区给出修复思路:

### 方案:强制恢复 pointer-events
在 `dialog.tsx` 的 `DialogPrimitive.Content` className 中加入 `pointer-events-auto!`(尾部 `!` 表示 important):

```tsx
<DialogPrimitive.Content
  ref={ref}
  className={cn(
    'pointer-events-auto! fixed left-1/2 top-1/2 z-50 ...',
    className,
  )}
  {...props}
>
```

这样 DialogContent 强制接受点击事件,既阻止了点击穿透到 overlay,又不影响 Select 自身的"点击外部关闭"行为。
