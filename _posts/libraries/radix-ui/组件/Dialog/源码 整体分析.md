
>详细分析 Dialog 源码

## 一、整体架构

Dialog 是一个**复合组件(Compound Component)**,通过 Context 共享状态,各子组件各司其职:

```
Dialog              ← 状态容器 + Context Provider
├── DialogTrigger   ← 触发器(button)
├── DialogPortal    ← Portal + Presence 控制
│   ├── DialogOverlay  ← 遮罩层(仅 modal)
│   └── DialogContent  ← 内容主体
│       ├── DialogTitle
│       └── DialogDescription
└── DialogClose     ← 关闭按钮
```

依赖的核心 primitive 见 [package.json](file:///workspace/packages/react/dialog/package.json#L37-L52):`react-dismissable-layer`、`react-focus-scope`、`react-portal`、`react-presence`、`react-remove-scroll`、`aria-hidden` 等。

## 二、核心组件解析

### 1. Dialog — 根状态容器

[dialog.tsx#L50-L84](file:///workspace/packages/react/dialog/src/dialog.tsx#L50-L84)

- 使用 `useControllableState` 实现**受控/非受控双模式**:`open`/`defaultOpen`/`onOpenChange`
- 通过 `createContextScope` 创建带 scope 隔离的 Context,支持嵌套同类型组件互不干扰
- 持有 `triggerRef`、`contentRef`,以及由 `useId()` 生成的 `contentId`、`titleId`、`descriptionId`,用于 ARIA 关联
- `modal` 默认 `true`,决定 Content 走 Modal 还是 NonModal 分支

### 2. DialogTrigger — 触发器

[dialog.tsx#L98-L116](file:///workspace/packages/react/dialog/src/dialog.tsx#L98-L116)

- 渲染为 `Primitive.button`,`type="button"` 防止表单提交
- ARIA 属性:`aria-haspopup="dialog"`、`aria-expanded`、`aria-controls`(仅在 open 时指向 contentId)
- `data-state` 用于 CSS 动画( `open` / `closed` )
- `composeEventHandlers` 合并用户传入的 `onClick` 与内部 `onOpenToggle`,用户回调优先且可通过 `preventDefault` 阻止默认行为

### 3. DialogPortal — Portal + Presence 层

[dialog.tsx#L145-L159](file:///workspace/packages/react/dialog/src/dialog.tsx#L145-L159)

- `Presence present={forceMount || context.open}` 控制**挂载/卸载时机**,配合 CSS 动画
- `PortalPrimitive asChild` 把子节点 portal 到指定 container(默认 document.body)
- 单独的 `PortalProvider` 向下传递 `forceMount`,让 Overlay/Content 可继承

### 4. DialogOverlay — 遮罩层

[dialog.tsx#L178-L224](file:///workspace/packages/react/dialog/src/dialog.tsx#L178-L224)

- **仅在 `context.modal` 为 true 时渲染**,这是 modal 与非 modal 的关键区别之一
- 用 `react-remove-scroll` 的 `RemoveScroll` 包裹:
  - `allowPinchZoom`:允许双指缩放
  - `shards={[context.contentRef]}`:把 Content 也加入"可滚动分片",避免 Content 内部滚动被锁定
  - `as={Slot}`:通过 `createSlot('DialogOverlay.RemoveScroll')` 让 RemoveScroll 能透传 props 给真正的 DOM 节点
- 调用 `useDismissableLayerSurface()` 注册为 dismissable surface,即使消费者在 overlay 上 `stopPropagation` 仍能正确关闭(修复 [issue #3346](https://github.com/radix-ui/primitives/issues/3346))
- `pointerEvents: 'auto'` 重置,因为 Content 会临时禁用外层 pointer-events

### 5. DialogContent — 内容主体(核心)

[dialog.tsx#L241-L256](file:///workspace/packages/react/dialog/src/dialog.tsx#L241-L256) 入口:
- 根据 `context.modal` 分发到 `DialogContentModal` 或 `DialogContentNonModal`
- `Presence present={forceMount || context.open}` 控制挂载

#### Modal 版本 [dialog.tsx#L268-L309](file:///workspace/packages/react/dialog/src/dialog.tsx#L268-L309)

- **`hideOthers(content)`**:用 `aria-hidden` 隐藏页面其余内容,这是 `aria-modal` 的更好替代方案(浏览器支持更广泛)
- `trapFocus={context.open}`:仅在 open 时陷阱焦点(closed ≠ unmounted,动画退出时不该再 trap)
- `disableOutsidePointerEvents={context.open}`:阻止外层交互
- `onCloseAutoFocus`:关闭后**把焦点还给 trigger**
- `onPointerDownOutside`:**右键和 Ctrl+左键不关闭**(等价于右键 overlay)
- `onFocusOutside`:Modal 下 focus 必须被困在内部,所以总是 `preventDefault()`

#### NonModal 版本 [dialog.tsx#L313-L365](file:///workspace/packages/react/dialog/src/dialog.tsx#L313-L365)

- `trapFocus={false}`、`disableOutsidePointerEvents={false}`:不陷阱焦点、不阻断外层交互
- `hasInteractedOutsideRef` / `hasPointerDownOutsideRef` 两个 ref 记录交互状态:
  - 若用户点击 trigger 关闭,**不让 onInteractOutside 再次触发**(否则会 close→reopen)
  - 处理 Safari 边界 case:trigger 在 `tabIndex={0}` 容器内时,pointerdown 外 → focusin 外,需要忽略后续 focusin
- 关闭时焦点归还策略:若用户没在外部交互,则回到 trigger;否则交给浏览器

#### 共同实现 DialogContentImpl [dialog.tsx#L393-L426](file:///workspace/packages/react/dialog/src/dialog.tsx#L393-L426)

- `useFocusGuards()`:在 DOM 树末尾插入 focus guard(tabindex),防止 Tab 跳出 Portal
- `FocusScope`:`loop` 循环焦点、`trapped` 控制陷阱、`onMountAutoFocus` / `onUnmountAutoFocus` 处理开关自动聚焦
- `DismissableLayer`:
  - `role="dialog"`、`aria-describedby`、`aria-labelledby` 完整 ARIA
  - `deferPointerDownOutside`:延迟外部点击检测,让动画类库先处理
  - `onDismiss`:Esc 关闭 → `onOpenChange(false)`

### 6. DialogTitle / DialogDescription

[dialog.tsx#L438-L466](file:///workspace/packages/react/dialog/src/dialog.tsx#L438-L466)

- Title 渲染为 `h2`,绑定 `id={context.titleId}`
- Description 渲染为 `p`,绑定 `id={context.descriptionId}`
- Content 通过 `aria-labelledby` / `aria-describedby` 自动关联,无需用户手动连线

### 7. DialogClose

[dialog.tsx#L477-L490](file:///workspace/packages/react/dialog/src/dialog.tsx#L477-L490)

- 渲染为 `button type="button"`,`onClick` 调用 `onOpenChange(false)`
- 同样用 `composeEventHandlers` 合并用户回调

## 三、关键技术点总结

### 1. 状态管理双模式
通过 `useControllableState` 同一 API 同时支持受控(`open` + `onOpenChange`)与非受控(`defaultOpen`),`caller: DIALOG_NAME` 用于 dev 警告定位。

### 2. Modal 与 NonModal 的差异

| 特性 | Modal | NonModal |
|------|-------|----------|
| Overlay | 渲染 | 不渲染 |
| 焦点陷阱 | open 时 trap | 不 trap |
| 外层 pointer events | 禁用 | 允许 |
| `aria-hidden` 其余内容 | 是(hideOthers) | 否 |
| 点击外部 | 关闭(右键除外) | 关闭,但处理 trigger 边界 |
| 关闭后焦点 | 归还 trigger | 视交互情况归还 |

### 3. 事件合成的统一模式
`composeEventHandlers(props.onXxx, internalHandler)`:
- 用户回调**先执行**
- 用户调用 `event.preventDefault()` 即可阻止默认行为
- 体现了 Radix "可控但不过度限制"的设计哲学

### 4. 动画友好的生命周期
`Presence` + `forceMount` 让用户能:
- 用 CSS keyframes 配合 `data-state` 做进出动画
- 通过 `forceMount` 配合 React 动画库(如 framer-motion)完全控制挂载

### 5. Scope 隔离机制
`createContextScope` 让每个 Dialog 实例可拥有独立的 scope,在嵌套 Dialog / 与其他 Radix 组件组合时不会 Context 冲突。所有 `useDialogContext` 都接收 `__scopeDialog` 参数。

### 6. 可访问性的"防御性"实现
- 不直接用 `aria-modal="true"`(浏览器支持参差),而用 `aria-hidden` 隐藏其余 DOM
- Focus guards 防止 Tab 跳出 Portal
- 完整的 `aria-labelledby`/`aria-describedby` 自动关联
- 右键、Ctrl+点击在 Modal 下不关闭(更符合直觉)

## 四、典型使用范式

```tsx
<Dialog>
  <DialogTrigger>Open</DialogTrigger>
  <DialogPortal>
    <DialogOverlay />
    <DialogContent>
      <DialogTitle>Title</DialogTitle>
      <DialogDescription>Description</DialogDescription>
      <DialogClose>Close</DialogClose>
    </DialogContent>
  </DialogPortal>
</Dialog>
```

这种显式组合的方式让用户能自由调整结构(如不渲染 Overlay、把 Content 放在不同 Portal),同时保持可访问性约束。
