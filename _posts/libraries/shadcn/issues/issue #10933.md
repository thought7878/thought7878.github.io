### 2. 问题背景

该 Issue 指出 shadcn-ui 的 `calendar` 组件在 **base-nova** 风格（以及其他风格）中使用了错误的 `react-day-picker` classNames 键名 `table`，而 `react-day-picker` v9+ 版本已将此键名重命名为 `month_grid`。

**问题起源**：`react-day-picker` 从 v8 升级到 v9/v10 时，将 `ClassNames` 类型中的 `table` 键重命名为 `month_grid`。shadcn-ui 的 registry 声明的依赖为 `react-day-picker@latest`（当前 v10.0.1），但其 `calendar` 组件的 `classNames` 对象中仍保留了旧的 `table` 键。

---

### 3. 复现步骤

1. 新建一个 Next.js 16 + Tailwind v4 项目
2. 执行 `npx shadcn init`，选择 **base-nova** 风格
3. 执行 `npx shadcn add calendar`（安装 `react-day-picker@latest`，当前为 v10.0.1）
4. 执行 `npx tsc --noEmit`

**结果**：TypeScript 编译报错 `TS2353`

---

### 4. 预期行为

生成的 `calendar` 组件应该能够针对所安装的 `react-day-picker` 版本正确编译，不出现 TypeScript 类型错误。

### 7. 问题根本原因分析

**核心原因**：`react-day-picker` v9+ 的 API 变更未被 shadcn-ui 完整适配。

具体来说，`react-day-picker` v9 将 `ClassNames` 接口中的以下键进行了重命名：
- `table` → `month_grid`

shadcn-ui 的 `calendar.tsx` 第 94 行使用了旧的 `table` 键：
```typescript
table: "w-full border-collapse",
```

而 `react-day-picker` v10 的 `ClassNames` 类型只接受 `month_grid`，不接受 `table`。因此 TypeScript 在严格类型检查下报错 TS2353。

**为何是隐蔽的 Bug**：
- 该组件的其他 `classNames` 键（如 `weekdays`、`weekday`、`week`、`day`、`month_caption` 等）已经正确地使用了 v9/v10 的 API
- 唯独 `table` 这一个键未被迁移，属于部分迁移遗漏
- 运行时可能不会报错（CSS 类仍然会被应用），但 TypeScript 编译会失败

---

### 8. 影响范围

根据 PR #10940 的描述，该问题**不限于 base-nova 风格**，而是影响所有风格的 `calendar` 组件：
- 10 个源 `.tsx` 文件受影响
- 18 个 registry JSON 文件受影响
- 涉及风格包括：base-nova, base-mira, base-maia, base-lyra, base-vega, radix-*, new-york-v4, default, new-york

---

### 9. 已提出的修复方案

**Issue 提出者建议**（[描述中的 Suggested fix](https://github.com/shadcn-ui/ui/issues/10933#issue-4654193693)）：

```diff
- table: "w-full border-collapse",
+ month_grid: cn("w-full border-collapse", defaultClassNames.month_grid),
```

**PR #10940**（[mixelburg](https://github.com/mixelburg) 提交）：
- 将所有 10 个源 `.tsx` 文件中的 `table` 重命名为 `month_grid`
- 同时更新了 18 个 registry JSON 文件（包括已废弃的 www 版本）
- PR 状态：等待审核和合并，Vercel 部署需要授权，GitHub Actions 要求签署 commit

### 12. 隐含问题与额外发现

1. **部分迁移问题**：Issue 报告者指出其他 `classNames` 键已正确迁移至 v9/v10 API，唯独 `table` 被遗漏。这暗示代码库中可能存在类似的**部分迁移遗漏**，建议对其他组件进行系统性检查。

2. **registry JSON 与源文件同步问题**：PR 需要同时更新 10 个 `.tsx` 和 18 个 JSON 文件，说明 registry 的 JSON 配置文件与源文件之间存在手动同步的需求，这种同步机制容易出错。

3. **依赖版本策略问题**：registry 使用 `react-day-picker@latest` 作为依赖声明，这意味上游 API 变更会直接影响下游用户。建议考虑使用锁定版本范围（如 `^9.0.0` 或 `^10.0.0`），而非裸 `latest`。

4. **测试覆盖缺失**：该 Bug 能存活到生产环境，说明可能缺少针对 TypeScript 类型检查的 CI 步骤，或测试未覆盖所有风格变体。
