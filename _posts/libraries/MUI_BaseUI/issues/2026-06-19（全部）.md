# MUI Base UI 最近6个月未解决 Bug 系统性总结

基于对 https://github.com/mui/base-ui/issues 的全面分析，以下是最近6个月（2025年12月-2026年6月）的未解决 bug 总结。

---

## 一、严重级别：Critical（关键）

### 4. **[dialog] 带 handle 的 Dialog 在路由切换后状态未清理**
- **链接**: https://github.com/mui/base-ui/issues/4951
- **问题描述**: 当 Dialog 使用 `handle` 控制时，在 Dialog 打开状态下导航离开再返回，Dialog 会保持之前的状态。在官方文档的"Controlled mode with multiple triggers"示例中，点击触发器后导航离开再返回，页面会完全无法操作。
- **复现步骤**:
  1. 打开带 handle 的 Dialog
  2. 在 Dialog 打开状态下导航离开
  3. 返回原页面
  4. 观察到 Dialog 保持之前状态或页面无法操作
- **预期行为**: Dialog 应正确清理状态
- **实际行为**: Dialog 保持旧状态或页面被锁定
- **环境信息**: Base UI v1.5.0, Chrome, macOS
- **当前状态**: 已确认 bug
- **影响**: 严重影响路由切换场景下的用户体验

## 二、严重级别：High（高）

### 9. [menu] 子菜单项 VoiceOver 播报问题
- **链接**: https://github.com/mui/base-ui/pull/5067
- **问题描述**: 当通过键盘（箭头键、Enter、Space）打开子菜单时，VoiceOver（Chrome 和 Safari）不会播报初始聚焦的子菜单项。NVDA 不受影响。
- **复现步骤**:
  1. 使用键盘导航打开子菜单
  2. VoiceOver 不会播报聚焦的菜单项
- **预期行为**: VoiceOver 应正确播报子菜单项
- **实际行为**: 无播报
- **环境信息**: Chrome/Safari, VoiceOver
- **当前状态**: 已确认 bug，有 PR 在修复中

## 三、严重级别：Medium（中）

### 15. [context menu] ContextMenuTrigger 长按不支持触控笔输入
- **链接**: https://github.com/mui/base-ui/issues/4854
- **问题描述**: `ContextMenuTrigger` 的长按路径使用 TouchEvents，不支持触控笔输入。Apple Pencil 在 iOS Safari 上仅派发 PointerEvents，大多数 Surface pen 和 Wacom 平板也是如此。
- **复现步骤**:
  1. 使用触控笔（如 Apple Pencil）在 ContextMenuTrigger 上长按
  2. 上下文菜单不出现
- **预期行为**: 触控笔长按应触发上下文菜单
- **实际行为**: 无响应
- **环境信息**: iOS Safari, Apple Pencil; 部分 Android 平板和 Wacom 平板
- **当前状态**: 已确认 bug，有 PR 在修复中

### 16. [docs] StackBlitz 链接重复
- **链接**: https://github.com/mui/base-ui/issues/5044
- **问题描述**: 文档页面中 StackBlitz 链接出现两次。
- **复现步骤**:
  1. 访问动画文档页面
  2. 观察到 StackBlitz 链接重复显示
- **预期行为**: 只显示一个链接
- **实际行为**: 显示两个链接
- **环境信息**: 所有环境
- **当前状态**: 已分配处理

### 17. [popups] hover、focus 和 anchor 定位多个 bug
- **链接**: https://github.com/mui/base-ui/pull/5037
- **问题描述**: 共享的 hover/focus 交互 hooks 和 anchor 定位存在多个潜在 bug，包括：
  - `useFocus` 在窗口失焦时未正确记录阻塞引用
  - `useHoverInteractionSharedState` 的引用计数问题导致 popup 卸载时可能取消触发器的待打开状态
  - `useHoverReferenceInteraction` 从布局提交的 ref 读取 `handleClose` 选项导致过期
  - `useAnchorPositioning` 在 `transformOrigin` 中间件中读取 `sideOffset` 过期
- **复现步骤**: 多种场景，涉及复杂的 hover/focus 交互
- **预期行为**: hover/focus 交互应稳定可靠
- **实际行为**: 存在多种边缘情况 bug
- **环境信息**: 所有环境
- **当前状态**: 已确认 bug，有 PR 在修复中（草稿状态）

## 五、关键发现与趋势

### 1. **Dialog 组件问题最为严重**
- 4 个 Critical 级别 bug，涉及状态管理、滚动锁定、Escape 键处理等核心功能
- 多实例场景下的协调问题尤为突出

### 2. **React StrictMode 兼容性问题**
- Select 组件在 StrictMode 下的严重 bug 影响所有 Next.js 开发环境用户
- 需要加强 StrictMode 场景的测试覆盖

### 3. **SSR/水合相关问题**
- Avatar 和 Navigation Menu 在 SSR 场景下存在交互问题
- 需要更好地处理水合前后的状态同步

### 4. **动画/过渡期间的定位问题**
- Tabs 和 Menu 在动画期间的定位和渲染存在问题
- 需要改进动画完成后的位置计算逻辑

### 5. **移动端和辅助功能**
- iOS 键盘打开时的视口高度计算问题
- 触控笔输入支持不足
- VoiceOver 播报问题

### 6. **跨库互操作性**
- 与其他 UI 库（如 Ariakit）的滚动锁定协调问题
- 需要更好地处理跨库的状态交接

## 六、建议优先级

1. **立即修复**: Dialog 相关的 4 个 Critical bug（#4698, #4583, #4640, #4951）
2. **高优先级**: Menu、Tabs、Avatar 的 High 级别 bug
3. **中优先级**: 其余 Medium 级别 bug
4. **持续改进**: 加强 SSR、StrictMode、移动端场景的测试覆盖

所有列出的 bug 均已由维护者确认，大部分有相关 PR 在修复中。建议优先关注影响核心交互功能的 Dialog 和 Select 问题。
