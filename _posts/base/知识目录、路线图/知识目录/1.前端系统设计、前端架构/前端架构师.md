
# 前端架构师能力与知识点体系

## 一、技术深度能力

### 1. JavaScript / TypeScript 核心功底
- **语言机制**：执行上下文、作用域链、闭包、原型链、this 绑定、事件循环（宏任务/微任务）
- **异步编程**：Promise 原理、async/await、Generator、异步并发控制
- **TypeScript**：类型系统、泛型、类型体操、声明文件、类型推导、模块系统
- **ES6+ 新特性**：Proxy/Reflect、Symbol、Iterator/Generator、装饰器

### 2. 框架与生态深度
- **React**：Fiber 架构、Reconciler、Hooks 原理、Concurrent Mode、Suspense、Server Components
- **Vue**：响应式系统、模板编译、Composition API、Vue 3 编译优化
- **状态管理**：Redux 中间件机制、MobX 响应式、Zustand、Jotai、Recoil 设计思想
- **路由体系**：History API、Hash 路由、动态路由、嵌套路由、路由守卫

### 3. CSS 与样式架构
- **布局体系**：Flex、Grid、Container Queries、Subgrid
- **样式方案**：CSS Modules、CSS-in-JS、Tailwind/UnoCSS、CSS Variables
- **主题化**：Design Token、暗色模式、多主题切换
- **CSS Houdini**、`@layer`、`:has()` 等新特性

## 二、工程化能力

### 1. 构建工具链
- **Webpack**：Loader/Plugin 机制、Tapable、Module Federation、构建优化
- **Vite/Rollup**：ESM 预构建、HMR 原理、Tree Shaking、插件机制
- **esbuild/SWC**：底层编译原理、性能优势
- **Turbopack/Rspack**：新一代构建工具

### 2. 包管理与 Monorepo
- **pnpm/Yarn Berry**：依赖解析、Workspace、PnP
- **Monorepo**：Turborepo、Nx、Lerna、Changesets
- **包发布策略**：Semver、Beta/RC 流程、Peer Deps

### 3. 代码质量体系
- **静态分析**：ESLint 自定义规则、TypeScript 严格模式
- **代码规范**：Prettier、EditorConfig、Commitlint、Husky
- **测试体系**：Vitest/Jest 单元测试、Playwright E2E、组件测试、覆盖率
- **CI/CD**：GitHub Actions、GitLab CI、自动化发布流水线

## 三、性能优化能力

### 1. 加载性能
- **指标体系**：Core Web Vitals（LCP/FID/CLS/INP）、TTFB、FCP
- **资源优化**：代码分割、懒加载、预加载、图片优化、字体优化
- **网络优化**：HTTP/2 多路复用、HTTP/3、CDN、缓存策略（强缓存/协商缓存）
- **SSR/SSG/ISR**：Next.js、Nuxt、Astro 渲染策略

### 2. 运行时性能
- **渲染优化**：虚拟列表、防抖节流、requestAnimationFrame、避免重排重绘
- **内存管理**：内存泄漏排查、垃圾回收、WeakMap/WeakSet
- **长任务优化**：Time Slicing、Web Worker、OffscreenCanvas
- **性能监控**：Performance API、Lighthouse、Web Vitals、火焰图

## 四、架构设计能力

### 1. 组件架构
- **组件设计原则**：单一职责、开放封闭、组合优于继承
- **设计模式**：Headless UI、Compound Component、Render Props、HOC、Hooks
- **组件库建设**：Design System、Token 体系、可访问性（a11y）、主题化
- **微前端**：qiankun、Module Federation、Single-SPA、Web Components

### 2. 应用架构
- **分层架构**：UI 层、业务逻辑层、数据层、基础设施层
- **状态架构**：本地状态 vs 全局状态、服务端状态（React Query/SWR）、URL 状态
- **API 设计**：RESTful、GraphQL、tRPC、BFF 层
- **错误边界**、**降级策略**、**灰度方案**

### 3. 基础设施
- **脚手架**：CLI 工具、模板生成、项目初始化
- **公共组件库**、**工具函数库**、**业务组件库**的分层与治理
- **文档站点**：Storybook、Docusaurus、VitePress

# 前端架构师能力分析

前端架构师是技术深度、工程广度与业务理解的综合体。下面从**职责、能力、知识点**三个维度展开。

## 一、核心职责

### 1. 技术架构设计
- 制定前端整体技术方案（技术选型、分层架构、模块划分）
- 设计跨团队、跨端（Web/Mobile/SSR/SSG/微前端）的统一架构
- 制定长期技术路线图（Roadmap），平衡短期交付与长期演进

### 2. 工程体系建设
- 搭建并维护前端工程化链路：脚手架、构建、CI/CD、发布、监控
- 建立代码规范与质量门禁（Lint、TypeScript、测试覆盖率、Code Review 机制）
- 推动组件库、工具链、设计系统（Design System）建设

### 3. 性能与质量保障
- 制定性能指标（Core Web Vitals、首屏、TTI）与优化策略
- 建立监控告警体系（错误、性能、行为埋点）
- 主导重大技术问题攻坚与线上故障复盘

### 4. 技术领导力
- 指导高级/资深工程师，做技术决策与方案评审
- 跨团队协作（后端、设计、产品、测试），对齐接口与协议
- 技术布道：推动最佳实践落地，组织技术分享

### 5. 业务支撑
- 理解业务目标，将业务需求转化为可演进的技术方案
- 识别技术债，制定偿还计划，避免架构腐化
- 评估新技术对业务的实际价值，避免盲目追新

## 三、知识点地图

### 1. 计算机基础
- 数据结构与算法、设计模式
- 操作系统、网络（HTTP/1.1、HTTP/2、HTTP/3、QUIC、TCP）
- 编译原理基础（AST、词法/语法分析）

### 2. 浏览器与运行时
- 渲染管线（DOM → Style → Layout → Paint → Composite）
- 事件循环（宏任务/微任务）、垃圾回收、内存模型
- 同源策略、CSP、CORS、Cookie/Session、安全（XSS/CSRF）
- WebAssembly、WebWorker、Service Worker、SharedArrayBuffer

### 3. 语言与框架
- **JavaScript**：ES6+、闭包、原型链、异步、模块化（ESM/CJS）
- **TypeScript**：类型系统、泛型、类型体操、声明文件设计
- **React**：Fiber、Concurrent、Suspense、Server Components、状态管理
- **Vue / Svelte / Solid**：响应式原理、编译优化差异
- **Node.js**：事件循环、流、模块、性能调优

### 4. 架构模式
- 组件化、模块化、分层架构（Clean Architecture、Hexagonal）
- 微前端（qiankun、Module Federation、Single-SPA、Web Components）
- BFF（Backend for Frontend）、GraphQL、RPC、tRPC
- SSR / SSG / ISR / Streaming SSR、边缘渲染
- 状态管理范式（Flux、原子化、Signal、Proxy）

### 5. 工程化
- 构建工具：Webpack、Vite、Rollup、esbuild、Turbopack、Rspack
- Monorepo：pnpm workspace、Turborepo、Nx、Lerna
- 包管理、版本策略（SemVer、Changesets）
- 代码质量：ESLint、Prettier、Stylelint、TypeScript、Husky
- 测试体系：单元（Vitest/Jest）、集成、E2E（Playwright）、视觉回归

### 6. 性能与可观测
- 性能指标：LCP、FID/INP、CLS、TTFB、FCP
- 性能工具：Lighthouse、Performance、Coverage、火焰图
- 监控：Sentry、自研 SDK、埋点、APM
- 优化手段：代码分割、预加载、缓存、CDN、HTTP/2、图片优化、虚拟列表

### 7. 跨端与全栈
- React Native / Flutter / Electron / Tauri
- 小程序生态、Hybrid、JSBridge
- 容器化（Docker）、Serverless、边缘函数（Edge Functions）

### 8. 安全与合规
- XSS、CSRF、SSRF、供应链安全、依赖审计
- 数据隐私（GDPR、个人信息保护法）、内容安全策略

### 9. 软技能与方法论
- 技术写作、RFC/ADR（架构决策记录）
- 敏捷、OKR、技术规划
- 团队建设、人才培养、技术文化

## 四、成长路径建议

1. **深耕一个方向**：先把某一框架/领域做到极致（如 React 生态、性能、工程化）。
2. **建立系统观**：从单点问题跳到全链路思考（请求 → 构建 → 运行 → 监控）。
3. **做有影响力的项目**：主导一次架构升级、一次性能战役、一个组件库。
4. **输出与影响力**：写技术博客、做分享、参与开源、写 ADR。
5. **补齐业务与软技能**：架构师的价值最终通过业务结果和团队提升体现。
