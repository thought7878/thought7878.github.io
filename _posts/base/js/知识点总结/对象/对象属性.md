在 JavaScript 中，对象属性*远不止“键值对”那么简单*。它是***带元数据的描述符系统***，与原型链、V8 引擎优化、响应式框架底层紧密耦合。掌握属性机制，是写出高性能、可维护代码的必修课。

---

## 一、核心概念：**属性的本质**

JS 对象的每个属性都由三部分组成：
1. **键（Key）**：只能是 `String` 或 `Symbol`
2. **值（Value）**：任意 JS 数据类型
3. **描述符（Descriptor）**：控制属性的行为（可写、可枚举、可配置、getter/setter）

属性分为两类：
- `自有属性（Own Property）`：直接定义在对象自身上
- `继承属性（Inherited Property）`：通过原型链查找获得

---

## 二、属性的两种类型

### 1. 数据属性（Data Property）
**存储具体值，描述符**包含：

| 标志位            | 说明                                          | 默认值（字面量创建） |
| -------------- | ------------------------------------------- | ---------- |
| `value`        | 属性值                                         | 赋值的内容      |
| `writable`     | 是否可修改值                                      | `true`     |
| `enumerable`   | *是否可被枚举（`for...in`/`Object.keys`）*。枚举/列举/遍历 | `true`     |
| `configurable` | *是否可删除/修改描述符/转为访问器属性*                       | `true`     |

### 2. 访问器属性（Accessor Property）
**不直接存值，由 `get`/`set` 函数拦截读写，描述符**包含：

| 标志位            | 说明                |
| -------------- | ----------------- |
| `get`          | 读取时调用的函数，返回值作为属性值 |
| `set`          | 赋值时调用的函数，接收新值作为参数 |
| `enumerable`   | 同上                |
| `configurable` | 同上                |
⚠️ 注意没有上面的`value`、`writable`
⚠️ **互斥规则**：数据属性与访问器属性**不能共存：** *一个属性要么有 `value/writable`，要么有 `get/set`*。

```js
const obj = {
  _age: 20,
  get age() { return this._age; },
  set age(v) { if (v > 0) this._age = v; }
};
console.log(Object.getOwnPropertyDescriptor(obj, 'age'));
// { get: ƒ, set: ƒ, enumerable: true, configurable: true }
```

## 三、属性描述符操作 API

### 1. 读取描述符
```js
Object.getOwnPropertyDescriptor(obj, prop)   // 单个自有属性
Object.getOwnPropertyDescriptors(obj)        // 所有自有属性（ES2017）
```

### 2. 定义/修改属性
```js
Object.defineProperty(obj, prop, descriptor)
Object.defineProperties(obj, descriptors)
```
📌 **关键差异**：通过 `defineProperty` 创建的属性，`writable/enumerable/configurable` **默认均为 `false`**。

```js
const obj = {};
Object.defineProperty(obj, 'id', { value: 1 });
console.log(obj.id); // 1
obj.id = 2;          // 静默失败（严格模式报错）
console.log(obj.id); // 1
```

> 💡 *Vue 2 响应式原理*正是基于 `Object.defineProperty` 拦截 getter/setter。

## 四、属性键（Key）的规则与陷阱

### 1. 键的类型限制
- 仅支持 `String` 和 `Symbol`
- 其他类型会**隐式调用 `ToString()`** 转换：
```js
const obj = {};
obj[1] = 'a';        // 转为 "1"
obj[true] = 'b';     // 转为 "true"
obj[{}] = 'c';       // 转为 "[object Object]"
obj[{id:1}] = 'd';   // 同样转为 "[object Object]"，覆盖上一条！
console.log(obj);    // { "1": "a", "true": "b", "[object Object]": "d" }
```

### 2. Symbol 键的特性
- **唯一性，永不冲突**
- 默认**不可枚举**（`for...in`、`Object.keys` 会忽略）
- *适合做“隐藏属性”或框架内部标识*

```js
const SECRET = Symbol('secret');
const user = { name: 'Alice', [SECRET]: 'token123' };
console.log(Object.keys(user)); // ["name"]
console.log(user[SECRET]);      // "token123"
```

## 五、属性的访问、赋值与原型链交互

### 1. 访问规则
```js
obj.key 或 obj[key]
// 查找顺序：自有属性 → 原型链 → 找不到返回 undefined
```

### 2. 赋值规则（极易踩坑 ⚠️）
`obj.prop = value` 的行为取决于原型链：

| 原型链状态 | 赋值结果 |
|------------|----------|
| 无同名属性 | 在 `obj` 自身创建新属性 |
| 有同名数据属性且 `writable: true` | 修改原型上的值（不推荐） |
| 有同名数据属性且 `writable: false` | **静默失败**（严格模式抛 `TypeError`） |
| 有同名访问器属性（setter） | **调用 setter**，不会创建自有属性 |

```js
const proto = { count: 0 };
const obj = Object.create(proto);
obj.count = 5; // 在 obj 自身创建 count，屏蔽原型（Shadowing）
console.log(proto.count); // 0（原型未变）
```

### 3. 删除属性
```js
delete obj.prop
// 仅删除自有属性，返回 boolean
// 严格模式下删除 configurable: false 的属性会抛错
// 无法删除原型链上的属性
```

## 六、属性*遍历*与*检测* API 对比（核心）

| API                              | 范围          | 包含继承？ | 包含不可枚举？ | 包含 Symbol？ |
| -------------------------------- | ----------- | ----- | ------- | ---------- |
| `for...in`                       | 键           | ✅     | ❌       | ❌          |
| `Object.keys()`                  | 键           | ❌     | ❌       | ❌          |
| `Object.values()`                | 值           | ❌     | ❌       | ❌          |
| `Object.entries()`               | `[key,val]` | ❌     | ❌       | ❌          |
| `Object.getOwnPropertyNames()`   | 键           | ❌     | ✅       | ❌          |
| `Object.getOwnPropertySymbols()` | 键           | ❌     | ✅       | ✅          |
| `Reflect.ownKeys()`              | 键           | ❌     | ✅       | ✅          |
| `prop in obj`                    | 检测          | ✅     | ✅       | ✅          |
| `Object.hasOwn(obj, prop)`       | 检测          | ❌     | ✅       | ✅          |

> ✅ **现代推荐：**  
> - *检测自有属性* → `Object.hasOwn(obj, key)`（ES2022）  
> - *完整键列表* → `Reflect.ownKeys(obj)`  
> - *安全遍历* → `Object.keys()` + `for...of`

## 七、对象保护与属性控制

| 方法 | 禁止新增 | 禁止删除/改描述符 | 禁止修改值 | 可逆？ |
|------|----------|-------------------|------------|--------|
| `Object.preventExtensions()` | ✅ | ❌ | ❌ | ❌ |
| `Object.seal()` | ✅ | ✅ | ❌ | ❌ |
| `Object.freeze()` | ✅ | ✅ | ✅ | ❌ |

```js
const config = Object.freeze({ api: 'https://x.com', timeout: 5000 });
config.timeout = 3000; // 严格模式报错，非严格静默失败
delete config.api;     // 失败
config.retry = 3;      // 失败
```
⚠️ **注意**：三者均为**浅层操作**。嵌套对象需递归冻结或使用 `deep-freeze` 库。

## 八、底层机制：V8 隐藏类与性能

V8 引擎为提升属性访问速度，引入了 **Hidden Classes（隐藏类/对象形状）**：
- 结构相同的对象共享同一个隐藏类，属性访问可优化为**固定偏移量读取**（接近 C 结构体）
- **动态增删属性、不同顺序初始化、使用 `delete`** 会导致隐藏类分裂或退化为**字典模式（Dictionary Mode）**，性能下降 5~50 倍

### **性能最佳实践**
```js
// ✅ 推荐：构造函数中声明所有属性，保持形状稳定
class Point {
  constructor(x, y) {
    this.x = x;
    this.y = y;
    this.z = 0; // 即使初始为 0 也先声明
  }
}

// ❌ 避免：运行时动态改变对象形状
const p = {};
p.x = 1;
p.y = 2;
delete p.x;        // 触发隐藏类退化
p.z = 3;           // 创建新隐藏类
```
> 现代框架（如 Vue 3、React）均避免 `delete`，改用 `null/undefined` 标记或 `Map` 管理动态键。

## 九、常见陷阱与工程建议

| 陷阱 | 现象 | 解决方案 |
|------|------|----------|
| `obj[{}]` 键冲突 | 所有对象键都转为 `"[object Object]"` | 改用 `Map` 或 `Symbol` 作键 |
| `for...in` 遍历到原型 | 意外处理继承属性 | 用 `Object.hasOwn()` 过滤或改用 `Object.keys()` |
| 赋值覆盖原型 setter | `obj.val = 1` 未创建自有属性 | 明确设计意图，或使用 `Object.defineProperty` |
| `delete` 性能杀手 | 频繁删除导致 V8 反优化 | 用 `obj.key = undefined` 替代，或改用 `Map` |
| 浅冻结泄露 | `Object.freeze({ a: { b: 1 } })` 内层仍可改 | 递归冻结或使用 Immutable.js / Immer |
| `hasOwnProperty` 被覆盖 | `obj.hasOwnProperty` 可能不是原生方法 | 统一使用 `Object.hasOwn(obj, key)` |

### **现代选型指南**

| 需求         | 推荐方案                            |
| ---------- | ------------------------------- |
| 固定结构配置/DTO | 对象字面量 + TS 接口                   |
| 动态键值对/频繁增删 | `Map`                           |
| 私有状态       | `class` 的 `#` 字段（ES2022）        |
| 属性拦截/响应式   | `Proxy`（Vue 3 / 现代库首选）          |
| 不可变数据      | `Object.freeze` + Immer / 结构共享库 |

## 十、**总结**

| 维度       | 核心结论                                   |
| -------- | -------------------------------------- |
| **本质**   | 属性 = 键 + 值 + 描述符（控制读写/枚举/配置）           |
| **类型**   | 数据属性（存值） vs 访问器属性（getter/setter）       |
| **键规则**  | 仅 String/Symbol，其他类型隐式转字符串易冲突          |
| **赋值行为** | 优先创建自有属性；受原型 `writable`/setter 影响      |
| **遍历检测** | 明确“自有/继承”“可枚举/全量”“String/Symbol”边界     |
| **性能**   | 保持对象形状稳定，避免 `delete` 与动态增删             |
| **现代实践** | 静态结构用对象，动态键用 `Map`，私有用 `#`，拦截用 `Proxy` |

> 📌 一句话记忆：  
> **对象属性不是简单的键值对，而是*带权限控制的描述符系统*。字面量创建宽松，`defineProperty` 创建严格；赋值优先建自有属性，遍历需分清自有/继承/枚举；保持形状稳定是 V8 性能的关键，现代开发应按需选用 `Map`/`Proxy`/`#` 替代传统黑魔法。**
