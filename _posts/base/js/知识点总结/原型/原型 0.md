
`原型`是 JavaScript **面向对象编程的底层基石**。与 Java/C++ 等基于“类（Class）”的继承不同，JS 采用**基于原型的委托机制**。理解原型，是掌握 JS 对象模型、继承演进、V8 性能优化及现代 class 语法糖的前提。

---

## 一、什么是原型？

- 在 ECMAScript 规范中，***几乎所有对象**都有一个内部槽 `[[Prototype]]`，它指向另一个对象（或 null），这个被指向的对象*就是该对象的***原型***。
- ***原型的作用***：***属性与方法共享***。*当访问对象属性时，若自身不存在，JS 引擎会沿着 `[[Prototype]]` 向上查找，直到找到或抵达 null*。

### 访问原型的方式
| 方式                                  | 说明             | 推荐度               |
| ----------------------------------- | -------------- | ----------------- |
| `obj.__proto__`                     | 历史遗留的非标准属性，已废弃 | ❌ 仅用于调试           |
| `Object.getPrototypeOf(obj)`        | ES5 标准读取方式     | ✅ 生产首选            |
| `Object.setPrototypeOf(obj, proto)` | ES6 标准设置方式     | ⚠️ 谨慎使用（破坏 V8 优化） |

```js
const obj = {};
console.log(Object.getPrototypeOf(obj) === Object.prototype); // true
```

## 二、原型链：属性查找的底层机制

`原型链`**是 `[[Prototype]]` 串联形成的*单向链表*结构，是 JS *实现“继承”的核心***。

### 查找规则（伪代码）
```js
function getProperty(obj, prop) {
  let current = obj;
  while (current !== null) {
    if (Object.hasOwn(current, prop)) return current[prop];
    current = Object.getPrototypeOf(current);
  }
  return undefined; // 抵达 Object.prototype.__proto__ (null)
}
```

### 原型链终点
```js
Object.getPrototypeOf(Object.prototype) === null; // true
```
所有普通对象的原型链最终都会收敛到 `Object.prototype`，再往上就是 `null`，查找终止。

## 三、构造函数、`prototype` 与 `new` 的完整关系

### 1. 函数的 `prototype` 属性
**只有函数对象**在创建时会自动获得一个 `prototype` 属性，它指向一个普通对象：
```js
function Person(name) { this.name = name; }
console.log(Person.prototype); // { constructor: ƒ Person }
```
该对象默认包含一个 `constructor` 属性，指回函数本身。

### 2. `new` 操作符的 4 步底层行为
```js
const p = new Person('Alice');
// 等价于：
// 1. 创建空对象：const obj = {};
// 2. 链接原型：Object.setPrototypeOf(obj, Person.prototype);
// 3. 绑定 this 执行：Person.call(obj, 'Alice');
// 4. 返回对象：return obj; (若构造函数返回对象则覆盖)
```
✅ 核心等式：`p.__proto__ === Person.prototype`

### 3. `instanceof` 的原理
```js
p instanceof Person; // true
// 引擎实际检查：Person.prototype 是否在 p 的原型链上
```

## 四、原型操作核心 API

| API | 作用 | 备注 |
|-----|------|------|
| `Object.getPrototypeOf(obj)` | 获取 `[[Prototype]]` | 标准读取 |
| `Object.setPrototypeOf(obj, proto)` | 设置 `[[Prototype]]` | ⚠️ 性能杀手，避免在热点代码使用 |
| `Object.create(proto)` | 以指定原型创建新对象 | 安全、高效的原型继承方式 |
| `Object.hasOwn(obj, prop)` | 检查是否为**自身属性** | ES2022 推荐，替代 `hasOwnProperty` |
| `proto.isPrototypeOf(obj)` | 检查原型关系 | `Person.prototype.isPrototypeOf(p)` |

```js
const parent = { greet() { return 'hi'; } };
const child = Object.create(parent);
console.log(child.greet()); // 'hi'（委托查找）
console.log(Object.hasOwn(child, 'greet')); // false
```

## 五、关键特性与底层规则（重点）

### 1. 原型是动态的
修改原型会影响**所有已创建的实例**（除非实例自身已覆盖该属性）：
```js
function Dog() {}
const d1 = new Dog();
Dog.prototype.bark = () => 'woof';
console.log(d1.bark()); // 'woof'（动态生效）
```

### 2. 属性屏蔽（Shadowing）
对实例**赋值**会在实例自身创建属性，**不会修改原型**：
```js
const proto = { count: 0, list: [] };
const obj = Object.create(proto);

obj.count = 1;        // 创建自身属性，屏蔽原型
obj.list.push(1);     // ⚠️ 修改的是原型上的引用对象！

console.log(proto.count); // 0（未变）
console.log(proto.list);  // [1]（被污染）
```
> 💡 **铁律**：原型上只放方法或不可变值。**引用类型属性绝不要放在原型上**。

### 3. `__proto__` vs `prototype` 终极区分
| 名称 | 所属对象 | 作用 | 规范状态 |
|------|----------|------|----------|
| `__proto__` | **实例对象** | 指向该实例的原型（即 `[[Prototype]]`） | 非标准，已废弃 |
| `prototype` | **构造函数** | 用于指定 `new` 出来实例的原型 | 标准属性 |

```js
function Fn() {}
const inst = new Fn();
inst.__proto__ === Fn.prototype; // true
```

## 六、ES6 `class` 与原型的真实关系

`class` 只是**原型继承的语法糖**，底层完全基于原型链：

```js
class Animal {
  constructor(name) { this.name = name; }
  speak() { console.log(`${this.name} makes noise`); }
  static info() { return 'base class'; }
}

// 底层等价结构：
// Animal 是函数
// Animal.prototype = { constructor: Animal, speak: ƒ }
// Animal.info 是构造函数自身的属性（静态方法）
```

| `class` 语法 | 原型层面对应 |
|--------------|--------------|
| 实例方法 | 挂载到 `Constructor.prototype` |
| 静态方法 | 挂载到 `Constructor` 自身 |
| `extends` | 设置子类构造函数的 `[[Prototype]]` 指向父类，且子类 `prototype.__proto__` 指向父类 `prototype` |
| `super()` | 调用父类构造函数并绑定 `this` |

> ✅ 现代开发优先使用 `class`，但调试、库开发、性能调优时仍需理解其原型本质。

## 七、常见陷阱与工程最佳实践

| 陷阱 | 现象 | 规避方案 |
|------|------|----------|
| **重写 `prototype` 丢失 `constructor`** | `Fn.prototype = {};` 后 `inst.constructor !== Fn` | 手动补回：`Fn.prototype.constructor = Fn;` 或使用 `Object.assign` |
| **原型上放引用类型** | 多个实例意外共享数组/对象状态 | 引用类型属性一律在构造函数内初始化 |
| **滥用 `setPrototypeOf` / `__proto__`** | V8 隐藏类失效，内联缓存退化，性能骤降 10~100 倍 | 对象创建时确定原型（`Object.create` / `class` / `new`），运行期不修改 |
| `for...in` 遍历到原型属性 | 意外处理继承来的键 | 配合 `Object.hasOwn()` 过滤，或改用 `Object.keys()` |
| `hasOwnProperty` 被覆盖 | `obj.hasOwnProperty` 可能不是原生方法 | 使用 `Object.hasOwn(obj, key)`（ES2022）或 `Object.prototype.hasOwnProperty.call(obj, key)` |

### 🛠 现代工程建议
1. **业务开发**：统一使用 `class` / `extends`，语义清晰且引擎优化完善。
2. **对象创建**：需要纯净对象时用 `Object.create(null)`（无原型链，适合做字典）。
3. **类型检查**：优先 `Object.hasOwn()` + `instanceof`，避免 `typeof` 判断对象。
4. **性能敏感场景**：避免动态增删原型属性；保持对象结构稳定以利用 V8 隐藏类（Hidden Classes）。

## 八、总结

| 维度 | 核心结论 |
|------|----------|
| **本质** | JS 没有类继承，只有**对象委托**。原型是属性查找的 fallback 机制。 |
| **链路** | `实例 → Constructor.prototype → Object.prototype → null` |
| `new` | 创建对象 + 链接原型 + 绑定 `this` + 返回实例 |
| `class` | 语法糖，底层完全映射到构造函数与原型链 |
| **性能** | 原型查找极快（V8 内联缓存），但**运行期修改原型链会触发反优化** |
| **实践** | 原型放方法，实例放数据；用 `Object.create`/`class` 管理继承；杜绝 `__proto__` 赋值 |

> 📌 一句话记忆：  
> **`prototype` 是函数的“图纸”，`[[Prototype]]` 是实例的“指针”。`new` 按图纸造对象并连上指针，属性查找沿指针向上委托，`class` 只是这套机制的现代化包装。**
