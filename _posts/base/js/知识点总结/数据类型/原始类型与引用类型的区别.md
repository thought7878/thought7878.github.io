在 JavaScript 中，数据类型被严格划分为两大阵营：**原始类型（Primitive Types）** 和 **引用类型（Reference Types / Object Types）**。理解它们的区别是掌握 JS 内存管理、深/浅拷贝、闭包以及函数传参等高级特性的基石。

以下是全方位、多维度的详细对比与总结：

---

### 一、 包含的类型

#### 1. 原始类型（共 7 种）
存储在栈（Stack）内存中的简单数据段，占据空间小、大小固定。
* `String`（字符串）
* `Number`（数字）
* `Boolean`（布尔值）
* `Undefined`（未定义）
* `Null`（空值）
* `Symbol`（ES6 引入，表示唯一值）
* `BigInt`（ES2020 引入，表示任意精度的整数）

#### 2. 引用类型（统称为 Object）
存储在堆（Heap）内存中的复杂数据结构，占据空间大、大小不固定。栈中只存储指向堆内存的**引用地址（指针）**。
* `Object`（普通对象）
* `Array`（数组）
* `Function`（函数）
* `Date`（日期）
* `RegExp`（正则表达式）
* `Map`、`Set`、`WeakMap`、`WeakSet` 等。

---

### 二、 核心区别详解

#### 1. **内存存储机制（栈 vs 堆）**
* **原始类型**：直接*存储在`栈内存`中。变量名直接对应具体的值*。
* **引用类型**：*实际数据存储在`堆内存`中，`栈内存`中存储的是一个`指针（内存地址）`，该指针指向堆中的实际数据*。

> **为什么这么设计？**
> 栈内存*访问速度快，但空间小，适合存储大小固定的简单数据*；
> 堆内存*空间大，但访问速度慢，适合存储大小动态变化的复杂对象*。

  ```javascript
  let a = 100;
  let b = a; // 复制值
  a = 200;
  console.log(b); // 100 (b 不受影响)
  ```

```javascript
  let a = {age:20};
  let b = a; 
  a.age = 21;
  console.log(b.age); // 21
```

#### 2. 赋值与变量拷贝
* **原始类型（值拷贝）**：*赋值时，会在新变量中**完整复制**一份实际的值*。*两个变量完全独立，互不影响*。
  ```javascript
  let a = 10;
  let b = a; // 复制值
  b = 20;
  console.log(a); // 10 (a 不受影响)
  ```

* **引用类型（地址拷贝 / 浅拷贝）**：*赋值时，复制的是**堆内存的地址（指针）***。*两个变量指向同一块堆内存，修改其中一个，另一个也会受到影响*。
  ```javascript
  let obj1 = { name: 'Alice' };
  let obj2 = obj1; // 复制地址
  obj2.name = 'Bob';
  console.log(obj1.name); // 'Bob' (obj1 被影响，因为它们指向同一个对象)
  ```

#### 3. 比较机制
* **原始类型（值比较）**：*比较的是**具体的值**是否相等*。
  ```javascript
  console.log(1 === 1);       // true
  console.log('a' === 'a');   // true
  ```

* **引用类型（地址比较）**：*比较的是**内存地址**是否相同*。即使两个对象的内容完全一样，只要地址不同，就不相等。
  ```javascript
  console.log({} === {});     // false (两个不同的堆内存地址)
  console.log([] === []);     // false
  
  let a = {};
  let b = a;
  console.log(a === b);       // true (指向同一个地址)
  ```

#### 4. 可变性（Mutability）
* **原始类型（不可变 Immutable）**：原始类型的值本身是**不可改变**的。任何看似修改原始值的操作，实际上都是**返回了一个全新的值**。
  ```javascript
  let str = 'hello';
  str[0] = 'H'; // 尝试修改
  console.log(str); // 'hello' (原字符串未改变，严格模式下会报错)
  
  str = str.toUpperCase(); // 实际上是创建了新字符串 'HELLO' 并重新赋值给 str
  ```

* **引用类型（可变 Mutable）**：引用类型的值是可以**动态修改**的（增删改属性），而内存地址保持不变。
  ```javascript
  let arr = [1, 2, 3];
  arr.push(4); // 原地修改
  console.log(arr); // [1, 2, 3, 4]
  ```

### 三、 进阶考点：属性访问与“自动装箱”

**规则**：原始类型**不能**添加属性和方法，引用类型**可以**。

```javascript
// 引用类型可以添加属性
let obj = {};
obj.name = 'test'; 
console.log(obj.name); // 'test'

// 原始类型不能添加属性
let str = 'hello';
str.name = 'test'; 
console.log(str.name); // undefined
```

**❓ 疑问：为什么原始类型可以调用方法？（如 `'hello'.toUpperCase()`）**
**💡 自动装箱（Auto-Boxing）机制**：
当你在原始类型上调用方法或访问属性时，JS 引擎会在后台**临时**创建一个对应的**包装对象（Wrapper Object）**（如 `String`、`Number`、`Boolean`），在这个临时对象上执行操作，操作完成后**立即销毁**该临时对象。
```javascript
let str = 'hello';
// JS 引擎内部实际执行了：
// let temp = new String('hello');
// let res = temp.toUpperCase();
// temp = null; // 销毁
```

### 四、 高频面试题：函数参数传递的真相

很多教程说“原始类型按值传递，引用类型按引用传递”，这种说法**不完全准确**。
在 JavaScript 中，**所有的函数参数都是按值传递（Call by Value）的**。对于引用类型，传递的是 **“内存地址的值”**（有时被称为按共享传递 Call by Sharing）。

**代码证明：**
```javascript
function changeStuff(num, obj) {
  num = 100;        // 修改原始类型的副本，不影响外部
  obj.name = 'Bob'; // 修改引用类型指向的堆内存，影响外部
  obj = { name: 'Charlie' }; // 将参数 obj 重新指向一个新的堆内存地址，不影响外部
}

let myNum = 10;
let myObj = { name: 'Alice' };

changeStuff(myNum, myObj);

console.log(myNum);       // 10 (未变)
console.log(myObj.name);  // 'Bob' (被修改了，因为共享了同一个堆地址)
```
**解析**：如果引用类型真的是“按引用传递”，那么 `obj = { name: 'Charlie' }` 这行代码会把外部的 `myObj` 也指向新对象，但并没有。这证明传入函数的只是**地址的副本**。

### 五、 类型检测方法的区别

#### 1. `typeof` 操作符
* 对**原始类型**非常准确（除了 `null`）。
* 对**引用类型**无能为力，除了 `function`，其他所有对象（数组、正则、日期等）都会返回 `'object'`。

```javascript
typeof 'str'      // 'string'
typeof 123        // 'number'
typeof true       // 'boolean'
typeof undefined  // 'undefined'
typeof Symbol()   // 'symbol'
typeof 10n        // 'bigint'

typeof null       // 'object'  (历史遗留 Bug)
typeof []         // 'object'
typeof {}         // 'object'
typeof function(){} // 'function'
```

#### 2. `instanceof` 操作符
* 用于检测**引用类型**的原型链，判断一个对象是否是某个构造函数的实例。
* **不能**用于检测原始类型（因为原始类型没有原型链）。

```javascript
[] instanceof Array    // true
{} instanceof Object   // true
new Date() instanceof Date // true

'str' instanceof String // false (原始类型不是对象)
```

#### 3. 终极检测方案：`Object.prototype.toString.call()`
这是最严谨的类型检测方法，能够精准区分所有内置类型。

```javascript
const getType = (val) => Object.prototype.toString.call(val).slice(8, -1);

getType('str')       // 'String'
getType(123)         // 'Number'
getType(null)        // 'Null'
getType(undefined)   // 'Undefined'
getType([])          // 'Array'
getType({})          // 'Object'
getType(/regex/)     // 'RegExp'
```

### 六、 总结对比表

| 维度 | 原始类型 (Primitive) | 引用类型 (Reference / Object) |
| :--- | :--- | :--- |
| **包含类型** | String, Number, Boolean, Null, Undefined, Symbol, BigInt | Object, Array, Function, Date, RegExp, Map, Set 等 |
| **内存存储** | 栈内存 (Stack) | 堆内存 (Heap)，栈中存指针 |
| **赋值机制** | 拷贝具体的值 (深拷贝) | 拷贝内存地址 (浅拷贝) |
| **比较机制** | 比较具体的值 | 比较内存地址 |
| **可变性** | 不可变 (Immutable) | 可变 (Mutable) |
| **添加属性** | 不允许 (静默失败或报错) | 允许 |
| **方法调用** | 依赖自动装箱 (临时包装对象) | 直接调用原型链上的方法 |
| **函数传参** | 传递值的副本 | 传递地址的副本 (按共享传递) |
| **检测方式** | `typeof` | `instanceof`, `Object.prototype.toString` |

**面试/实战建议**：
在实际开发中，处理引用类型时务必警惕**浅拷贝带来的副作用**（如 React/Vue 中的状态更新，必须生成新对象而不是修改原对象）。需要独立副本时，必须使用**深拷贝**（如 `structuredClone()`、`JSON.parse(JSON.stringify())` 或 lodash 的 `_.cloneDeep`）。
