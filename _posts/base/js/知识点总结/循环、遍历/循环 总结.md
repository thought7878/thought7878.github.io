>JavaScript 循环全面总结

JavaScript 提供了多种循环与迭代机制，涵盖传统命令式、现代可迭代协议、函数式高阶方法以及异步场景。合理选型能显著提升代码可读性、安全性与性能。

---

## 一、基础命令式循环

| 循环 | 语法 | 特点 | 适用场景 |
|------|------|------|----------|
| `for` | `for (init; condition; update) { ... }` | 最灵活，支持索引控制，性能最优 | 已知次数、需精确控制步长/索引、性能敏感场景 |
| `while` | `while (condition) { ... }` | 条件前置，可能一次都不执行 | 依赖外部状态、未知迭代次数（如读取流、轮询） |
| `do...while` | `do { ... } while (condition);` | 条件后置，**至少执行一次** | 用户交互、重试逻辑、初始化后判断的场景 |

```js
// for 示例
for (let i = 0; i < arr.length; i++) {
  if (arr[i] === target) break;
}

// while 示例
let node = root;
while (node) {
  process(node);
  node = node.next;
}
```

## 二、遍历专用循环

### 1. `for...in`（遍历对象键）
- 遍历对象**自身及原型链上可枚举的字符串键**
- **不保证顺序**（ES6 仅对整数样式的键保证升序）
- ❌ **严禁用于数组**：会遍历到自定义属性、原型属性，且顺序不可靠

```js
const obj = { a: 1, b: 2 };
Object.prototype.c = 3; // 原型污染测试
for (const key in obj) {
  if (Object.hasOwn(obj, key)) { // ES2022 推荐替代 hasOwnProperty
    console.log(key, obj[key]);
  }
}
```

### 2. `for...of`（遍历可迭代对象的值）✅ 现代首选
- 基于 `Symbol.iterator` 协议，遍历**值**而非键
- 支持：`Array`、`String`、`Map`、`Set`、`arguments`、`NodeList`、`TypedArray` 等
- 支持 `break` / `continue` / `return`
- 可通过 `.entries()` 同时获取索引与值

```js
const arr = ['x', 'y', 'z'];
for (const val of arr) console.log(val);

// 带索引
for (const [i, val] of arr.entries()) {
  console.log(i, val);
}
```

## 三、数组高阶迭代方法（函数式）

| 方法 | 返回值 | 是否短路 | 典型用途 |
|------|--------|----------|----------|
| `forEach` | `undefined` | ❌ | 纯副作用遍历（DOM操作、日志） |
| `map` | 新数组（等长） | ❌ | 数据映射/转换 |
| `filter` | 新数组（子集） | ❌ | 条件过滤 |
| `reduce` / `reduceRight` | 累积值 | ❌ | 求和、扁平化、分组、管道计算 |
| `some` / `every` | `boolean` | ✅ | 存在性/全量校验 |
| `find` / `findIndex` | 元素 / 索引 | ✅ | 查找首个匹配项 |

```js
const nums = [1, 2, 3, 4, 5];

const doubled = nums.map(n => n * 2);
const evens = nums.filter(n => n % 2 === 0);
const sum = nums.reduce((acc, n) => acc + n, 0);
const hasNeg = nums.some(n => n < 0); // false
const firstGt3 = nums.find(n => n > 3); // 4
```

> ⚠️ **注意**：高阶方法内部是函数调用，**无法使用 `break`/`continue`**。`forEach` 中 `return` 仅跳过当前迭代，不会终止循环。

## 四、循环控制语句

| 语句 | 作用 | 备注 |
|------|------|------|
| `break` | 立即终止整个循环 | 可配合标签跳出多层嵌套 |
| `continue` | 跳过当前迭代，进入下一次 | 仅影响当前循环层 |
| 标签 `label:` | 为循环命名，供 `break/continue` 定向跳转 | 可读性差，现代开发极少使用 |

```js
outer: for (let i = 0; i < 3; i++) {
  for (let j = 0; j < 3; j++) {
    if (i === 1 && j === 1) break outer; // 直接跳出外层
  }
}
```

## 五、核心对比与选型指南

| 方式 | 遍历目标 | 支持中断 | 性能 | 现代推荐度 | 典型场景 |
|------|----------|----------|------|------------|----------|
| `for` | 自定义 | ✅ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | 索引敏感、性能关键、复杂步长 |
| `while`/`do...while` | 自定义 | ✅ | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐ | 条件驱动、状态机、重试 |
| `for...in` | 对象可枚举键 | ✅ | ⭐⭐⭐ | ⭐⭐（谨慎） | 遍历普通对象键（需 `hasOwn` 过滤） |
| `for...of` | 可迭代对象的值 | ✅ | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | 数组/字符串/Map/Set 遍历首选 |
| `forEach` 等 | 数组元素 | ❌ | ⭐⭐⭐ | ⭐⭐⭐⭐ | 数据转换、链式处理、声明式逻辑 |

## 六、关键注意事项与最佳实践

### 1. 🚫 常见陷阱
- **`for...in` 遍历数组**：会包含非数字键、原型属性，且顺序不可靠。数组永远用 `for`/`for...of`/数组方法。
- **循环中修改集合长度**：`arr.splice()` 或 `delete obj[key]` 会导致索引错位或跳过元素。建议倒序遍历或收集待操作项后统一处理。
- **`forEach` 不等待异步**：
  ```js
  // ❌ 错误：所有请求并发发出，且外层不等待
  arr.forEach(async item => await fetch(item));
  
  // ✅ 串行等待
  for (const item of arr) await fetch(item);
  
  // ✅ 并行等待
  await Promise.all(arr.map(item => fetch(item)));
  ```

### 2. ⚡ 性能建议
- V8 对 `for`、`for...of`、数组方法均有深度优化。**业务开发中性能差异通常 < 5%**，优先保证可读性。
- 仅在**热点路径**（如 Canvas 渲染、大规模数据计算、游戏循环）中优先使用传统 `for` 或 `while`。
- 避免在循环体内创建闭包、重复查询 DOM 或执行正则编译。

### 3. 🔁 异步迭代（ES2018+）
```js
// for await...of：遍历 AsyncIterable（如流、分页API、Generator）
async function processStream(asyncIterable) {
  for await (const chunk of asyncIterable) {
    console.log(chunk);
  }
}
```

## 七、一句话选型口诀

> - **要索引/控步长/拼性能** → `for`  
> - **条件驱动/未知次数** → `while` / `do...while`  
> - **遍历数组/字符串/Map/Set** → `for...of`  
> - **遍历对象键** → `for...in` + `Object.hasOwn()`  
> - **数据转换/过滤/聚合** → `map` / `filter` / `reduce`  
> - **需要中途跳出** → 避开 `forEach`，选 `for...of` 或 `some`/`every`  
> - **异步串行** → `for...of` + `await`；**异步并行** → `Promise.all(arr.map())`

---

掌握循环的本质是理解 **“控制流”** 与 **“数据流”** 的边界。现代 JS 开发中，优先使用 `for...of` 与高阶数组方法提升表达力，在性能瓶颈或复杂控制流时回退到传统循环，配合异步迭代协议处理流式数据，即可覆盖 99% 的工程场景。
