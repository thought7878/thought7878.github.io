Using Emotion (Another CSS-in-JS Library)

视频讲述了*从 JSS 切换到 Emotion*（另一个 CSS-in-JS 库）的相关内容，包括 *Emotion 的介绍、安装、代码改写、使用优势*等，具体如下：

## 引入 Emotion 库背景

[00:00]
之前视频探讨了 CSS-in-JS 概念的实现方案 JSS，此次介绍*同类理念的另一个实现——Emotion 库*，其是用于*用 JavaScript 编写 CSS 样式*的库，*具备强大且可预测的样式组合功能*，还*拥有源码映射等优秀开发者体验*功能。

## Emotion 的两种使用方式

[00:37]
第一种是**框架无关**的使用方式，第二种是**与 React 配合使用**的方式，由于课程不使用 React，故先介绍框架无关的使用方式。

## Emotion 的安装准备

[00:54]
首先卸载与 JSS 相关的包，因为要改用 Emotion 重写样式，随后安装 Emotion，其中 `@emotion/css` 包是*框架无关的，且无需额外配置*，不需要 babel 插件或其他配置更改。

## Emotion 的功能支持

[01:17]
支持**自动添加厂商前缀、嵌套选择器、媒体查询**。

## Emotion 的代码使用方法

[01:28]
前往之前使用 JSS 的文件，删除与 JSS 相关且不再需要的代码，导入刚安装的 Emotion 新包，将原有样式对象转换为模板字符串，并修改代码中样式引用的方式，且改动微小，完成这些操作后可前往浏览器确认应用是否正常运行。

## CSS-in-JS 的项目使用建议

[03:22]
若在项目中使用 CSS-in-JS，建议全部采用这种方式编写样式，此时无需使用 css 加载器或 webpack 配置中的 style 加载器，因为所有 CSS 都将由 JavaScript 处理。
