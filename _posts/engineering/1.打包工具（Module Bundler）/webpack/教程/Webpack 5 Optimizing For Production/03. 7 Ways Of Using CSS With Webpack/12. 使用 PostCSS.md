Using PostCSS

视频讲述了*在应用程序样式处理中 PostCSS 的相关知识*，包括 *PostCSS 的定义、功能、使用场景*，以及*结合 after-prefixer 插件和 Webpack 的配置方法*等内容，具体如下：

## 介绍 PostCSS 及其定位

[00:00]
`PostCSS` 是*用 JavaScript 转换 CSS 的工具，也是开发 CSS 工具的框架*，可通过插件实现如 Sass 和 Less 的模板语言功能。
其**工作流程**是*先将 CSS 字符串转为对象树，插件分析修改该树后，再生成新的 CSS 字符串*。
还能提供浏览器特定前缀、CSS 模块、CSS 优化等功能，本课程将用其添加浏览器特定前缀。

## 说明浏览器厂商前缀的必要性

[01:38]
以 display flex 为例，它并非所有浏览器一开始都支持，浏览器会逐步添加支持，且常为前沿功能添加厂商前缀直至规范正式发布，像 Internet Explorer 10 和 11、Firefox 28 版本前仅支持带厂商前缀的 display flex，全屏 API 在 Safari、Internet Explorer 11 及部分移动浏览器中也需厂商前缀，大型应用中需前缀的 CSS 属性更多，手动添加繁琐，需自动化工具。

## 介绍 after-prefixer 工具及相关配置准备

[04:12]
after-prefixer 可分析 CSS 并基于浏览器版本自动添加所需前缀，可独立使用或作为构建流程一部分，本视频将其作为 PostCSS 插件，需先安装 prefixer 和 PostCSS 加载器，再创建 PostCSS 配置文件（可选 JavaScript 或 JSON 格式，课程用 JavaScript 格式，需导出含 plugins 属性的对象并指定 after-prefixer 插件），还需创建 browserslist 文件指定支持的浏览器（如每个浏览器的最后十个版本），让 after-prefixer 确定需加前缀的 CSS 属性。

## 讲解 Webpack 配置及效果验证

[06:30]
需在 Webpack 配置中嵌入 PostCSS 作为额外步骤，且仅在生产环境进行（开发阶段用最新 Chrome 浏览器，多数功能无需厂商前缀），注意 post css 加载器要放在 sass 加载器之前、css 加载器之后，移除手动添加的厂商前缀后，运行 webpack，在开发者工具中可看到厂商前缀已自动添加。
