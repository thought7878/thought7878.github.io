Using JSS

视频围绕 CSS-in-JS 概念及其中的 JSS 实现展开，介绍了 JSS 的相关特性、使用方法及实践操作，具体如下：

## 回顾 CSS-in-JS 概念并引出 JSS

[00:00]
先回顾此前讨论的 CSS-in-JS 概念，说明*其是通用概念而非特定库，存在多种实现方式*，接着引出*本期要介绍的实现方案 JSS*，并提及 JSS 是 CSS 创作工具，*可在浏览器、服务器端或 Node.js 构建时编译，不依赖特定框架，且因广泛使用受欢迎*，自定义 *Material UI 组件默认样式*时可能已使用过。

## 明确 JSS 使用所需依赖及预设优势

[01:41]
使用 JSS 需导入 `jss` 和 `jss-preset-default`(默认预设) 两个 npm 包，默认预设能快速配置默认设置和插件，无需手动设置，还可避免手动检查插件顺序的麻烦，若不想用自定义插件且不希望检查插件顺序，建议使用默认预设。

## JSS 样式定义与调整方法

[02:31]
需定义 jssStyles 变量，将此前 CSS 中的复选框样式复制过来，然后进行调整以符合有效 JavaScript 语法，同时提到因使用常规 JavaScript 定义样式，可利用 JavaScript 变量，比如将复选框 30 像素的宽高提取为变量复用。

## JSS 样式编译与类的使用

[03:32]
**要编译定义好的样式并插入文档对象模型**，编译函数返回包含类、关键帧等属性及触控、分离等方法的对象，重点关注 classes 属性，其包含 jss 生成的唯一类，可用于渲染 HTML，且 jss 生成的唯一类能避免类名冲突，但不能在代码其他部分依赖该类名，指定数据属性更安全，还需为复选框添加数据属性。
