## 明确 CSS-in-JS 的定义

[00:24]
`CSS-in-JS` **并非特定库，而是解决传统 CSS 问题的一种抽象理念（使用JS编写CSS）**。
*不同实现该理念的库可能采用不同方法*，但所有实现**都以 JavaScript 作为定义样式的语言（使用JS编写CSS，JS运行时生成样式，样式是JS逻辑的一部分）**。

## CSS-in-JS 的优势一：避免类名和样式冲突
[01:02]
**传统 CSS 因使用全局命名空间的 CSS 选择器易导致样式冲突**，虽**可通过嵌套选择器或 BEM 等规范避免，但会变得复杂难扩展**；
而 **CSS-in-JS 通过生成唯一类名将样式转换为 CSS，能完全避免此问题**，允许*在组件级别*思考样式，*无需担心其他地方的冲突样式*。

## CSS-in-JS 的优势二：业务逻辑与组件样式共存
[01:45]
**CSS-in-JS 可让组件的样式与组件本身放在一起**，要么*在同一文件中*，要么*在同文件夹的单独文件中*。

## CSS-in-JS 的优势三：样式可依赖 JavaScript 变量
[02:02]
**当需要根据变量（JavaScript 中存储的数据）定义样式时**，CSS-in-JS 具备优势，比如在 React 中，*样式可依赖组件状态或属性，这是传统 CSS 方法无法实现的*。

## CSS-in-JS 还有其他未详述优势并建议尝试

[02:27]
表示 CSS-in-JS *还有更多优点未提及*，建议未尝试过的用户进行尝试。

## ## CSS-in-JS 的缺点一：包体积问题
[02:38]
**使用 CSS-in-JS 时，样式会成为 JavaScript 代码的一部分**，最终生成包含 JavaScript 代码和 CSS 的单一包；
**相较于传统 CSS 生成的两个独立包（一个 JS 包、一个 CSS 包），体积更大，且无法并行加载 JavaScript 和 CSS**；
**不过可通过`代码拆分`和`懒加载`生成多个包以提升加载性能**。

## ## CSS-in-JS 的缺点二：无法单独缓存 JS 和 CSS
[03:44]
将 JavaScript 和 CSS 放在同一文件，导致无法单独缓存 JavaScript 和 CSS。

## ## CSS-in-JS 的缺点三：样式运行时生成影响交互时间（性能）
[03:56]
**CSS-in-JS 的样式是 JavaScript 逻辑的一部分，需在应用加载完成后执行，这会使应用变得可交互所需时间更长**。
