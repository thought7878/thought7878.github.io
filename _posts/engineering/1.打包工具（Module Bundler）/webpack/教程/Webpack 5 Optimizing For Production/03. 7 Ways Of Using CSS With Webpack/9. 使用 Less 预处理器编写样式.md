Using Less Preprocessor For Writing Your Styles

视频讲述了*如何在 Webpack 中配置和使用 LESS 预处理器*处理样式，包括*LESS 的核心功能介绍、样式文件改造、Webpack 配置调整*及验证等内容，具体如下：

## LESS 基础介绍

[00:00]
视频围绕应用程序样式处理技术展开，提及此前学习过 Webpack 配置 CSS 模块，此次聚焦 LESS 预处理器；

`LESS` 是 CSS 的向后兼容**语言扩展**，*在常规 CSS 基础上提供`变量`、`混合器`、`嵌套规则`等功能*。虽现在可直接用 CSS 变量，但 LESS 创建时无此功能，*变量和嵌套规则是其核心优势*，本视频主要使用这两个功能。

## LESS 变量定义操作

[01:12]
首先需将 CSS 文件扩展名改为 LESS，然后遍历代码提取颜色值作为变量，在文件顶部声明这些变量，还可根据需求提取更多变量，*尤其适用于样式中有大量重复值的情况*。

## LESS 嵌套规则应用

[03:10]
指出 DOM 元素相互嵌套（如待办事项容器包含新待办事项和待办事项列表，待办事项列表包含 li 元素，li 元素包含自定义复选框），而 CSS 本身不支持嵌套，会导致选择器冗长；LESS 原生支持嵌套功能，可调整代码使 CSS 结构模拟 DOM 结构，让代码结构更清晰（虽可能存在个人偏好差异），同时需更新 JavaScript 代码中对 CSS 文件的引用。
