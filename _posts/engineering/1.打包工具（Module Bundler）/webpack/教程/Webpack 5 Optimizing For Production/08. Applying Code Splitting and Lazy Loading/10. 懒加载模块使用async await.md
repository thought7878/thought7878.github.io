Using Async Await With Lazy Loaded Modules

视频讲述了如何*将动态导入（用于懒加载模块）的代码从传统 Promise 写法重构为更易读的 async/await 写法*，并验证功能正常，具体如下：

## 回顾动态导入与 Promise.all 基础

[00:00]
提及之前视频内容，说明动态导入函数返回 Promise，可通过 Promise.all 并行导入多个 JavaScript 库，为后续使用 async/await 优化做铺垫。

## async/await 优化动态导入的核心逻辑

[00:25]
因动态导入返回 Promise，可像处理普通 Promise 一样使用 async/await，通过在 Promise 前添加 await 操作符，摆脱 then 块，将变量赋值从 then 块移到普通变量，同时需将外层函数转为异步函数以使用 await 关键字。

## 对单个动态导入进行 async/await 重构

[01:02]
以单个动态导入为例，演示添加 await 操作符，迁移变量赋值和剩余代码，转换外层函数为异步函数，完成从传统 Promise 到 async/await 的重构，提升代码可读性。

## 对 Promise.all 并行导入进行 async/await 重构

[02:01]
针对多个模块并行导入的场景，在 Promise.all 前添加 await 操作符，将变量赋值移到 await 操作符前的常量表达式，迁移相关代码并转换函数为异步函数，完成重构。

## 验证重构后应用功能正常

[02:42]
运行应用程序，在浏览器网络标签页查看，通过删除待办事项操作，确认能正常触发加载 jQuery 和 Bootstrap 的 HTTP 请求，验证重构后的代码功能无误。
