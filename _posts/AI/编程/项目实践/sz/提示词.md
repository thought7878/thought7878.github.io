# Vibe Coding 提示词 - 京东商智数据分析平台

## 项目概述

构建一个企业级的 Web 数据分析管理后台，帮助京东商家分析和利用电商数据。

- **前端**: React 18+ + TypeScript + Ant Design 5+ + Zustand + React Query + ECharts + Vite
- **后端**: NestJS 10+ + TypeScript + Prisma + JWT + Redis + BullMQ
- **数据库**: PostgreSQL 15+
- **缓存**: Redis 7+

---

## Agent 架构与职责

### 主 Agent 职责
- 生成并调度子 Agent 按模块顺序实现
- 监控整体开发进度
- 协调模块间的依赖关系
- 确保各模块接口一致性
- 最终集成验证

### 子 Agent 职责
- 负责实现指定的单个模块（前端 + 后端）
- 编写该模块的单元测试和集成测试
- 确保模块可独立运行和测试
- 完成后向主 Agent 报告状态

### 执行顺序（按依赖关系）
1. **基础设施**: 项目初始化、数据库连接、Redis 连接、公共模块
2. **认证授权模块**: Auth + JWT + RBAC
3. **多租户模块**: Tenant CRUD + 配置
4. **系统管理模块**: 用户/角色/权限管理 + 个人中心
5. **数据源管理模块**: DataSource + 同步引擎
6. **商品分析模块**: Product Analysis
7. **交易分析模块**: Trade Analysis
8. **行业分析模块**: Industry Analysis
9. **流量分析模块**: Traffic Analysis
10. **报表模块**: Report + 导出 + 定时报告
11. **预警模块**: Alert + 通知
12. **前端公共组件**: 布局、图表、筛选器、Hooks
13. **前端页面集成**: 所有页面组件 + 路由 + 状态管理

## 各模块实现指南

### 1. 公共基础设施

#### 缓存服务 (Cache Service)
- `src/shared/cache/cache.service.ts`: Redis 封装，支持 get/set/del/hSet/hGet/zAdd/zRange/lPush/lRange/xAdd/xRead
- `src/shared/cache/cache-manager.service.ts`: getOrSet、按模块/租户清除缓存
- `Cacheable` / `CacheEvict` 装饰器: 方法级缓存

#### 任务队列服务 (Queue Service)
- `src/shared/queue/queue.service.ts`: BullMQ 封装，exportQueue / syncQueue / alertQueue
- Processors: `export.processor.ts`, `sync.processor.ts`, `alert.processor.ts`

#### 通知服务 (Notification Service)
- `src/shared/notification/notification.service.ts`: 多渠道发送 (email/sms/webhook)
- 子服务: `EmailService`, `SmsService`, `WebhookService`

#### 日志服务 (Logger Service)
- `src/shared/logger/logger.service.ts`: 扩展 NestJS ConsoleLogger，支持业务日志/操作日志
- `OperationLogInterceptor`: 自动记录操作日志到数据库

#### 文件服务 (File Service)
- `src/shared/file/file.service.ts`: 上传/下载/删除/预签名URL

#### 定时任务服务 (Scheduler Service)
- `src/shared/scheduler/scheduler.service.ts`: 定时报告调度
- Tasks: `DataSyncTask` (每分钟), `AlertCheckTask` (每5分钟), `ReportGenerateTask` (每小时)

### 2. 认证授权模块 (Auth Module)

**后端接口**:
| 路径 | 方法 | 描述 | 权限 |
|------|------|------|------|
| `/api/auth/login` | POST | 登录 | 公开 |
| `/api/auth/register` | POST | 注册 | 公开 |
| `/api/auth/logout` | POST | 登出 | 已认证 |
| `/api/auth/refresh-token` | POST | 刷新Token | 公开 |
| `/api/auth/reset-password` | POST | 重置密码 | 公开 |
| `/api/auth/change-password` | PUT | 修改密码 | 已认证 |

**核心组件**:
- `AuthController`, `AuthService`, `JwtService`, `PasswordService`
- Guards: `JwtAuthGuard`, `PermissionGuard`
- Strategies: `JwtStrategy`, `LocalStrategy`
- DTOs: `LoginDto`, `RegisterDto`, `RefreshTokenDto`, `ResetPasswordDto`, `ChangePasswordDto`

**前端**:
- `src/api/auth.ts`: 认证 API
- `src/store/authStore.ts`: Zustand 状态管理 (login/logout/refreshToken/clearAuth)
- `src/pages/Login/index.tsx`: 登录页面
- Axios 拦截器: 请求附加 Token，401 自动刷新

**密码规则**: 至少8位，包含大小写字母和数字，正则: `/^(?=.*[a-z])(?=.*[A-Z])(?=.*\d)[a-zA-Z\d]{8,}$/`

## 安全要求

- HTTPS 全站加密
- JWT Token 认证，AccessToken 2h 过期，RefreshToken 7d 过期
- bcrypt 密码加密
- SQL 注入防护 (Prisma 参数化查询)
- XSS/CSRF 防护
- 敏感数据脱敏展示
- 操作日志审计
- Token 黑名单机制 (Redis)

---

## 性能要求

- 页面首屏加载 < 2秒
- API 响应 < 500ms (95% 请求)
- 大数据查询 < 3秒 (百万级)
- 支持 1000+ 并发
- 系统可用性 99.9%

## 注意事项

1. 每个子 Agent 实现模块时，必须确保该模块可独立测试
2. 所有 DTO 使用 `class-validator` 进行参数校验
3. 所有业务 Service 必须注入 tenantId，确保数据隔离
4. 查询类接口优先考虑缓存策略
5. 大数导出走异步队列
6. 前端所有页面组件使用 TypeScript，严格模式
7. 遵循 RESTful API 规范
8. 每个模块完成后先运行测试，再报告完成
