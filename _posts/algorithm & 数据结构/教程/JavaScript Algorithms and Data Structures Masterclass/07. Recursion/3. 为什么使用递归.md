## 递归的定义
[00:00]
`递归`是**一个调用自身的过程**，在JavaScript等代码编写场景中，递归指**调用自身的函数**。*与函数内部调用其他函数不同，递归函数会调用自身*，且**该过程需有终止条件**，*否则会出现类似无限循环的问题*（后续会详细讲解其与无限循环的区别）。并以checkForOdds函数调用自身处理更小列表为例进行说明。

什么是递归？
**一个（在我们的例子中是函数）调用自身的过程**

## 学习递归的原因之一：递归无处不在且常被使用
[01:38]
*在课程后续很多解决方案中会大量使用递归*；即使在编写JavaScript时，也可能不知不觉使用过递归函数，如JSON解析或JSON字符串化，以Mozilla的Rhino引擎（用Java编写）中JSON解析的实现为例，其中readValue方法和readObject方法相互调用形成递归链条，还展示了用JavaScript编写的简化版walk方法（用于JSON解析相关操作），其内部会调用自身处理嵌套的JSON字符串，直到遇到结尾。

它无处不在！
• JSON.parse / JSON.stringify
• document.getElementById 和 DOM 遍历算法
• 对象遍历
• 在更复杂的算法中非常常见
• 有时它是迭代的一种更简洁的替代方案

## 学习递归的原因之二：递归在特定场景有优势
[04:21]
DOM是树状结构，存在大量嵌套元素，遍历所有元素时，*递归是一种流行的实现方法*；
当创建自定义数据结构（如树或图）并需要遍历、搜索其中内容时，*递归常是解决方案的一部分*。
虽然不强制使用递归，但在这些场景中，*递归实现往往更简单、清晰、简洁易懂，有时是迭代的更优替代方案*，后续会通过用递归和非递归两种方式编写函数来进一步清晰展示这一点。
