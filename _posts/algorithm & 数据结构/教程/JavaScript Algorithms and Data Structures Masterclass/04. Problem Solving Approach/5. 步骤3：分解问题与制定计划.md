## 拆解分析的重要性、核心价值
[00:00]
*在理解问题、探讨具体示例后，写代码前进行拆解分析十分关键*。

无需完整伪代码或有效代码，**明确写出你需要采取的步骤，只需写出解决方案的基本组件、步骤（不需逐行描述），用简短注释作为步骤指引即可**。

***好处**，尤其在面试场景中：*
- **能帮助清晰表达思路，避免盲目写代码**，
- 还可获取面试官提示，
- 同时**迫使自己提前思考，发现并解决遗留问题或不理解的部分**，
- **理清步骤、保持专注**，
- 帮你突出可能仍不理解的问题点。

![[_posts/algorithm & 数据结构/教程/JavaScript Algorithms and Data Structures Masterclass/04. Problem Solving Approach/media/b7d35d083f2b9a78436cc51245f54946_MD5.jpeg]]
明确写出你需要采取的步骤。
这会迫使你在编写代码之前先思考将要写的代码，并且能帮助你在深入编写代码、不得不操心细节（例如语言语法）之前，发现任何潜在的概念性问题或误解。

## 拆解的第一步：宏观步骤
[01:47]
*以字符计数问题为例*，针对“编写接受字符串并返回字符串中每个字符计数”的问题：
- **先确定函数骨架**，*函数名*为charCount，*参数*为字符串str，明确*返回键*为小写字母数字字符的对象（仅关注字母数字字符且转换为小写），
- 接着**确定基本步骤（宏观步骤）**：创建要返回的对象、遍历字符串填充对象细节、最后返回对象。

![[_posts/algorithm & 数据结构/教程/JavaScript Algorithms and Data Structures Masterclass/04. Problem Solving Approach/media/e86cf5c680cdcc3bff6e9e836b77db52_MD5.jpeg]]

## 拆解的第二步：细化核心逻辑
[03:04]
*细化字符计数问题的遍历核心逻辑*。
遍历字符串时，需检查每个字符。可选择一开始将整个字符串转为小写或遍历中逐个转小写，
**核心逻辑为**：
- 若字符是字母、数字且已在对象中，计数加1；
- 若字符是字母数字但不在对象中，添加到对象并设值为1；
- 若字符非字母、数字（如空格、句号等），则不做处理，逐步构建对象后返回。

## 总结
[06:32]
重申拆解分析对复杂问题、面试的意义。
**对于复杂问题，拆解分析作用显著，写下步骤能明确自身要做的事，即便不确定实现方式，也能向面试官展示已形成方案**。
***面试中问题常设计得较难或耗时久*，未必要求完成，*重点考察过程*，许多人因清晰的拆解思路（即便未完成问题）获得工作**。
