Singly Linked List: Set Solution


视频讲述了单向链表（Singly Linked List）中*set方法的实现思路、代码逻辑*及测试过程，具体如下：

## 明确set方法参数与核心依赖
[00:00]
set方法需接收两个参数，分别是目标索引和要设置的新值；实现过程核心依赖已定义的get方法，通过调用get方法传入目标索引来获取对应节点。

## set方法代码逻辑
[00:14]
先调用get方法并将结果赋值给foundNode变量，get方法会返回null或实际节点；若foundNode有效（即找到对应节点），则将foundNode.val更新为传入的新值并返回true；若未找到节点（foundNode为null），则直接返回false，无需使用else分支，因仅存在找到和未找到两种情况。

## set方法功能测试（有效索引场景）
[01:15]
先调用list.get(2)，确认索引2处元素为“感叹号”；再调用list.set(2, "三个感叹号")设置新值；最后再次调用list.get(2)，验证索引2处元素已更新为“三个感叹号”，证明有效索引下set方法可正常修改节点值。
## set方法功能测试（无效索引场景）
[01:38]
测试一，尝试设置索引6（该索引不存在），set方法返回false，查看列表尾部，最后一个元素仍为“笑脸”，未发生改变；测试二，尝试设置索引##1（负索引无效），因get方法拒绝该索引并返回null，set方法同样返回false，证明无效索引下set方法不会修改列表且返回正确标识。
