13. Naive String Search Implementation


视频讲述了*朴素字符串搜索算法的实现过程*，包括*函数定义、循环逻辑设计、匹配判断、计数统计及不同场景测试*等内容，具体如下：

## 明确算法实现目标与函数定义
[00:00]
视频目标是实现朴素字符串搜索算法，首先定义`naive search`函数，该函数需接受长字符串（或文本）和短字符串（或模式）两个参数，并以“在字符串'lorrie lo lol'中搜索'lol'”为例进行后续代码实现与演示。


## 设计嵌套循环遍历逻辑
[00:55]
外层循环用变量`i`遍历长字符串，从`i=0`开始，当`i`小于长字符串长度时`i`自增；内层循环用变量`j`遍历短字符串，从`j=0`开始，当`j`小于短字符串长度时`j`自增。通过嵌套循环实现对两个字符串字符的逐一比对，同时提及可通过打印`i`和`j`对应的字符来观察遍历过程。


## 确定字符匹配判断方式
[04:17]
为准确比对长字符串中对应位置的字符，采用“将`i`和`j`相加得到长字符串的索引”的方式，即比较短字符串`j`位置的字符与长字符串`i+j`位置的字符。若两者不匹配，则跳出内层循环，重置`j`为0，外层循环`i`继续自增；若匹配则继续内层循环直至遍历完短字符串。


## 添加匹配计数与结果返回机制
[07:49]
创建`count`变量并初始化为0，当内层循环完整遍历完短字符串（即`j`等于短字符串长度）时，说明找到一次完整匹配，此时`count`自增1。最终函数返回`count`值，以此统计长字符串中短字符串的匹配次数。


## 测试不同场景验证算法有效性
[10:28]
一是测试“在对应字符串中搜索'lo'”的场景，结果得到2次匹配，验证算法对多匹配情况的处理能力；二是测试“搜索不存在的字符串'pop'”的场景，结果返回0，说明算法能正确处理无匹配的边界情况，同时提及无需额外处理错误情况，因`count`变量初始为0可直接反映无匹配结果。


## **优化**代码格式与**回顾**算法流程
[09:54]
为使代码更整洁，可将部分条件判断逻辑写成单行形式，如简化匹配判断与循环跳出的关联代码，且优化后代码仍能正常运行。同时，再次梳理算法完整流程，帮助观众精确理解从遍历、匹配判断到计数的整个工作原理，强调只要嵌套循环逻辑正确，实现细节可稍有不同。
