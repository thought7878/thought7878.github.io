Data Structures: The Class 


视频讲述了在ES2015中*使用class关键字声明类的基础语法、构造函数作用、类实例化方法、相关注意事项*，具体如下：

## 类声明基础、命名规范
[00:00]
介绍在ES2015中声明类的方式，以Student类为例，说明类名通常**以大写字母开头并采用驼峰命名法**，***类可作为组织代码的蓝图***，*让代码更易操作、理解和结构清晰，虽普通键值对对象也能实现类似功能，但类的模式更具优势*。

## 类的构造函数定义与作用
[00:45]
类中**必须有一个名为constructor的特殊方法，它用于实例化新的类实例**。在Student类构造函数中，可接收名字、姓氏、年级等参数，并*通过this关键字将这些参数分配给新创建的学生实例，为实例绑定对应的属性*。


## 类的实例化方法与示例
[01:25]
说明仅定义类的蓝图不会创建实例，必须使用new关键字实例化类。以Student类为例，通过new Student（并传入名字、姓氏、年级等参数）可创建具体学生实例，还可将实例保存到变量中，之后能访问或更新实例的属性，如更新学生的grade属性。

## this关键字在构造函数中的含义
[04:52]
在类的构造函数内部，**this关键字指向当前类的实例**。以创建Kitty（名字为Kitty、姓氏为Cat、年级为一年级）这个Student实例为例，构造函数中的this分别指向该实例，将传入的参数对应赋值给实例的属性。


## 构造函数的自动执行机制
[06:11]
**无需显式调用类的构造函数**（如不写Student.构造函数），**使用new关键字实例化类时，构造函数会自动执行，从而完成数据绑定，在实例化新类实例的过程中发挥作用**。


## 类在数据结构中的应用铺垫
[04:10]
指出后续会涉及树、单向链表等数据结构的类定义，强调定义这些数据结构的类只是创建模式，之后仍需用new关键字实例化（如new tree、new单向链表），再对实例对象的方法进行操作。
