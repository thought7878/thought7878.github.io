
Which Data Structure Is The Best?

## 课程阶段转折与数据结构学习预告
[00:00]
课程进入重大转折点，从之前讨论的*算法、问题解决方法、大O表示法、递归排序、搜索*等内容，转向数据结构的学习；后续将围绕多种新数据结构及相关内容展开，且会*先讲解ES2015 JavaScript中创建类的语法*，该语法是定义后续数据结构的方式，若已掌握可跳过，但建议不熟悉者学习。


## 课程将涵盖的主要数据结构类型
[01:20]
包括*二叉搜索树、队列、单向链表、无向无权图、二叉堆、优先队列、有向图、哈希表、双向链表、栈*等，后续会为每种数据结构单独创建类，从头定义，类似定义自己的数组或对象，但结构更复杂。

## 数据结构的本质与共性
[02:03]
`数据结构`是**值的集合，包含值之间的关系以及可应用于数据的操作**。
- ***存储数据***
- ***数据间有明确关系***
- **操作数据的方法**

**以数组为例**，数组*存储多个值*，*有值之间的顺序关系*，还自带添加、移除、排序、反转等*内置方法和交互功能*。

**后续要定义的单向链表、图、树、堆等结构也符合此模式**，会*存储数据*，并以特定方式组织，*数据间有明确关系*，还会编写*多种交互方法*。


## 数据结构多样的原因及特性
[03:46]
不同数据结构在不同方面表现优异，部分专业、部分通用。
**像数组、JavaScript对象这类常用结构，编程语言会自带实现**；而*红黑树、无向图等则需自行实现*，它们虽需自行实现但在特定场景下很有用。
*学习时需了解其适用场景*，且*没有绝对最好的 data structure，各有特定用途和专长*。

## 学习数据结构的重要原因
[05:35]
一是随着开发者经验增长，会遇到用数组无法解决的问题，需要更高效、能表达不同关系或处理非线性复杂数据的数据结构；二是开发者可能在不知情时已使用数据结构，如操作JavaScript的DOM实际是在操作树结构；三是数据结构是许多面试的核心内容，同时在现实世界中也极具实用性。

## 不同数据结构的适用场景示例
[07:56]
处理地图或位置数据、开发导航应用（如路线规划、找最近加油站）适合用图结构；需要有序列表且要快速在两端插入和删除大量数据时，链表比数组更实用；进行嵌套HTML的网络爬虫或处理相关数据时，数据常以树形结构返回；编写调度器、需找到当前最高优先级任务处理时，可能会用到二叉堆（及优先队列）。


## 数据结构部分的学习建议
[09:53]
数据结构内容密集难懂，不建议短时间内快速学完，课程中该部分需两周密集训练（每天学习8 - 10小时），建议学习单个结构后适当休息，且要留意各章节先修要求，数据结构间相互关联，虽无需严格按线性顺序学习，但从头开始按合理顺序学习（从单向链表到图）更佳。
