6. Anagram Challenge Solution


视频讲述了*使用频率计数法解决变位词问题*的具体实现步骤、时间复杂度优势、该方法的应用场景，具体如下：

- **明确变位词问题解决方法及目标时间复杂度**：[00:00]指出变位词问题有多种解决方法，本次选择用频率计数法，借助对象通过循环构建频率计数器，再用非嵌套的第二层循环处理，以保持O(n)的时间复杂度。
![[_posts/algorithm & 数据结构/教程/JavaScript Algorithms and Data Structures Masterclass/05. Problem Solving Patterns/media/56dbfecccdd25e48a3a01523aa2e1542_MD5.jpeg]]

- **定义函数并进行初始长度检查**：[00:19]定义名为validAnagram的函数，接收两个字符串参数；首先检查两个字符串长度，若长度不同，直接返回false，因为不同长度的字符串无法形成变位词。
- **构建第一个字符串的频率计数器（lookup对象）**：[00:44]创建lookup对象作为频率计数器，遍历第一个字符串，对于每个字符，若该字符不在lookup对象中，则将其在对象中的值设为1，若已存在，则将对应值加1，以此完成第一个字符串字符频率的统计。
- **利用频率计数器验证第二个字符串**：[02:45]遍历第二个字符串，对于每个字符，先检查该字符是否在lookup对象中，若不存在则返回false；若存在则将该字符在lookup对象中的值减1。若减1后遇到值为0的情况，继续处理后续字符，若处理过程中出现字符不存在于对象或值异常的情况，均返回false，若顺利遍历完第二个字符串，则说明两个字符串是变位词。
- **说明该频率计数法的优势及应用场景**：[05:13]该方法仅需两个简单的非嵌套循环，易于实现，*避免了嵌套循环可能带来的更高时间复杂度*；**适用于多个数据比较场景**，如*判断字母异位词、两个数组是否由相同元素组成、两个数字是否由相同数字组成（仅顺序不同）* 等情况。
