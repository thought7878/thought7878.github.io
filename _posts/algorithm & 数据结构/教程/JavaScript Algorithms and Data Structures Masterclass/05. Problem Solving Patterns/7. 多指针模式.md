7. Multiple Pointers Pattern


视频讲述了*多指针模式（Multiple Pointers Pattern）的定义、应用场景、具体案例、优化效果*，通过*对比朴素解法与多指针解法*，突出该模式在提升算法效率上的优势，具体如下：

## 定义与核心思想
[00:00]
非官方名称，`核心`是**在数组、字符串、双向链表等*线性结构*中，创建对应*索引*或*位置*的指针**（变量，如i、j），**基于特定条件让指针向开头、结尾或中间移动，通常*用于寻找满足条件的值（常为一对值），一次循环的两个指针替代两个嵌套循环***，模式定义较松散但**应用频率高**。

多指针
创建与索引或位置相对应的指针或值，并根据特定条件向开头、结尾或中间移动。
对于以最小的空间复杂度解决问题也非常高效。

## 问题引入
[01:45]
需编写sumZero函数，该函数接收已排序（从小到大）的整数数组，*任务*是找到首个和为零的数对并以数组形式返回，若不存在则返回未定义，且强调*数组有序是高效解决该问题的前提，无序则无法高效处理*。

### 朴素解法及缺陷
[02:59]
`朴素解法`采用**两层嵌套for循环**，*外层循环定位一个数，内层循环遍历剩余数组寻找与其和为零的数*，**时间复杂度为O(n²)**。若数组包含一万项，嵌套循环会导致迭代次数极多，操作效率低。

![[_posts/algorithm & 数据结构/教程/JavaScript Algorithms and Data Structures Masterclass/05. Problem Solving Patterns/media/79eb3e51972a9bb5c28c67d3630d09b1_MD5.jpeg]]

### 多指针解法原理
[04:35]
- 利用数组有序特性，设置两个指针，左指针从数组索引0（最小值端）开始，右指针从数组长度减一的位置（最大值端）开始。
- 计算两指针指向元素的和，若和大于零，将右指针左移（减小数值）；
- 若和小于零，将左指针右移（增大数值）；
- 若和等于零，返回两元素组成的数组。

![[_posts/algorithm & 数据结构/教程/JavaScript Algorithms and Data Structures Masterclass/05. Problem Solving Patterns/media/fc35020ba96af8404292581f982cdab6_MD5.jpeg]]

#### 时间复杂度与循环终止条件
[06:32]
多指针解法**时间复杂度为O(n)**，优于朴素解法的O(n²)。`循环终止条件`为左指针小于右指针，而非小于等于，避免左、右指针指向同一元素时，出现自身与自身相加为零的假阳性结果，若循环结束未找到和为零的数对，返回未定义。
