Creating Queues Using Arrays


视频讲述了*如何用数组构建队列*，介绍了两种实现方法及其性能问题，并提及后续将讲解更优的自定义队列类实现，具体如下：

## 引入用数组构建队列的主题
[00:00]
明确本次内容是快速讲解如何用数组构建队列，指出至少有两种实现方法，即向数组开头添加元素或向数组末尾添加元素。


## 第一种数组实现队列的方法及操作
[00:26]
选择从数组末尾添加元素，使用push方法添加第一个、第二个、第三个元素；
为遵循队列先进先出规则，需从开头移除元素，此时不能用pop方法（栈适用），而要用shift方法，通过该方法可依次得到添加的元素，此方法能实现队列功能。


## 第一种方法的性能问题
[01:09]
当数组中有上万个项目时，从开头用shift移除元素会导致所有元素重新索引，整体移动所有元素，存在性能隐患；而向数组末尾用push添加元素无需重新索引，只需在末尾添加新元素。


## 第二种数组实现队列的方法及操作
[01:31]
从数组开头添加元素，使用unshift方法添加第一个、第二个、第三个元素；按照队列先进先出规则，需从末尾移除元素，使用pop方法，可依次得到添加的元素，该方法同样能实现队列先进先出的功能。


## 第二种方法的性能问题
[02:11]
当扩展到一万个项目时，向开头用unshift添加元素，每次插入都会导致整个队列重新索引，移动所有元素并为每个元素分配新索引，性能不佳；但从末尾用pop移除元素效果良好。


## 对比栈与队列的数组实现差异及后续计划
[02:34]
用push和pop实现栈无需重新索引整个数组结构，而**数组实现队列无法避免重新索引的问题**；
*若关注性能，创建自定义队列类很有必要*，后续视频将展示用链表队列定义自定义队列类，*其性能优于当前数组实现方案*，同时说明当前用数组实现的队列虽性能有不足，但确实实现了先进先出功能，在后续复杂算法中可用于保持代码简洁。
