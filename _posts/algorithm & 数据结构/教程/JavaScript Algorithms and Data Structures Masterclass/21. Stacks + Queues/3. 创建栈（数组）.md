Creating a Stack with an Array


视频讲述了*如何使用数组创建栈结构*，包括*栈的核心概念、两种基于数组的实现方法、效率对比*及后续学习方向，具体如下：

## 栈的核心概念回顾
[00:00]
`栈`是一种**后进先出（LIFO）的数据结构**，仅**定义数据存储和移除的规则**（最先添加的数据最后移除，最后添加的数据最先移除），未限定具体实现方式，可通过多种方式创建。


## 基于数组的第一种栈实现方法（push+pop）
[00:58]
先新建空数组作为栈，*使用数组的push方法向栈中添加元素*（如依次添加google.com、Instagram、YouTube模拟网页浏览历史），再*使用pop方法从数组末尾移除元素*，确保最后添加的元素最先被移除（如先移除YouTube，再移除Instagram，最后移除google.com），只要仅通过push添加、pop移除，即可实现栈结构。


## 基于数组的第二种栈实现方法（unshift+shift）
[03:07]
同样新建空数组，*使用unshift方法在数组开头添加元素*（如模拟Photoshop操作记录：新建文件、调整文件大小、移除皱纹等），*再使用shift方法从数组开头移除元素*，也能满足后进先出原则，实现栈功能。


## 两种数组实现方法的效率对比
[04:46]
从效率角度看，push+pop方法更优。因为unshift在数组开头添加元素、shift从开头移除元素时，需重新调整数组中所有元素的索引（元素整体下移或上移），数据量较大时效率低；而push在数组末尾添加、pop从末尾移除时，仅需操作新索引或末尾索引，无需调整其他元素索引，效率更高。


## 数组实现栈的局限性与后续学习方向
[05:53]
若栈中存储大量数据且需高效实现后进先出功能，数组并非最佳选择，后续将学习使用链表结构实现栈，链表无需基于索引访问数据，能更好地按插入时间顺序处理元素，适配栈的需求。


## 两种数组实现方法的核心原则
[06:38]
无论选择哪种数组实现方式，核心原则是“添加和移除元素需在同一方向”，若在数组中间或非对应方向（如用push添加却用shift移除）操作元素，则无法实现栈结构。
