When to Use BFS and DFS

视频讲述了*树的遍历方法中 BFS（广度优先搜索）和 DFS（深度优先搜索）的适用场景、时间与空间复杂度差异*，以及 *DFS 不同变体*的应用示例，还回顾了树相关的基础概念，具体如下：

## BFS 与 DFS 选择需结合场景

[00:00]
视频开篇指出，BFS 和 DFS *没有绝对的优劣*，选择哪种遍历方法*取决于具体场景*，每种方法都有优缺点，需根据实际情况判断。

## BFS 与 DFS 时间复杂度相同

[01:18]
两种遍历方法的**时间复杂度一致**，*因为每个节点都会被访问且仅访问一次*，无论采用 BFS 还是 DFS，*处理所有节点的工作量相同*。

## 树结构影响 BFS 与 DFS 空间复杂度

[01:30]
**当树较宽时**，`BFS` 需*用队列存储*大量待访问节点，*空间消耗大*；而 `DFS` 仅跟踪当前分支节点，*空间占用更少*。
**当树较深且长时**，`DFS`（尤其*递归实现*）需*保留大量层级节点在内存（调用栈），空间消耗大*；而 `BFS` 队列最多仅存一个元素，内存占用几乎可忽略。

## DFS 不同变体的应用示例

[03:38]
`中序遍历`在二叉搜索树中应用时，**能得到有序的节点集合（从最小到最大排列）**，可用于**将节点存入列表或数据库等按序处理的场景**；
`前序遍历`（先访问节点，再处理左、右子树）在克隆/复制树、将树展平存储（如存入文件或数据库）及后续从序列化结构重建树时有用，因其能明确树的根节点及节点顺序。

## 树相关基础概念回顾

[06:21]
树是非线性数据结构，有单一根节点和多个子节点；二叉树是特殊树，每个节点最多有两个子节点；二叉搜索树是更具体的二叉树，需满足“父节点左侧节点值更小、右侧节点值更大”的条件，且只能用可比较的数据构建；树的遍历可采用 BFS 和 DFS，DFS 包含中序、后序、前序三种变体。
