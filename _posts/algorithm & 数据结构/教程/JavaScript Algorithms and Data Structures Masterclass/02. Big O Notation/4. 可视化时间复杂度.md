4. Visualizing Time Complexities


视频围绕*展示自制的时间复杂度可视化工具*，对比两种“累加”算法的执行时间趋势展开，帮助*理解算法性能差异*，具体如下：

- **介绍工具及核心关注对象**：[00:00]说明视频简短，将*展示自制的用于理解、绘制函数执行时间图表以呈现大致趋势*的小部件，该工具包含七个内容，重点关注前两个“累加”相关内容，且需为它们起独特名字，分别是用循环的“累加到第一个”和用快捷方式的“累加到第二个”。
![[_posts/algorithm & 数据结构/教程/JavaScript Algorithms and Data Structures Masterclass/02. Big O Notation/media/873d097d89d38495e3ec76feff0bed9d_MD5.jpeg]]

- **工具操作方式与原理**：[00:31]说明可选择具体“累加”类型，指定n的值后点击绘图，工具会通过后台计时器计时，执行操作后停止计时并绘图，再次操作会连接点构建图表，以此观察n增大时的时间变化情况。
- **“累加到第二个”算法的时间表现**：[01:09]指出*该算法无论n值（如10、10亿）如何，始终只有三个需要时间的操作，时间相当稳定*；以n=100、1000、10000为例绘图，虽因尺度小图表有波动，但整体随n增大时间基本无变化，仅存在细微差异。
- **“累加到第一个”算法的时间表现**：[02:11]表明*该算法因循环存在，操作次数取决于n值*；分别以n=100、1000、10000绘图，*可见其执行时间明显慢于“累加到第二个”，且随n增大，时间呈比例增长，形成明显可追踪的直线趋势，n越大耗时越长*，后续更大n值甚至需要几秒。
- **两种算法性能对比及工具后续用途**：[03:50]对比可知，*“累加到第二个”算法性能远优于“累加到第一个”，前者时间稳定，后者时间随n增长*；同时说明后续会继续使用该工具讨论更多内容，此次展示的是前两条通用趋势，能清晰看到实现相同功能的两种算法性能大局观截然不同。
