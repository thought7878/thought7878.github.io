3. Counting Operations

视频围绕“*Counting Operations（统计操作次数）*”展开，解释了*为何用统计操作次数替代时间来衡量代码效率*，以及*不同代码方案操作次数的差异和核心关注点*，具体如下：

## 用统计操作次数替代时间衡量代码的原因
[00:00]
**用时间比较代码存在问题**，时间会因计算机规格、设备状态等因素变化。
而***计算机执行的简单操作次数*保持不变，可作为更可靠的衡量标准**。

![[_posts/algorithm & 数据结构/教程/JavaScript Algorithms and Data Structures Masterclass/02. Big O Notation/media/5abab656cf65aebd2650cb06a30470de_MD5.jpeg]]

## 明确“简单操作”的范畴及示例
[00:50]
简单操作包括乘法、加法、除法、赋值、比较等，像数字1或2这类无需计算机过多处理的不算；以简洁单行代码为例，其包含乘法、加法、除法三次简单操作，且操作次数不随n的大小变化。

![[_posts/algorithm & 数据结构/教程/JavaScript Algorithms and Data Structures Masterclass/02. Big O Notation/media/653c175c28ff97114b897b4274b80ea6_MD5.jpeg]]

## 分析循环代码方案的操作次数
[01:42]
循环中的加法、赋值（如等号操作、i自增的赋值）、比较（i小于等于n）等操作次数会随n线性增长；循环外的部分赋值操作（如初始赋值i等于一）仅执行一次；以n为不同数值时的情况举例，说明该方案操作次数不是固定常数。

![[_posts/algorithm & 数据结构/教程/JavaScript Algorithms and Data Structures Masterclass/02. Big O Notation/media/8015556e5d0a9940778ca394a4001785_MD5.jpeg]]

## 衡量代码效率的核心关注点
[02:24]
无需过于纠结逐个计算每步操作的细节，关键在于整体量级和整体趋势；以“5n+2”“2n”“3n”“50n”等表达式为例，说明**当n增大时，具体系数和常数项无关紧要，*重要的是操作次数与n的增长比例关系*，这也是大O符号关注的重点**。
