```tsx
function Num(props) {
  return <div id='num'>{props.value}</div>;
}

function App() {
  const [count, setCount] = useState(0);

  const debugHandleClick = () => {
    setCount((c) => c + 1);
  };

  return (
    <div id='app'>
      <header className='app-header'>
        <span>header</span>
        <p>title</p>
      </header>
      <Num value={count}></Num>
      <button id='add' onClick={debugHandleClick}>
        Add
      </button>
    </div>
  );
}

export default App;

```

**算法/逻辑/伪代码：**
- 从rootFiber开始，wip=rootFiber
- 判断是否存在子节点
	- 存在，执行beginWork
	- 不存在，执行completeWork
- 判断是否存在兄弟节点
	- 存在，执行beginWork；再执行“是否存在子节点”的逻辑
	- 不存在，向上执行父节点的completeWork

创建/更新此轮渲染的 rootFiber（在prepareFreshStack）
```tsx
// prepareFreshStack
const rootWorkInProgress = createWorkInProgress(root.current, null);
workInProgress = rootWorkInProgress;
```

##深度优先遍历（DFS）

## beginWork：递阶段（首次渲染/mount）
首次渲染时，只有rootFiber存在current（createRoot创建的），其他fiber不存在。

**关键逻辑：**
- *为wip执行beginWork的最终目的*是，为workInProgress.child构建fiber，根据ReactElement（reconcileChildren）
- 根据wip.tag，走不同的创建fiber的流程（不同的updateXXX函数）
- reconcileChildren，根据是否存在current决定是否标记flags，mount时不存在current不标记
- reconcileChildFibers，根据wip fiber的子ReactElement的不同类型，执行不同的创建wip.child fiber的函数
