# 1
## 问题分析

根据 GitHub issue #36169 的描述，问题是：
- **影响版本**: React 19.2.4
- **核心问题**: 当在 `<head>` 中使用 `<script defer>` 时，会导致 hydration 错误
- **复现步骤**: 
  1. 创建包含 `<script src="foo" defer />` 的 React 应用
  2. 使用 `react-dom/static#prerenderToNodeStream` 保存为 HTML
  3. 使用 `react-dom/client#hydrateRoot` 在浏览器中进行 hydration
  4. 结果：hydration 警告被打印

## 根本原因

通过分析代码，我找到了问题所在：

### 1. SSR 端 (`ReactFizzConfigDOM.js`)

在 `pushScript` 函数中，只有 `async` 脚本才会被作为 Resource 处理（可提升）：

```javascript
// pushScript 函数 - 第 3771-3786 行
const asyncProp = props.async;
if (
  typeof props.src !== 'string' ||
  !props.src ||
  !(
    asyncProp &&
    typeof asyncProp !== 'function' &&
    typeof asyncProp !== 'symbol'
  ) ||
  props.onLoad ||
  props.onError ||
  // ...
) {
  // This script will not be a resource, we bailout early and emit it in place.
  return pushScriptImpl(target, props);
}
```

这意味着带有 `defer` 属性（但没有 `async`）的脚本会被视为**非可提升资源**，直接在当前位置输出。

### 2. 客户端 Hydration (`ReactFiberConfigDOM.js`)

在 `canHydrateInstance` 函数中，对于 `async` 脚本有特殊处理（第 3877-8384 行）：

```javascript
case 'script': {
  const srcAttr = element.getAttribute('src');
  if (
    srcAttr !== (anyProps.src == null ? null : anyProps.src) ||
    element.getAttribute('type') !==
      (anyProps.type == null ? null : anyProps.type) ||
    element.getAttribute('crossorigin') !==
      (anyProps.crossOrigin == null ? null : anyProps.crossOrigin)
  ) {
    // ...
    if (
      srcAttr &&
      element.hasAttribute('async') &&  // <-- 只检查 async！
      !element.hasAttribute('itemprop')
    ) {
      // This is an async script resource
      break;  // 跳过，不视为 hydration 错误
    }
  }
  return element;
}
```

**问题**：代码检查了 `async` 属性来跳过 hoistable scripts，但**没有检查 `defer` 属性**！

### 3. `clearHead` 函数（第 3724-3743 行）

```javascript
function clearHead(head: Element): void {
  let node = head.firstChild;
  while (node) {
    const nextNode = node.nextSibling;
    const nodeName = node.nodeName;
    if (
      isMarkedHoistable(node) ||
      nodeName === 'SCRIPT' ||  // 所有 SCRIPT 都被保留
      nodeName === 'STYLE' ||
      (nodeName === 'LINK' &&
        ((node: any): HTMLLinkElement).rel.toLowerCase() === 'stylesheet')
    ) {
      // retain these nodes
    } else {
      head.removeChild(node);
    }
    node = nextNode;
  }
}
```

`clearHead` 会保留所有 `SCRIPT` 节点（无论是否 hoistable）。

## 解决方案

**核心修复**：在 `canHydrateInstance` 的 `case 'script':` 分支中，为 `defer` 脚本添加与 `async` 相同的处理逻辑。

### 修复代码

在 [ReactFiberConfigDOM.js](file:///Users/ll/Desktop/资料/编程/仓库/react/react/packages/react-dom-bindings/src/client/ReactFiberConfigDOM.js#L3877-L3884) 中：

```javascript
// 原来的代码（第 3877-8384 行）
if (
  srcAttr &&
  element.hasAttribute('async') &&
  !element.hasAttribute('itemprop')
) {
  // This is an async script resource
  break;
}

// 修改为：
if (
  srcAttr &&
  (element.hasAttribute('async') || element.hasAttribute('defer')) &&
  !element.hasAttribute('itemprop')
) {
  // This is an async or defer script resource
  break;
}
```

### 修复原理

1. **SSR 端**：`defer` 脚本不被视为 Resource（因为没有 `async` 属性），所以在 `<head>` 中直接渲染

2. **客户端 Hydration**：
   - 当 `defer` 脚本有匹配的 `src` 时，应该被识别为 hoistable Resource
   - `break` 语句会跳过该元素，不将其视为 hydration 错误
   - 这与 `async` 脚本的处理方式一致

3. **一致性**：修改后，SSR 和客户端对 `defer` 脚本的处理逻辑一致：
   - 两者都将带有 `src` 的 `defer` 脚本视为 Resource/hoistable
   - Hydration 时不会因为 script 顺序而产生 mismatch 错误

### 额外考虑

从语义上看，`async` 和 `defer` 脚本在以下方面是相同的：
- 都不会阻塞 HTML 解析
- 都带有 `src` 属性
- 都应该被视为"可提升的"资源
- 浏览器会自动处理它们的加载和执行

因此，将两者使用相同的跳过逻辑是合理的。

// Regression test for https://github.com/facebook/react/issues/36169
  // @gate大王
  it('should not warn about hydration mismatch for script with defer attribute in head', async () => {
    const writable = new Stream.PassThrough();
    writable.setEncoding('utf8');
    const chunks = [];
    writable.on('data', chunk => {
      chunks.push(chunk);
    });

    await act(() => {
      const {pipe} = ReactDOMServer.renderToPipeableStream(
        <html>
          <head>
            <script src="app.js" defer={true} />
          </head>
          <body>
            <div>Hello</div>
          </body>
        </html>,
      );
      pipe(writable);
    });

    const fullHTML = chunks.join('');
    const doc = document.implementation.createHTMLDocument('');
    doc.open();
    doc.write(fullHTML);
    doc.close();

    await act(() => {
      ReactDOMClient.hydrateRoot(
        doc,
        <html>
          <head>
            <script src="app.js" defer={true} />
          </head>
          <body>
            <div>Hello</div>
          </body>
        </html>,
      );
    });

    const clientScript = doc.querySelector('script[src="app.js"]');
    expect(clientScript).not.toBeNull();
    expect(clientScript.hasAttribute('defer')).toBe(true);
  });
