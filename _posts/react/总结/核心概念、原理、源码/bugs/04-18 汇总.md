# React Issues 适合研究的 Issues 总结

基于对 GitHub facebook/react 仓库的调研，以下是当前**具有研究价值**的 Issues 分类整理：

---

## 一、水合 (Hydration) 相关

### 1. **Hydration Mismatch 错误信息改进**
- **Issue**: [#36241](https://github.com/facebook/react/issues/36241)
- **日期**: 2026年4月（具体日期从搜索结果推断）
- **作者**:Kartik Pant
- **状态**: Status: Unconfirmed / Open
- **描述**: 当 React SSR 发生 hydration mismatch 时，错误信息过于通用（如 "Text content does not match server-rendered HTML."），缺乏可操作的调试信息。建议改进点包括：
  - 显示具体是哪个组件导致的不匹配
  - 提供服务端与客户端输出的 diff
  - 增强 stack trace 的有用性
  - 为大型组件树提供结构化诊断信息
- **研究价值**: ⭐⭐⭐⭐⭐
- **难度**: 中等 — 涉及 React 错误报告系统、水合机制调试输出
- **相关技术**: SSR、React.StrictMode、Next.js 等框架

### 2. Hydration mismatch on `<script defer>` inside `<head>`
- Issue: [#36169](https://github.com/facebook/react/issues/36169)
- **状态**: Status: Unconfirmed / Open
- **描述**: 当在 `<head>` 中渲染带有 `defer` 属性的 `<script>` 时出现 hydration 不匹配
- **研究价值**: ⭐⭐⭐⭐
- **难度**: 中等 — 涉及 DOM head 处理、script 标签 hydration 特殊逻辑

## 二、Effects 与生命周期相关

### 3. useEffect cleanup 在 StrictMode + concurrent features 下未调用
- **Issue**: [#36284](https://github.com/facebook/react/issues/36284) [[36284]]
- **日期**: 2026年4月17日
- **作者**: lukomb
- **状态**: Status: Unconfirmed / Open
- **描述**: 当组件在 `React.StrictMode` 且启用 concurrent features 的情况下 unmount 时，`useEffect` 的 cleanup 函数不会被调用。这可能与 React 18+ 的 StrictMode double-invocation 机制有关。
- **研究价值**: ⭐⭐⭐⭐⭐
- **难度**: 高 — 涉及 React 并发模式、StrictMode 行为、组件卸载生命周期
- **相关技术**: Concurrent Mode、useEffect cleanup、StrictMode

- **难度**: 高 — 涉及 React Fiber 架构、内存优化
- **相关技术**: Fiber、props/state 管理、内存泄漏
