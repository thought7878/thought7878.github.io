### 1. 面试题一：用过哪些状态管理方案，怎么选择？

#### 常见状态管理方案

- `useState` / `useReducer` / `useContext`（基础状态管理）
- Redux（集中状态管理）
- Zustand（轻量级集中状态管理）
- Jotai（原子状态管理）
- Valtio / XState / MobX / Recoil（其他状态管理库）

#### 选择依据

- 项目规模
- 状态复杂度
- 团队熟悉度
- 性能需求
- 开发工具支持

### 2. 面试题二：说说你对 Redux 的理解？

#### Redux 核心概念

- 单一状态树：整个应用的状态存储在一个对象中
- 纯函数 reducer：接收当前状态和动作，返回新状态
- 动作（action）：描述状态变化的普通对象，包含 `type` 和相关数据
- 中间件（middleware）：处理异步操作和副作用（如 Redux Thunk、Redux Saga）
- 单向数据流：组件派发 action → reducer 处理 → 生成新状态 → 更新组件
- 工具支持：如 Redux DevTools，用于调试状态变化

### 3. 面试题三：状态改变引发视图频繁更新，怎么优化？

#### 优化策略

1. 避免不必要的状态更新
    - 仅在必要时更新状态
2. 组件拆分
    - 拆分为多个小组件，只更新受影响部分
3. React Context 优化
    - 拆分多个 Context，避免单一 Context 引发全量更新
4. Redux 优化
    - 使用 `reselect` 创建 memoized 选择器
    - 使用 `redux-thunk` 或 `redux-saga` 处理复杂异步逻辑
    - 确保 reducer 是纯函数且高效
5. 批量更新
    - React 默认批量处理状态更新
    - 使用 `ReactDOM.unstable_batchedUpdates`手动控制
6. 避免过深的组件嵌套
    - 保持组件层次扁平化
7. 使用合适的状态管理工具
    - 小型项目可使用 Context API 或 Zustand
8. 细颗粒度更新
    - 使用 Zustand 的 `useSelector` API
    - 使用 Immer（基于 Proxy）实现细粒度更新

### 1. useState基础用法

#### 状态定义与更新

tsx

```tsx
import React, { useState } from "react";

export const UseStateDemo = () => {
  const [count, setCount] = useState(0);
  
  return (
    <div>
      {count}
      <button onClick={() => setCount(count + 1)}>+</button>
    </div>
  );
}
```

#### 两种状态更新方式

- 直接赋值更新  
    `setCount(count + 1)`
- 回调函数更新  
    `setCount((c) => c + 1)`

### 2. 状态更新触发机制

#### 触发条件

- 用户交互事件（如 `onClick`）
- 副作用（如 `useEffect` 中触发）
- 网络请求响应

#### 性能优化建议

- 使用 `useCallback` 缓存事件处理函数：
    
    tsx
    
    ```tsx
    const handleClick = useCallback(() => {
      setCount((c) => c + 1);
    }, []);
    ```
    

### 3. 状态初始化优化

#### 本地缓存初始化示例

tsx

```tsx
const getInitialCount = () => {
  const count = localStorage.getItem("count");
  return count ? Number(count) : 0;
};

  const [count, setCount] = useState(getInitialCount);
  
  return (
    <div>
      {count}
      <button onClick={() => setCount((c) => c + 1)}>+</button>
    </div>
  );
}
```

#### 初始化方式对比

|初始化方式|示例|特点|
|---|---|---|

- 状态更新是异步的，不能立即依赖更新后的值
- 使用回调函数方式更新状态可避免闭包问题
- 当状态值来源于异步操作时需注意类型转换（如 `localStorage` 读取返回字符串）

```tsx
  age?: number;
```

```tsx
      return {
        ...state,
        age: action.payload
      };
    default:
      return state;
  }
};
```

```tsx
    name: "John",
    age: 20
```

### 1. 集中状态管理核心思想

#### 解决的问题

- 管理跨模块/跨组件的复杂状态
- 统一状态管理入口
- 实现状态共享与同步

#### 核心原则

1. 使用对象存储状态（state）
2. 通过 reducer 管理状态变更
3. 定义动作（action）描述状态变化
4. 状态变更驱动视图更新

### 2. Redux与useReducer关系

#### 相似性

- 都使用 reducer 函数管理状态
- 都通过 dispatch 发起状态变更
- 都遵循单向数据流

#### 差异性

- `useReducer` 是 React 内置 Hook，适用于组件内部复杂状态
- `Redux` 是独立的状态管理库，适用于全局状态管理
- `Redux` 提供了更完善的状态持久化、中间件等扩展能力

### 3. Redux核心概念

#### 核心要素

- Store：存储整个应用的状态树
- Action：描述状态变化的普通对象，包含 type 和 payload
- Reducer：纯函数，接收当前状态和 action，返回新状态
- Dispatch：发起 action 的方法
- Selector：从 store 中提取状态的函数

#### 单向数据流

1. View 触发 action
2. Store dispatch action
3. Reducer 处理 action 生成新 state
4. Store 更新状态
5. View 订阅状态变化并更新

### 4. Redux使用基础

#### 安装依赖

bash

```bash
npm install redux react-redux
## 或
pnpm add redux react-redux
```

#### 简单示例

tsx

```tsx
// 定义 action type
const INCREMENT = 'INCREMENT';
const DECREMENT = 'DECREMENT';

// 定义 reducer
function counterReducer(state = { count: 0 }, action) {
  switch (action.type) {
    case INCREMENT:
      return { count: state.count + 1 };
    case DECREMENT:
      return { count: state.count - 1 };
    default:
      return state;
  }
}

// 创建 store
import { createStore } from 'redux';
const store = createStore(counterReducer);

// React 组件中使用
import { Provider, useSelector, useDispatch } from 'react-redux';

function Counter() {
  const count = useSelector(state => state.count);
  const dispatch = useDispatch();
  
  return (
    <div>
      <p>{count}</p>
      <button onClick={() => dispatch({ type: 'INCREMENT' })}>+</button>
      <button onClick={() => dispatch({ type: 'DECREMENT' })}>-</button>
    </div>
  );
}

// 根组件包裹
function App() {
  return (
    <Provider store={store}>
      <Counter />
    </Provider>
  );
}
```

```ts
function combineReducers(reducers) {
```

#### applyMiddleware实现

```ts
function applyMiddleware(...middlewares) {
  return (createStore) => (reducer, initialState) => {
    const store = createStore(reducer, initialState);
    let dispatch = store.dispatch;
    
    const middlewareAPI = {
      getState: store.getState,
      dispatch: (action) => dispatch(action)
    };
    
    const chain = middlewares.map(middleware => middleware(middlewareAPI));
    dispatch = compose(...chain)(store.dispatch);
    
    return { ...store, dispatch };
  };
}
```
