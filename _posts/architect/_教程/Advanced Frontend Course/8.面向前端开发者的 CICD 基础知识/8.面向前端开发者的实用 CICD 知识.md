回滚错误部署的第一种方法是什么？它涉及什么？  
答：回滚错误部署的第一种方法是部署最后一个已知的良好项目。 这涉及到保留代表每个版本的不可变工件，如Docker标签。 如果新部署出现问题，例如遇到错误或错误率增加，该过程涉及停止有问题的应用程序版本，然后部署上一个已知的稳定版本以将系统恢复到工作状态。  
  
蓝绿部署如何工作，为什么它在关键系统中很受欢迎？  
答：蓝绿部署的工作原理是使应用程序的新旧版本同时运行。 部署新版本时，用户会被定向到新的 “绿色” 版本。 如果出现问题，例如错误率峰值或监控警报，可以通过更新负载均衡器或DNS以重新指向旧的 “蓝色” 版本，立即将流量切换回旧的 “蓝色” 版本。 此方法在关键系统中很受欢迎，因为它提供了即时回滚功能，确保旧版本随时可用并且可以切换回而无需重新部署或重新启动任何内容。  
  
使用金丝雀部署降低到0% 的过程是什么，它的优势是什么？  
答：使用金丝雀部署将减少到0% 的过程涉及最初将一小部分流量 (例如5%) 定向到新版本，并使用如新遗物，Grafana或哨兵等工具监控系统的性能。 如果检测到问题，例如错误率或响应时间增加，负载均衡器配置中的权重可以降低到0%，立即将所有流量引导回旧的稳定版本。 这种方法的优点是它限制了潜在问题的影响，因为只有一部分用户受到新版本的影响，而不是整个用户群 (如果已经完全推出) 的影响。 这允许检测和解决较小的受控用户组的问题，限制任何潜在问题的爆炸半径。

## 重新部署 回滚策略
![[_posts/architect/_教程/Advanced Frontend Course/8.面向前端开发者的 CICD 基础知识/media/7be2219325a8acfe3b6012997cd98f4a_MD5.png]]

回滚策略 (Rollback Strategies)

**重新部署已知良好的构件 (Re-deploy last known good artifact)**
保留不可变构件（Docker 标签、构建 ID）。如果出现问题，执行回滚。

流程图解：
1.  **最新版本 (Latest version v1.2.0)**  
    *   当前运行的是最新版本（v1.2.0），但检测到问题（黄色警告三角）。
    *   Nginx 将流量导向该版本。
2.  **停止应用 (Stop app)**  
    *   检测到问题后，立即停止当前应用（黄色警告三角）。
    *   Nginx 不再将流量导向此版本。
3.  **回滚到已知稳定版本 (Rollback to the last known stable version v1.1.5)**  
    *   回滚到上一个已知稳定的版本（v1.1.5）。
    *   Nginx 重新将流量导向该稳定版本。
4.  **运行稳定版本 (Run stable version)**  
    *   稳定版本（v1.1.5）成功运行（绿色对勾）。
    *   Nginx 正常将流量导向该稳定版本，服务恢复。


## 蓝绿部署 回滚策略
![[_posts/architect/_教程/Advanced Frontend Course/8.面向前端开发者的 CICD 基础知识/media/70b1488425e9fae89646a11b128c5b2a_MD5.png]]

回滚策略 (Rollback Strategies)

**切换流量（蓝绿部署） (Flip traffic (Blue-Green))**
保持蓝色（旧版本）运行。部署绿色（新版本）。
如果新版本有问题 -> 切换回蓝色。

图解说明：
*   **步骤 1 (左侧图)：**
    *   **Old Version (旧版本/蓝色)**：正在运行，但目前的连线是灰色的（表示没有接收主要流量）。
    *   **New Version (新版本/绿色)**：部署完成，Nginx 将流量（人形图标）切换到了这里。但是，新版本旁边出现了**黄色警告标志**，表示出现了问题。
*   **步骤 2 (右侧图)：**
    *   **回滚操作**：因为新版本有问题，Nginx 将流量（人形图标）重新切换回了 **Old Version (旧版本/蓝色)**。
    *   **New Version (新版本/绿色)**：虽然还在运行（带有警告标志），但不再接收流量（连线变灰）。这就是蓝绿部署回滚的优势：只需切换路由，速度极快。


## 金丝雀部署 回滚策略
![[_posts/architect/_教程/Advanced Frontend Course/8.面向前端开发者的 CICD 基础知识/media/ceea3b1b2b99fe29fa899c0b1d59ce9b_MD5.png]]

回滚策略 (Rollback Strategies)

**降至 0%（金丝雀部署） (Reduce to 0% (Canary))**
从 1-5% 开始 -> 观察指标 -> 如果出现问题，将金丝雀（新版本）的权重设为 0%。

图解说明：
*   **步骤 1 (左侧图)：**
    *   **Nginx** 将流量分发给 **Old Version (Stable)**（旧版本/稳定版，蓝色图标）和 **New Version**（新版本，绿色图标）。
    *   图中显示新版本接收了 **33%** 的流量（虽然文字描述说是从 1-5% 开始，这里可能是示意流量已经增加到了 33%）。
    *   新版本旁边出现了**黄色警告标志**，表示监控发现新版本有问题。
*   **步骤 2 (右侧图)：**
    *   **回滚操作**：一旦发现问题，立即将新版本的流量权重降为 0。
    *   **Old Version**（旧版本）重新接收 **100%** 的流量。
    *   所有实例图标都变回了蓝色（代表旧版本/稳定版），服务恢复正常。
