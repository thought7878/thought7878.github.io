问：理解SASC管道中的分支策略和代码流的目的是什么？  
答：理解SASC管道中的分支策略和代码流至关重要，因为它会影响代码的管理和部署方式，进而影响管道在使用不同分支时的行为。  
  
问：典型的分支机构保护和工作流系统中有哪些不同类型的分支机构？  
答：在典型的分支保护和工作流系统中，有几种类型的分支: 功能分支、开发分支、主要 (暂存或主要) 分支和生产分支。  
  
问：功能分支在开发过程中如何运作？  
答：功能分支是开发人员开发特定功能的地方。 每个开发人员都会从主分支创建一个功能分支，在那里实现他们的功能，然后创建一个拉取请求以合并回主分支。  
  
问：开发分支在工作流程中的角色是什么？  
答：开发分支是可选的，如果团队使用功能分支，则可以跳过。 它用于开发人员的测试，并不期望它是稳定的; 它充当多个功能的集成点。  
  
问：为什么主要分支机构被认为是强制性和受保护的？  
答：主分支，也称为暂存或主分支，被认为是强制性和受保护的，因为它代表生产中运行的内容，除了通过功能分支的拉取请求外，不允许直接提交。  
  
问：生产分支在稳定性方面与其他分支有何不同？它代表了什么？  
答：生产分支是生产中运行的内容的精确副本，代表了实时环境，并且预计会保持稳定。 它用于利益相关者、产品经理、设计师和质量管理团队在更改部署到生产之前进行最终测试。  
  
问：为不同阶段 (开发、试运行、生产) 使用不同的数据库的目的是什么？  
答：为不同阶段使用不同的数据库的目的是防止开发人员使用真实的用户数据测试功能。 为每个阶段使用单独的数据库可确保开发人员在开发或分阶段环境中测试时不会意外污染生产数据。  
  
问：当开发人员将代码提交到功能分支时，预览环境如何运行？  
答：当开发人员将代码提交到功能分支时，它会触发自动构建过程 (例如，为单页应用程序运行NPM构建)，这会导致静态文件直接上传到AWS S3，而不需要Docker容器。 然后，CloudFront CDN生成一个特定于该功能分支的自定义URL，该URL用作预览环境的入口点。 这允许团队在合并到生产之前隔离访问和审查更改。

*   `Feature Branch (功能分支)`： 用于在接近生产的环境中进行测试，然后再实际合并 PR（拉取请求）。

# 预览环境 (Preview Environment)
在生产合并之前创建独立的预览环境以进行代码测试
03:12
开发人员将代码提交到功能分支，触发自动构建，生成上传到AWS S3的静态文件。 CloudFront CDN为每个功能创建独特的URL，使团队成员能够在合并到生产环境之前测试和查看隔离的预览，确保在开发期间不会干扰主要的暂存环境。


![[_posts/architect/_教程/Advanced Frontend Course/8.面向前端开发者的 CICD 基础知识/media/64c49f7ac9710e77a67f923490366174_MD5.png]]

预览环境 (Preview Environment)

*   顶部流程（部署流程）：
    *   Deploy on Preview Environment (在预览环境上部署)
    *   开发者提交代码 (Commit) 到 GitHub。
    *   分支 1： 运行 `npm run build` -> 上传至 Amazon S3。
        *   *注释：* (对于静态 Web 应用，我们不需要 Docker 镜像)。
        *   *说明：* AWS S3 是一个用于*存储文件的云盘*。
    *   分支 2： AWS CloudFront 自动生成自定义 URL (feature-123.my-app.com)。
*   底部流程（测试流程 - Testing）：
    *   用户访问 `feature-123.my-app.com` -> AWS CloudFront -> Amazon S3。
    *   *说明：* AWS CloudFront 将此 URL 指向 S3 中的静态文件。


*   **开发者将代码提交到功能分支**，触发 GitHub Actions 使用 `npm run build` 自动构建静态 Web 应用程序。
*   **静态文件直接上传到 AWS S3 存储**，无需 Docker 容器化，因为静态 Web 应用只是 HTML、CSS 和 JavaScript 文件。
*   **CloudFront CDN 自动生成一个自定义 URL** (feature-123.my-app.com)，作为预览环境的入口点。
*   **团队通过自定义 URL 访问预览环境**，CloudFront 从 S3 存储桶检索并提供静态文件以供测试和审查。
*   **整个过程是自动化的** —— 从代码提交到实时预览环境 —— 使团队能够在合并到生产环境之前，隔离地测试功能分支。
