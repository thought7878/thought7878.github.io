
React钩子的两个主要用例是什么？  
答：React钩子的两个主要用例是*逻辑提取*和*代码重用*。  
  
逻辑提取有什么好处？  
答：**逻辑提取的好处**包括*能够在不渲染的情况下测试逻辑，为数据获取和状态逻辑编写单元测试而无需安装正确的组件*，并且*通过处理除UI钩子之外的所有内容来保持组件的简单*。  
  
什么是代码重用，如何在提供的示例中演示？  
答：**代码重用**是将公共逻辑提取到可跨多个组件使用的单独组件或 “钩子” 中的实践。 示例包括管理用户数据获取的函数和控制组件状态 (如模型或侧边栏) 的切换钩子。  
  
什么时候不建议将代码提取到共享钩子中？  
答：*当逻辑服务于不同目的或代码重复正常时*，不建议将代码提取到共享钩子中，因为它会随着时间的推移而发散，例如由于不同的原因可能需要更改的运输和税收计算。  
  
虚拟DOM在React中的目的是什么？  
答：virtual DOM in React用于与旧版本进行高效比较，以仅更新浏览器中实际需要更改的内容，这有助于提高性能并使开发人员更高效。  
  
开发人员应该如何选择用于应用程序开发的框架？  
答：开发人员应该选择适合自己情况的框架，考虑最大的就业市场和社区 (React) 、欧洲和亚洲更好的开发人员经验 (Vue) 以及最小的社区和就业市场 (Svelte) 等因素。 对于简单的应用程序，钩子工作得很好，但复杂的应用程序需要分层或干净的架构模式。

![[_posts/architect/_教程/Advanced Frontend Course/2.现代前端框架/media/d6a35630235e5712def7bb220ab646d7_MD5.png]]

React Hooks 和 Vue Composables (组合式函数)

**使用场景 (Use Cases)**
*   **逻辑提取 (Logic Extraction)。** *从 UI 组件中提取有状态的行为和副作用，适用于小型/中型应用；大型应用应该使用合适的架构层来处理业务逻辑，而不是依赖 hooks*。
*   **代码复用 (Code Reuse)。** 共享因相同业务原因而变化的逻辑，并非所有重复代码都应该被提取——只移动那些**有相同变更原因**的代码。

**优势 (Benefits)**
*   **无需渲染即可测试逻辑。** 编写数据获取和状态逻辑的单元测试，而无需挂载 React 组件。
*   **保持组件简洁。** 组件只负责处理 UI —— hooks 处理其他所有事情。
*   **减少复制粘贴式编码。** 编写一次逻辑，在多个组件中复用。

右侧代码示例 (index.ts):
这是一个 React Custom Hook 的示例，展示了如何封装获取用户数据的逻辑：
```javascript
function useUser(userId) {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(false);

  useEffect(() => {
    fetch(`/api/users/${userId}`)
      .then(res => res.json())
      .then(setUser);
  }, [userId]);

  return { user, loading };
}
```
*(代码展示了如何封装状态 `user` 和 `loading` 以及副作用 `useEffect`，最后返回数据供组件使用。)*

标题：何时代码重复是可以接受的 (When duplication is fine)

**不要**将重复的代码提取为 `useCostCalculation()`
即使代码看起来*完全相同*，但它们服务于*不同的目的*：
*   当配送规则改变时，**运费 (Shipping)** 会发生变化
*   当政府法规改变时，**税费 (Tax)** 会发生变化

```javascript
// 这些看起来一样，但服务于不同的业务领域
function ShippingCalculator({ weight, distance }) {
  const [cost, setCost] = useState(0);

  useEffect(() => {
    // 目前公式相同，但将来会分化：
    // 运费：燃油成本、承运商费率、区域划分
    setCost(weight * distance * 0.1);
  }, [weight, distance]);

  return <div>Shipping: ${cost}</div>;
}

function TaxCalculator({ weight, distance }) {
  const [cost, setCost] = useState(0);

  useEffect(() => {
    // 目前公式相同，但将来会分化：
    // 税费：司法管辖区、免税政策、税率
    setCost(weight * distance * 0.1);
  }, [weight, distance]);

  return <div>Tax: ${cost}</div>;
}
```
  
# 如何选择React、Vue和Svelte
03:25
对话讨论了React、Vue和Svelte在web开发中的优势，强调了React的生产力、Vue的开发经验和Svelte的速度。 它建议对简单的应用程序使用钩子，对复杂的应用程序使用适当的架构，强调编写良好的应用程序优先考虑结构而不是框架选择。

![[_posts/architect/_教程/Advanced Frontend Course/2.现代前端框架/media/10107cb642814d16fff6bc0d7c3a5080_MD5.png]]

**关键要点 (Key Takeaways)**
*   **VDOM 是 React 和 Vue 的渲染方式** - 它们在 JavaScript 中创建你 UI 的副本，将其与旧版本进行比较，然后只更改浏览器中实际需要更新的部分。
*   **React 专注于提高开发者的生产力** - 开发速度更快，因为你不需要手动更新 DOM 元素，而且标准化的模式意味着开发者可以在不同的 React 项目或公司之间切换，而无需学习新的方法。
*   **选择适合你情况的框架** - React 拥有最大的就业市场和最大的社区。Vue 在欧洲/亚洲很受欢迎，并提供更好的开发者体验 (DX)。Svelte 速度最快，但社区和就业市场较小。
*   **简单应用使用 Hooks，复杂应用使用合适的架构** - Hooks 非常适合在小项目中将逻辑与 UI 分离，但大型应用程序需要真正的架构模式（如分层架构、整洁架构等）。
*   **你怎么构建比用什么更重要** - 一个写得好的 Vue 应用胜过一个写得混乱的 React 应用。
