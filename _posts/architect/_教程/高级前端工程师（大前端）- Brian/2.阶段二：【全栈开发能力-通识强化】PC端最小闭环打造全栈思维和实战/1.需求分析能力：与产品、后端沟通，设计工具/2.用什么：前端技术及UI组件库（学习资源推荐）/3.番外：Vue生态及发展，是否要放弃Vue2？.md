### 是否可以直接学习 Vue 3？
[02:09](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=129)

- **适用人群说明 [02:14](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=134)**  
    主要针对从未接触 Vue 2 的初学者。
- **建议直接学习 Vue 3 的三大原因 [02:18](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=138)**
    - **灵活性与心智模型更优**
        - 相较 React，Vue 的单文件组件（SFC）将 template、JS/TS 和 style 有效整合。
        - 代码结构更简洁，逻辑更清晰，易于理解。
        - 在纯 HTML 环境中使用便捷，生态适配性更强。
        - 社区驱动虽无大厂背书，但灵活性高，适合多种场景。
        - 心智模型比 React 更简单，避免 JSX 中模板与逻辑混杂带来的理解跳跃。
    - **紧跟行业发展趋势 [04:28](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=268)**
        - Vue 3 使用 TypeScript 重写，贴合现代前端主流技术栈。
        - 默认打包工具为 Vite，追求极致构建性能。
        - 在 SSR（服务端渲染）方面表现优异。
    - **继承并兼容 Vue 2 核心特性 [04:53](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=293)**
        - Vue 3 兼容 Vue 2 超过 90% 的写法。
        - 提供两种语法选项：`setup` 与 `options` API。
        - 学习者可根据习惯选择，平滑过渡。

### Vue 2 到 Vue 3 的升级策略 
[06:28](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=388)

- **主要变化点 [06:28](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=388)**
    - `mixins` 被废弃。
    - 状态管理方案从 Vuex 迁移至 Pinia（官方推荐）。
    - 编程风格转向函数式，推崇 Composition API。
- **升级路径建议 [06:44](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=404)**
    - **策略一：分阶段重构**
        - 先切换非核心业务模块，逐步推进至主干功能。
        - 确保不影响当前业务运行的前提下完成技术迭代。
    - **策略二：打补丁式升级**
        - 在 Vue 2 项目中引入 Composition API 插件。
        - 将部分代码改写为 Composition 风格，为后续升级做准备。
