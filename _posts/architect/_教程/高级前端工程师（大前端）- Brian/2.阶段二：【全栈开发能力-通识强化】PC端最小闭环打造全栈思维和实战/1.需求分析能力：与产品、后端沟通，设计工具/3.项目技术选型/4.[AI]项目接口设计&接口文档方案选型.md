### **接口文档工具概述与选型依据 [00:01](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=1)**

- **工具推荐与适用场景**
    - 推荐 ShowDoc、RAP2 三款主流接口文档工具，均支持本地化部署。
    - 金融等高安全要求企业可部署于内网，保障数据隐私与网络可达性。
- **部署灵活性**
    - 多数工具支持 Docker 及 docker-compose 一键部署，降低运维门槛。
    - 部署方式选择取决于团队技术栈偏好（如是否已集成 Docker 生态）。
- **协同替代方案**
    - OA 系统、石墨文档、飞书等亦被部分团队用作轻量级接口协同文档工具。

### **接口文档核心功能与模板机制 [01:21](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=81)**

- **标准化模板支撑**
    - 新建接口需包含：接口名称（如 `hello`）、URL（如 `/hello`）、请求方法（GET/POST）、请求参数、响应内容。
    - RAP2 已预置结构化模板，支持导入 JSON 格式需求文档数据并自动填充参数字段。
- **可视化编辑与预览**
    - 支持实时编辑请求参数与响应体；点击“预览”即可查看请求模板、示例数据、响应模板等完整视图。

### **ShowDoc 多端能力与调试集成 [02:08](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=128)**

- **跨平台访问支持**
    - 提供 PC 端与移动端客户端，便于随时随地浏览文档。
- **内置 API 调试工具**
    - “Run API” 客户端功能类似 Postman，支持直接发起请求、查看响应、调试接口。
- **线上服务特性**
    - 在线版提供 AI 助手生成能力（类 ChatGPT），可辅助编写代码示例（如冒泡排序）；但接口文档需严格遵循标准 API 描述格式。

### **本地化部署与线上版关键差异 [02:52](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=172)**

- **核心区别三点**
    - 数据安全性：本地部署确保敏感接口信息不出内网。
    - 网络可访问性：避免因外网断连导致文档不可用。
    - AI 功能限制：本地版无 AI 生成接口能力，但提供丰富实用工具（JSON 转参数表格、JSON 格式化、Mock 服务、在线测试等）。

### **高效文档生成工具链 [03:51](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=231)**

- **JSON 数据驱动表格生成**
    - 支持将业务需求文档中原始 JSON 数据粘贴至“JSON 转参数表格”工具，自动生成规范参数表。
    - 注意事项：需先在 VS Code 等编辑器中对 JSON 进行 format，消除首尾空格、末尾逗号等格式错误。
- **配套实用工具集**
    - 包含排序、JSON 格式化、粘贴插入表格、CURL 转表格、Mock 工具、在线 API 测试（Run API）等。

### **结构化项目文档目录体系 [05:22](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=322)**

- **三级标准目录框架**
    - **版本/项目说明**：含项目简介、作者、技术栈、架构、模块划分等。
    - **数据库**：基于数据库字典模板构建用户表等核心表结构。
    - **功能模块**：按业务域组织接口（如首页、学习页、社区页）。
- **项目说明 AI 辅助生成流程**
    - 第一步：向 AI（如 ChatGPT/CodeGeex）提问“接口文档项目说明的标准格式”，获取 Markdown 模板结构。
    - 第二步：构造精准 Prompt，明确要求：
        - 输入信息项（项目名、版本号、作者、邮箱、功能简介、模块列表、技术选型等）；
        - 输出格式为 Markdown，禁止额外解释或确认；
        - 按模板分类（如“技术选型”“模块划分”“部署说明”）逐项展开。
    - 第三步：人工校验并补充细节（如 `VO3 + TypeScript + Node.js + NestJS`，`Prisma + Docker + Linux`，`Next.js SSR + NestJS + HTTPS`），再交由 AI 重写扩展。

### **数据库与接口定义实操演示 [14:40](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=880)**

- **数据库字典构建**
    - 选用“数据库字典模板”，依原型图与需求文档设计 `users` 表：含 `id`、`username`、`password`、`nickname`、`created_at` 字段。
- **功能模块接口定义规范**
    - 接口命名习惯：`/home`（功能模块） + `GET`（请求类型），不重复添加 `/api/v1` 前缀。
    - 响应结构示例：
        - `code`: `$200$`（成功）或 `$500$`（服务端异常）；
        - `data`: `$object$`，含 `swipers`: `$array$`（字符串数组，存轮播图 URL）；
        - 可扩展字段：`title`、`link` 等，依实际业务需求补充。
- **后续扩展指引**
    - 首页除轮播图外，需定义项目分类、课程展示（含 `image`, `title`, `description`, `link`）、合作伙伴等模块接口；细节随开发迭代持续完善。
