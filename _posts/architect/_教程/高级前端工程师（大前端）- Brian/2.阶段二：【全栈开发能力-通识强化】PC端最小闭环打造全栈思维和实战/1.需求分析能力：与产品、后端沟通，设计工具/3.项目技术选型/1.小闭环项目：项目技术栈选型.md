## 基础通用功能层选型 
[03:53](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=233)
    
- **请求库选择 Axios**
    - 不使用原生 `fetch` 的原因：
        - `fetch` 未封装，需手动处理请求配置。
        - 缺乏拦截器和统一异常处理机制。
    - Axios 提供：
        - 请求/响应拦截器。
        - 自动转换 JSON 数据。
        - 更好的错误处理机制。
        - 封装完善，稳定性高。
    - 推荐在非学习目的且项目周期允许的情况下使用 Axios。
- **Mock 接口支持**
    - 前端常需模拟交互和页面模块，必须具备 mock 功能。
- **CSS 预编译语言与样式方案**
    - 个人偏好差异大，常见选择包括原生 CSS、Sass、Less。
    - 近年来流行直接使用 class 类名绑定的方式编写样式。
    - JSX/TSX 中避免行内样式，因其维护困难且结构混乱。
    - 使用 CSS 框架有助于减少项目样式体积。

## 服务端渲染与性能优化方案 
[06:46](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=406)
    
- 项目涉及服务端框架与性能优化。
- Vue 和 React 均有官方 SSR 框架：
    - React：Next.js
    - Vue：**Nuxt.js**
- 因此直接选用 **Nuxt.js** 框架进行服务端渲染与性能优化。
- 无需第三方替代方案。

## 服务端框架选型 
[07:21](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=441)
    
- 服务端技术选型不深入讲解，后续章节将详述。
- 综合对比 Node.js 相关框架，考量扩展性、便利性与性能。

## 服务器与部署环境 
[08:01](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=481)
    
- 企业级部署通常基于 Linux 系统。
