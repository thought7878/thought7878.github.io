### interface基础定义与使用

    Interface翻译为“接口”，是TS中最重要的概念；TS通过类型查验代码，基础类型仅定义简单类型，而接口可定义任意结构或类型。
- **接口语法结构** [00:45](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=45)

- **属性声明规范** [01:03](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=63)  
    属性名后跟冒号与类型（如`x: number`），冒号表示属性类型而非赋值；分号与逗号均可用于分隔属性。
- **接口作为类型使用** [01:43](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=103)  
    接口定义后可直接作为类型标注变量，如`const myPoint: Point = { x: 1, y: 2 };`。
- **强类型约束验证** [01:56](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=116)  
    实例化时必须包含所有必选属性且类型匹配：`x`和`y`必须为`number`；添加未定义属性（如`a`）或错误类型值均报错。

### 同名接口自动合并

    定义同名接口（如再次`interface Point`）时自动合并属性。
- **合并结果验证** [03:30](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=210)

    同名属性类型必须完全一致：`x: number`与`x: number`可合并；`x: number`与`x: string`或`x: string | number`均报错。

### 索引签名（Index Signature） 
[08:57](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=537)

- **数组索引签名** [09:01](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=541)  
    `interface Row { [index: number]: string; }`，定义数字索引对应字符串值。
- **索引签名使用验证** [09:21](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=561)  
    `const row: Row = ['super', 'me'];`（数组形式）与`const row: Row = { 0: 'super', 1: 'me', 5: 'user' };`（对象形式）均合法；但`{ a: 'str' }`因`a`非`number`索引而报错。
- **索引签名与原型链区别** [10:42](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=642)  
    `row.length`访问失败：索引签名定义的类型不继承数组原型方法，仅保留接口声明的索引结构。

### 绕过多余属性检查 
[11:44](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=704)

- **问题场景** [11:58](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=718)  
    函数参数为接口类型（如`getTypes(myType: MyType)`），传入对象字面量含额外属性（如`{ color: 'red', type: 'color' }`）时严格校验报错。
- **方案一：类型断言** [13:20](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=800)  
    `getTypes({ color: 'red', type: 'color' } as MyType)`，用`as`关键字强制转换类型，屏蔽多余属性校验。
- **方案二：索引签名（推荐）** [13:49](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=829)  
    在接口中添加`[propName: string]: any;`，允许任意字符串索引及任意类型值，从而接纳所有额外属性。
- **方案三：类型兼容（不推荐）** [15:00](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=900)  
    解构传入对象仅提取必需属性（如`const { color } = options; getTypes({ color });`），依赖TS类型兼容性，但易导致逻辑遗漏。
