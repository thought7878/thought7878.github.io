### 函数定义方式对比与规范建议 
[08:31](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=511)

- **三种定义方式归类 [08:37](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=517)**
    - 声明式：`function fn(...) {...}`
    - 匿名函数式：`const fn = function(...) {...}` 或 `const fn = (...) => {...}`
    - 类型注解式：`const fn: (...) => T = (...) => {...}`
- **参数类型统一规则 [08:48](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=528)**  
    所有方式中，参数类型均直接写在形参后（如 `a: number`）。
- **返回值类型标注差异 [09:00](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=540)[09:10](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=550)**
    - 声明式：函数名后 `:` 标注整体返回类型；
    - 匿名/注解式：参数列表后 `:` 或 `\=\>` 后标注返回类型。
- **风格一致性要求 [09:38](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=578)**  
    单项目或单文件内须统一采用一种函数定义风格，以保障可维护性与团队协作友好性。
