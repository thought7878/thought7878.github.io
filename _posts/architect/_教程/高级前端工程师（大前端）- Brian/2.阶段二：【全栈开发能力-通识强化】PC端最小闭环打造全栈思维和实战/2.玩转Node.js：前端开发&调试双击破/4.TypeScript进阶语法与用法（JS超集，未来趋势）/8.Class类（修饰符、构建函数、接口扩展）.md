- **类的基本定义与默认修饰符 [01:23](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=83)**
    
    - 使用`class`关键字（全小写）定义类，如`class Person`。
    - 类内可定义属性（如`name = "Tim Mark"`）和方法（如`getName() { return this.name; }`）。
    - 若属性/方法前未显式声明修饰符，默认为`public`。
- **访问修饰符详解 [00:32](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=32)**
    
    - **public [00:34](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=34)**：公共的，允许在类内部及外部访问和调用。
    - **protected [02:17](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=137)**：受保护的，允许在类内部及其子类中访问（非拼音课，但强调发音：/prəˈtɛk.tɪd/）。
    - **private [02:55](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=175)**：私有的，仅允许在定义该成员的类内部访问，外部不可见。
    - **readonly [00:38](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=38)**：只读属性，初始化后不可重新赋值。
    - 实际开发中多数使用`public`；`private`用于封装敏感数据，如`private name`导致外部直接访问`person.name`报错，需通过`public getName()`间接获取。
- **类的实例化与方法调用 [03:33](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=213)**
    
    - 使用`new Person()`创建实例，实例可直接调用`public`方法（如`person.getName()`）和`public`属性（如`person.name`）。
    - `private`属性无法被实例直接访问，但可通过`public`方法暴露其值。
- **implements关键字与类实现接口 [04:47](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=287)**
    
    - `implements`表示类实现（而非继承）接口，用于约束类必须具备接口所定义的结构。
    - 示例：定义`interface Food { type: string; }`，则`class FoodClass implements Food`必须包含`type`属性。
    - 若在类体外声明`type: string`但未初始化，会报错“Property 'type' has no initializer”；正确方式是在构造函数参数中使用修饰符快捷声明：`constructor(public type: string)`，等价于在构造函数体内手动赋值`this.type = type`。
- **类的继承（extends）机制 [04:50](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=290)**
    
    - `extends`表示类继承父类，子类获得父类所有`public`和`protected`成员。
    - 示例：`class PersonOne extends Person`，实例`new PersonOne()`可调用父类`getName()`，输出`"Tim Mark"`。
    - 子类可重写父类方法（如重写`getName()`返回`"New Tim Mark"`），调用时优先执行子类实现。
- **构造函数与super调用规则 [06:47](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=407)**
    
    - 构造函数`constructor()`在`new`实例化时自动执行。
    - 子类若定义了`constructor()`，**必须**显式调用`super()`，以确保父类初始化完成。
    - `super()`调用父类构造函数；`super.methodName()`可调用父类方法。
    - 若省略`super()`，编辑器报错：“Constructors for derived classes must contain a 'super' call”。
    - 逻辑类比：无父类（苹果）则无子类（香蕉苹果），`super`即先构建父类再构建子类。
- **接口继承类（interface extends class）[11:53](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=713)**
    
    - 接口可通过`extends`继承类，此时接口会继承该类的所有成员（含`private`和`protected`）的**类型定义**，但**不继承其实现**。
    - 示例：`interface I extends Person`，则`I`隐含要求实现类必须具备`name`（`private`）、`name1`（`protected`）及`getName()`（`public`）的类型签名。
    - 关键限制：**该接口只能被原类或其子类实现**（即`class C implements I`非法，除非`C extends Person`）。
    - 原因：`private`成员不可被外部类声明或覆盖，`protected`成员仅子类可访问；因此`C`必须是`Person`的子类才能满足约束。
    - 验证：`class C extends Person implements I`合法，且`C`实例可调用`getName()`，输出`"Tim Mark"`。
- **类与接口关系总结 [11:38](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=698)**
    
    - 类与类、接口与接口之间用`extends`（继承/扩展）。
    - 类与接口之间用`implements`（实现）。
    - 接口继承类时，`extends`仅继承成员类型，不继承实现；且因含`private`/`protected`成员，强制实现者必须是该类或其子类。
