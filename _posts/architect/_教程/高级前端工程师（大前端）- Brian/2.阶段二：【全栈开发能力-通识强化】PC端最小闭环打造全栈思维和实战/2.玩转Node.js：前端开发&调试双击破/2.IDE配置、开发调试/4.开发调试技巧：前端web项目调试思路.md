### **前端常规调试方法 [00:24](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=24)**

- **常见调试手段提问 [00:24](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=24)**  
    提出三个常见的前端调试关键词：`alert`、`console`和断点调试。
- **alert 与 console 的区别 [00:34](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=34)**
    - `alert` 和前面提到的某些方法是**阻塞式**的，程序执行到该处会暂停。
    - `console.log()` 不会阻塞程序运行。
- **断点调试简介 [00:42](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=42)**
    - 断点调试与 `alert` 类似，但由开发者在浏览器的 **Sources** 面板中手动设置。
    - 程序刷新后会在设定的断点处自动停止。
- **alert 的使用场景 [01:04](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=64)**
    - 属于较“暴力”的调试方式。
    - 多用于老旧项目（如 HTML + JS 而非 ES6 规范编写的项目）。

### **console 的高级用法扩展 [01:31](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=91)**

- **console 日志级别输出 [01:37](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=97)**  
    可使用不同级别的日志函数打印信息：
    - `console.info()`：普通信息。
    - `console.warn()`：警告信息，显示为黄色。
    - `console.error()`：错误信息，显示为红色。
- **时间测量功能 [01:52](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=112)**  
    使用 `console.time()` 和 `console.timeEnd()`测量代码段执行时间，用于性能调优。
- **清空控制台 [02:01](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=121)**
    - 通常点击浏览器调试面板上的清空按钮。
    - 也可通过命令 `console.clear()` 手动清除。
- **回放与选择功能 [02:10](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=130)**
    - `$0` 表示当前在 Elements 面板中选中的 DOM 元素。
        - 示例：选中一个 `<p>` 元素后，在 Console 输入 `$0` 回车，返回该元素对象。
    - 可对 `$0` 使用标准 DOM 查询方法，如 `$0.querySelector()`。
    - `$_` 表示上一次表达式的执行结果。
        - 示例：输入 `1 + 1` 后，再输入 `$_`，将返回 `2`，实现表达式回放。
- **表格输出 [03:08](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=188)**  
    使用 `console.table()` 以表格形式展示数组或对象，清晰显示其属性与方法。
- **分组输出 [03:20](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=200)**  
    使用 `console.group()` 和 `console.groupEnd()` 对日志进行分组折叠显示，提升可读性。

- **无痕模式的重要性 [04:53](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=293)**  
    测试时若不想保留本地数据，应使用无痕模式打开页面。
- **Devtools 插件在无痕模式下的启用 [05:07](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=307)**  
    安装 Vue Devtools 或 React Devtools 后需进行以下三项设置：
    - 勾选 **“在无痕模式下启用”**。
    - 勾选 **“允许访问文件网址”**。
    - 设置权限为 **“在所有网站上”**。

- **安装方式二：项目依赖安装 [06:13](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=373)**

    - 项目中需配置仅在开发环境（`process.env.NODE_ENV === 'development'`）下加载调试脚本。
    - 客户端通过指定端口连接服务端，实现远程调试。
- **跨设备调试能力 [07:28](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=448)**
