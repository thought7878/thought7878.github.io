### **第一款插件：TypeLine 的介绍与使用 [00:29](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=29)**

- **产品背景**  
    TypeLine 是较早推出的商业化代码提示插件，提供免费和付费版本。
- **功能差异**
    - 免费版仅支持单行代码提示。
    - 付费版（Pro）支持整块代码提示，价格约为15美元/月。
- **技术原理**  
    基于大量开源代码语料库训练，并结合本地编码数据进行个性化模型训练，实现个性化代码推荐。
- **安装与实践演示 [01:52](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=112)**
    - 在 VS Code 插件市场搜索 "TypeLine" 并安装。
    - 初始化完成后创建 `index.js` 文件进行测试。
    - 输入注释“邮箱的正则”，TypeLine 提示单行正则表达式。
    - 按 Tab 键插入代码，随后可用 `console.log` 测试邮箱匹配。
    - 使用 Node.js 执行文件验证结果。
- **使用建议**  
    免费版已能满足大多数用户需求；重度使用者可试用后决定是否购买年费版本（相对便宜）。
- **禁用插件**  
    演示结束后将 TypeLine 禁用以便切换至其他工具 [03:17](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=197)。

### **第二款插件：GitHub Copilot 的介绍与使用 [03:20](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=200)**

- **产品背景**  
    由 GitHub 推出，官方称为“AI结对编程助手”，名称寓意“领航员”。
- **品牌战略关联**  
    名称与微软后续推出的 Office + ChatGPT 组合（如 Pilot）有相似野心，体现其统治编程与办公领域的意图。
- **支持语言范围**  
    支持前端语言、Python、Ruby 等多种编程语言。
- **核心技术**  
    背后引擎为 GPT-3.5，经大规模代码语料训练，具备一定推理能力，效果惊艳但偶有错误。
- **定价信息**
    - 不提供完全免费版本。
    - 个人订阅价为10美元/月或100美元/年（约600元人民币）。
    - 学生可通过申请 GitHub Education Pack 免费使用并持续延期。
- **VS Code 实践演示 [05:18](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=318)**
    - 安装 Copilot 插件并重新加载。
    - 输入“手机号的正则”注释，回车后自动提示完整正则表达式。
    - 激活状态通过底部“领航员”图标确认。
    - 编写函数需求注释：“遍历数组ARR中的所有元素，若包含传入参数OBJ（string类型），则返回该元素。”
    - 回车后 Copilot 开始生成代码提示。
    - 初始提示存在问题：误将参数 `obj` 当作字符串字面量处理。
    - 修改注释为更清晰结构：“筛选函数 findStr，参数 arr 数组，item 为 string，返回数组中符合条件的元素。”
    - 新提示仍存在形参冲突问题，未能正确过滤 item。
    - 手动修正为正确逻辑：使用 `.filter(item => item.includes(obj))`。
- **总结评价**  
    Copilot 可作为参考，但不能完全依赖，最终代码需开发者自主判断与修改 [08:25](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=505)。

### **第三款插件：CodeGeeX 的介绍与使用 [08:38](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=518)**

- **产品定位**  
    开发者认为其非常“良心”，是近期使用频率最高的插件，课程中新增代码即由此工具生成。
- **功能定位**  
    功能与 TypeLine、Copilot 相同，均为智能代码提示。
- **商业策略**
    - 提供面向企业的收费版本。
    - 支持私有化部署，适用于对数据安全敏感的企业（如银行金融系统）。
    - 区别于前两者云端服务模式，适合无网络或封闭环境下的编程场景。
- **适用人群建议**  
    绝大多数个人用户使用免费版即可满足需求 [09:32](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=572)。
- **安装与实践演示 [09:39](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=579)**
    - 在 VS Code 插件市场搜索 “CodeGeeX” 并启用。
    - 需重新加载 Copilot 插件以避免冲突。
    - 首次使用需注册账号并登录。
    - 进行测试：
        - 输入“手机正则”注释，成功提示对应正则表达式。
        - 接着输入“username的正则，校验长度6到16位，仅含字母数字，不区分大小写”，正确生成 `$^[a-zA-Z0-9]{6,16}$^`。
        - 编写函数需求：“给定整数N，生成0到N的整数数组。”
        - CodeGeeX 正确提示代码：定义空数组，循环推入值，最后返回。
    - 再次测试复杂函数逻辑：
        - 粘贴原注释“遍历ARR，若元素包含OBJ（string），返回该元素。”
        - 参数命名混乱导致提示错误：`function item, item === item`。
        - 调整注释表述后，提示恢复正常逻辑。
- **性能对比结论**  
    CodeGeeX 的推理能力略强于 Copilot，但在模糊描述下依然可能出现误解。

### **总结与建议 [12:01](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=721)**

- **三款工具对比总结**  
    三款工具均能有效提升日常编码效率，满足基本使用需求。
- **核心提醒**  
    不应完全依赖 AI 工具生成代码，开发者必须保持独立思考与代码主导权。
- **工具定位重申**  
    所有 AI 辅助工具的本质是服务于编程工作，而非取代程序员。
