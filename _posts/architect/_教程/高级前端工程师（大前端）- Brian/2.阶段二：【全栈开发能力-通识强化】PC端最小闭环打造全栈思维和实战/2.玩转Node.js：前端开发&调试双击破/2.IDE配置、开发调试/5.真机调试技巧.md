### 两类原生调试方法共性总结 
[09:46](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=586)

- **三步统一流程** [09:54](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=594)
    - **第一步：开启手机调试模式** — iOS开Safari Web检查器，Android开USB调试；
    - **第二步：启用宿主端调试入口** — macOS开Safari开发菜单，Windows/macOS访问`chrome://inspect`；
    - **第三步：建立网络与物理连接** — USB直连 + 同局域网 + IP:端口访问服务。

### 代理劫持类调试（Charles）
[11:03](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=663)

- **适用场景** [11:07](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=667)  
    线上WAP页嵌入APP后无响应、接口加载卡顿等无法使用原生DevTools的场景。
- **Charles基础配置** [12:06](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=726)  
    启动Charles → Proxy → Proxy Settings → HTTP Proxy端口设为`8888`（默认），勾选“Enable transparent HTTP proxying”。
- **手机代理设置** [12:49](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=769)  
    iPhone Wi-Fi设置 → 当前网络 → HTTP代理 → 手动 → 服务器填本机IP（`192.168.31.109`），端口填`8888`。
- **HTTPS抓包配置** [13:27](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=807)  
    手机访问`chls.pro/ssl`下载并安装Charles根证书；设置 → 通用 → 关于本机 → 证书信任设置 → 启用Charles Proxy CA。
- **请求拦截与分析** [13:49](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=829)  
    Charles主界面实时捕获所有HTTP/HTTPS请求；可按域名（如`pascaljs.org`）、路径、状态码过滤；双击请求可查看Request Headers、Query String Parameters、Response Body等完整数据。
- **能力边界说明** [14:46](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=886)  
    仅支持接口层调试（请求/响应内容），不支持DOM操作、JS断点、性能分析等前端交互级调试功能。
