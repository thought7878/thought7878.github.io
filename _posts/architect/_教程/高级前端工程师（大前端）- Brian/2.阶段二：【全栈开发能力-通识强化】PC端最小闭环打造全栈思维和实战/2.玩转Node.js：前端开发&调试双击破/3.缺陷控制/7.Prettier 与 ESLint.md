[00:46](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=46)

- **什么是Prettier？ [00:59](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=59)**
    - Prettier一词意为“变美了，变漂亮了”。
    - 官方定义：**Opinionated code formatter**——即“有主见的代码格式化工具”。
    - 支持多种语言，与大多数编辑器集成。
- **使用Prettier的原因 [01:31](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=91)**
    - 代码在保存时自动格式化（`formatted on save`）。
    - 无需在代码审查中讨论代码风格（`No need to discuss style in code review`）。
    - 节省时间与精力（`Saves your time and energy`）。

### Prettier关键配置项详解 
[03:33](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=213)

- **`printWidth`（打印宽度） [03:46](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=226)**
    - 默认值为 `80`，表示每行最多容纳80个字符；超长则自动换行。
    - 官方建议上限为 `100–120`；VS Code插件默认推荐 `140`。
    - 该选项直接影响代码可读性，尤其在嵌套三元运算符、逻辑运算符等长表达式中至关重要。
- **其他重要配置 [06:33](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=393)**
    - `tabWidth`: 默认为 `2`。
    - `semi`: 是否保留分号；主讲人倾向设为 `false`（不加分号）。
    - `singleQuote`: 是否使用单引号；主讲人倾向设为 `true`。
    - `trailingComma`: 尾随逗号；一般设为 `"none"`（无尾随逗号）。

## ESLint
### ESLint核心定位与功能 
[07:44](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=464)

- **官方定义 [08:37](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=517)**
    - ESLint 是一个**可配置的JavaScript/TypeScript强制性linter**（代码检查工具）。
    - “Linter”原意为“去除线头上的基线”，在前端语境中指**静态分析并发现/修复代码质量问题**。
- **核心概念 [09:01](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=541)**
    - **Rules（规则）**: 强制执行的编码规范；违反即报错或警告。
    - **Plugins（插件）**: 扩展规则集的NPM模块（如 `eslint-plugin-vue`）。
    - **Processors（处理器）**: 解析非JS文件（如 `.vue`中的 `<template>` `<script>` `<style>`）并提取JS内容供ESLint检查。
    - **Formatters（格式化器）**: ESLint自身支持将部分规则转化为格式化能力（如自动删除分号）。

### ESLint与Prettier协同策略 
[11:08](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=668)

- **冲突根源 [11:08](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=668)**
    - Prettier负责风格（如分号、引号、换行），ESLint也含风格类规则（如 `semi`, `quotes`），二者易冲突。
- **协同方案 [11:19](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=679)**
    - 将ESLint配置为仅做**质量检查**，格式化完全交由Prettier。
    - 使用官方插件 `eslint-config-prettier`，其作用是**禁用所有与Prettier重叠的ESLint格式化规则**。
    - 关键配置项：`"extends": ["eslint:recommended", "plugin:vue/vue3-essential", "prettier"]`。
    - 原因：避免ESLint在lint过程中调用Prettier导致性能下降（[13:26](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=806)）。

### ESLint深度配置解析 
[13:03](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=783)

- **`extends`机制 [13:10](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=790)**
    - 通过 `extends` 字段继承预设规则集（如 `eslint:recommended`, `plugin:vue/vue3-essential`）。
- **`plugins`与`processor`配置 [17:22](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=1042)**
    - `plugins: ["vue"]` 启用 `eslint-plugin-vue`。
    - `processor: "vue/.vue"` 使ESLint能解析 `.vue` 文件各区块。
- **`env`与`parserOptions` [17:48](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=1068)**
    - `env`: 指定运行环境变量（如 `browser: true`, `es6: true`）。
    - `parserOptions`:
        - `sourceType: "module"`：支持ES模块语法。
        - `ecmaVersion: 2020`：启用动态 `import()`、`Promise.allSettled` 等ES2020特性。
        - 注：`ecmaVersion: "latest"` 可启用最新ECMAScript特性（如 `replaceAll`, `WeakRef`）。

### ESLint最佳实践建议 
[19:47](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=1187)

- **团队级实践 [19:50](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=1190)**
    - 优先采用框架官方ESLint插件（如 `eslint-plugin-vue`, `@typescript-eslint/eslint-plugin`）。
    - 不建议过度定制规则，除非大型团队有强代码规范诉求。
- **项目级选择 [20:20](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=9798c7c737b94409aa651a40d1a20896#?seek_t=1220)**
    - 小型项目或个人开发：可仅用Prettier，兼顾效率与一致性。
    - 大型项目：必须结合ESLint（质量）+ Prettier（风格），并通过 `eslint-config-prettier` 解耦。
