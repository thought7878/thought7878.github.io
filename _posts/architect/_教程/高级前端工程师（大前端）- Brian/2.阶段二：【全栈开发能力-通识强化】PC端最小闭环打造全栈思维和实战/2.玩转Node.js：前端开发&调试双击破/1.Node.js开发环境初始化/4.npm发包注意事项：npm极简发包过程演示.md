- **前置准备工作 [00:31](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=31)**
    
    - 安装Node.js与NPM（若未安装）。
    - 项目中需存在`package.json`文件，可通过`npm init`命令初始化。
    - 注册NPM账号（npmjs.com官网右上角“Sign Up”），**必须完成邮箱校验**，否则无法发包。

    - 登录后进入个人控制台界面。
    - 右侧显示高频使用记录；右上角点击“Packages”可查看当前管理与发布的所有包。
- **`package.json`核心字段规范 [01:28](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=88)**
    
    - **`name`** [01:36](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=96)：包名须全局唯一，仅允许小写英文字母、短横线（`-`），禁止大写、驼峰、特殊字符（除`@`）；`@scope/name`格式中`@`前为组织名，后为包名。
    - **`version`** [02:29](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=149)：须遵循语义化版本规范（如`1.0.0`）。
    - **`main`** [02:34](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=154)：指定包入口文件，通常为`index.js`、`src/index.js`或`src/index.ts`。
    - **`scripts`** [02:44](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=164)：定义可执行脚本，如`dev`、`build`、`test`等。
    - **`dependencies`与`devDependencies`** [02:55](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=175)：分别声明运行时依赖与开发依赖。
- **`npm init`交互式初始化实操 [03:08](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=188)**
    
    - 执行`npm init`启动向导。
    - 依次输入：包名（例：`sale-cli`，小写+短横线）、版本（例：`1.0.0`）、描述（`useful CLI tool`）、入口文件（`index.js`）、测试命令（留空）、Git仓库地址（`https://github.com/xxx/sale-cli`）、关键词、作者邮箱、许可证（`MIT`）。
    - 完成后生成标准`package.json`文件，需人工核对字段正确性。
- **配套文件编写 [04:43](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=283)**
    
    - 创建`README.md`文件，说明包功能（例：包名`sale-cli`标注`WIP`表示开发中）。
    - 创建入口文件`index.js`（因初始`package.json`中`main`指向该文件）。
    - 可选：在`scripts`中添加`"start": "node index.js"`等脚本。
- **`.npmignore`文件配置 [05:41](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=341)**
    
    - 创建`.npmignore`排除非必要文件，避免发布冗余内容。
    - 常见忽略项：`node_modules/`、`*.log`、`.DS_Store`、`.git`、`.gitignore`等（内容与`.gitignore`高度相似）。
- **发布前环境校准 [06:26](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=386)**
    
    - 执行`npm config get registry`检查当前registry源。
    - 若为国内镜像源（如淘宝、华为、腾讯），须强制切回官方源：  
        `$ npm config set registry https://registry.npmjs.org/`
    - 执行`npm login`登录；启用双重验证时，按提示访问官方链接完成验证。
- **包发布与验证 [07:32](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=452)**
    
    - 在项目根目录执行`npm publish`。
    - 成功标志：终端输出`+ package-name@1.0.0`。
    - 发布后搜索可能延迟（网页端暂不可见），可通过`npm view package-name`命令即时验证包元数据是否存在。
    - Stack Overflow确认该延迟属已知现象。
- **错误包撤回机制 [08:36](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=516)**
    
    - 发布后24小时内可撤回：`npm unpublish package-name@1.0.0 -f`（`-f`为强制参数）。
    - 撤回失败常见原因：版本号不存在或已超24小时窗口期。
    - 多版本需逐条执行撤回命令（例：`npm unpublish sale-cli@1.0.0 -f`）。
