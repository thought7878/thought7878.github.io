### 语义化版本号的规范结构 
[01:31](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=91)

- **基本格式定义** [01:31](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=91)  
    版本号格式为 `$MAJOR$.$MINOR$.$PATCH$`，三段式数字由英文点号分隔。
- **主版本号（MAJOR）含义** [01:49](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=109)  
    表示*不兼容的 API 修改*；版本号递增即*宣告向下不兼容，用户升级需高度谨慎*。
- **次版本号（MINOR）含义** [02:02](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=122)  
    表示*向下兼容的功能性新增*；虽兼容，但可能引入新 bug。
- **修订号（PATCH）含义** [02:17](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=137)  
    表示*向下兼容的问题修复（如 bug 修正），无功能变更*。
- **元数据与先行版本扩展** [01:39](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=99)  
    可在 `$PATCH$` 后附加 `-` 分隔的先行版本标识（如 `-alpha.1`）及 `+` 分隔的构建元数据（如 `+20230101`）。

### 先行版本（Pre-release）类型与开发阶段映射 
[03:37](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=217)

**针对不同的开发阶段，给包起的名字**

- **常见先行版本标识** [03:37](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=217)

    内部测试版，极不稳定，面向开发与测试人员，含大量未修复 bug，慎用。
- **Beta 版本特性** [04:13](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=253)

    正式发布版，无先行标识，格式为纯三位数字（如 `1.0.0`、`2.0.0`、`3.0.0`）。

### NPM version 命令
[05:48](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=348)

- **命令基础用法** [05:48](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=348)  
    `npm version` 是 *NPM 提供的版本管理命令*，支持*自动更新 package.json 中的 `version` 字段并触发 Git 提交操作*。
- **核心子命令** [06:07](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=367)
    - `npm version major`：主版本号 +1，次/修订号归零（如 `1.2.3` → `2.0.0`）；
    - `npm version minor`：次版本号 +1，修订号归零（如 `1.2.3` → `1.3.0`）；
    - `npm version patch`：修订号 +1（如 `1.2.3` → `1.2.4`）。
- **先行版本操作** [09:35](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=575)  
    `--preid` 参数用于指定先行版本标识（如 `alpha`、`beta`、`rc`），须与 `prerelease` 子命令配合使用。
- **预发布流程实操** [10:12](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=612)
    - 初始执行 `npm version prerelease --preid=alpha` → 生成 `1.0.2-alpha.0`；
    - 后续执行 `npm version prerelease` → 自动递增末位号（`1.0.2-alpha.1` → `1.0.2-alpha.2`）；
    - 切换类型：`npm version prerelease --preid=beta` → `1.0.2-beta.0`（末位重置为 0）；
    - 进入 RC：`npm version prerelease --preid=rc` → `1.0.2-rc.0`；
    - 正式发布：`npm version patch` → `1.0.3`（跳过所有预发布序列，仅基于上一个正式版递增）。

### Git 联动机制与选项控制

### 最佳实践与工具延伸 
[14:12](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=852)

- **先行版本使用策略** [14:15](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=855)  
    推荐仅在大版本迭代（如 `2.0.0`）时启用先行版本流程（`2.0.0-alpha.0` → `2.0.0-rc.10` → `2.0.0`）；小版本（minor/patch）直接使用对应子命令，不引入预发布标识。
- **替代工具生态** [13:25](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=845c61eab75a4b2b8c04cea32a174e1f#?seek_t=805)  
    实际项目中常采用更高级工具替代原生命令，如：
    - `lerna`：用于多包（monorepo）版本管理；
    - 提交规范工具（如 `commitizen`）：结合 `conventional-changelog` 自动推导版本号。
