## 引言：开发工具的重要性 
[00:02](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=2)
    
- 强调代码编辑器是开发者最常用的工具。
- 对比 WebStorm 与 VS Code：WebStorm 为集成开发环境（IDE），功能齐全；而 VS Code 是轻量级编辑器，需手动配置插件。

## 账号登录与设置同步 
[00:27](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=27)
    
- 在 VS Code 左下角点击“Sign in to Sync Settings”进行登录。
- 支持使用 GitHub 或 Microsoft 账号登录。
- 登录后可同步插件、快捷键、个人配置等信息至云端。
- 同步功能确保在不同设备上使用同一配置。
- 登录后进入齿轮菜单 → Profiles → 启用“Settings Sync is on”以开启同步。
- 若未启用，手动勾选即可激活同步选项。

## 安装中文语言包 
[01:53](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=113)
    
- 点击左侧 Extensions 图标。
- 搜索 “chinese”。
- 安装第一个结果“Chinese (Simplified) Language Pack for Visual Studio Code”。
- 安装完成后右下角提示“Change Language and Restart”，重启后界面变为中文。

## 创建配置文件以管理插件 
[02:22](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=142)
    
- 建议创建独立的配置文件用于不同开发场景（如前端、Python）。
- 示例：创建名为 `front end` 的配置文件。
- 配置文件作用：便于在多语言项目中快速切换所需插件集合，避免加载无关插件，提高启动速度。

## 选择插件的原则 
[03:06](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=186)
    
- 插件选择方法一：按开发框架搜索对应插件包（如 React、Vue）。
- 插件选择方法二：访问 VS Code 官网市场（visualstudio.com/vscode/extensions）浏览特性推荐。
	- 如 “Snap” 插件可用于美观地截取代码片段用于 PPT 或 Markdown。
	- 可查看 “Trending This Week/Month”、“Most Popular” 来筛选高人气插件。
- 插件选择原则：
	- 下载量越高通常代表越稳定、越受欢迎。
	- 使用插件包（Extension Pack）可一次性安装一组相关插件。

## 前端常用插件安装 
[06:15](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=375)
    
- 安装 `Vue Extension Pack` 和 `React Extension Pack`。
- 注意重复插件不会重复安装（如 ESLint 相关工具）。
- 安装美化类插件：
	- `Material Icon Theme`：为不同文件类型显示图标，便于资源管理器识别。
	- 主题插件：推荐使用 `Dracula Official`（深黑色主题）。
	- 可添加 `Webstorm` 主题使 VS Code 与 WebStorm 外观一致。

## Node.js 相关插件 
[08:59](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=539)
    
- 搜索并安装 `Node.js Extension Pack`。
- 包含智能提示、语法支持等功能。
- 安装 `Node Exec`：在 VS Code 中直接执行 Node 命令。
- 搜索 `NPM`，确认已安装 `NPM IntelliSense`（提供自动补全）。

## Git 版本控制插件 
[09:53](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=593)
    
- VS Code 自带基础 Git 功能（左侧“源代码管理”图标）。
- 推荐安装增强插件：
	- `GitLens` 或 `Git Extension Pack`。
	- 提供更详细的提交历史、行内作者标注等功能。
- 不推荐使用 `gitignore` 自动生成插件（因国内网络问题常无法获取模板）。
- 推荐手动创建 `.gitignore` 文件或使用在线生成器。

## TypeScript 相关插件 
[11:21](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=681)
    
- 安装 `TypeScript Extension Pack`。
- 具体包含：
	- `Move TS`：重构工具，支持移动 TypeScript 文件。
	- `TypeScript Importer`：自动补全模块导入路径。
	- `JSON to TS`：将 JSON 数据结构转换为 TypeScript 接口。

## 配置文件切换的实际应用 
[12:38](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=0fbbc2a0fb66460f9f46d04db1f9bd61#?seek_t=758)
    
- 创建新的配置文件（如命名为 `python`）。
- 切换后原前端插件全部隐藏，仅加载当前配置下的插件。
- 再次切换回 `front end` 配置，之前插件自动恢复。
- 优势：
	- 减少运行时内存占用。
	- 提升编辑器启动速度。
	- 实现按项目类型精准加载插件。
