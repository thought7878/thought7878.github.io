- **关键概念总览 [06:58](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=418)**  
	包含TypeScript、ES Modules、Server Engine、Rendering Models、Nuxt DevTools、Auto Imports等。
- **Auto Imports自动导入机制 [08:01](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=481)**
	- **内置自动导入（Built-in Auto Imports）** [08:01](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=481)：自动从服务端请求数据（如`useAsyncData`），绑定至应用上下文与运行时配置，支持状态管理（state manager）或自定义组件/插件。
	- **Vue API自动导入（Vue Auto Imports）** [08:52](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=532)：无需手动`import { ref, computed } from 'vue'`，Nuxt 3自动注入Vue核心API。
	- **目录结构驱动自动导入（Directory-based Auto Imports）** [09:49](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=589)：`components/`下组件、`composables/`下组合式API、`utils/`下工具函数等，按约定目录自动导入，与前端项目标准化实践高度一致。
- **Nuxt 3与Vue 3关系阐释 [10:56](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=656)**
	- **版本依赖** [11:02](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=662)：Nuxt 3整合Vue 3，学习Vue 3是使用Nuxt 3的前提。
	- **选用Vue 3原因** [11:13](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=673)：Vue 3响应式系统优越；组件系统保留HTML结构，兼顾简洁性与强大性；小型项目性能表现优异。
	- **Nuxt 3中Vue开发方式** [11:53](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=713)：仍用`.vue`文件开发；新增自动导入功能；路由由`pages/`目录自动生成，无需手动配置Vue Router。
- **Nuxt 2与Nuxt 3差异说明 [12:47](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=767)**  
	Nuxt 2基于Vue 2；Nuxt 3基于Vue 3，主要提升包括：性能优化（内存占用降低、渲染提速、包体积减半）、Composition API全面支持、TypeScript深度集成。
- ***Rendering Models渲染模型*** [13:38](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=818)
	- **Universal Rendering（通用渲染）** [13:59](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=839)：服务端（SSR）与客户端（CSR）协同渲染。*服务端生成完整HTML并执行Nuxt代码，客户端加载响应式JavaScript完成hydration（水合），实现首屏快速加载与交互响应兼得*。*优势*为性能更优、利于SEO；劣势为开发复杂度高、服务器负载大。
	- **Client-Side Rendering（客户端渲染）** [16:56](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=1016)：开箱即用的*传统SPA模式，先下载JS再渲染页面*；可借助Nuxt内置Auto Imports简化开发。
	- **Hybrid Rendering（混合渲染）** [17:48](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=1068)：按需配置路由规则（route rules），*指定部分页面预渲染（如首页）、部分页面客户端渲染（如Admin后台），兼顾性能与灵活性*。
	- **Edge-Side Rendering（边缘渲染）** [18:55](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=1135)：*依托CDN网络，在边缘节点执行渲染，缩短用户物理访问距离*；通过Nitro引擎实现，构建产物（`.output/`目录）包含跨平台运行时，脱离Node.js环境限制。
- **Server Engine服务端引擎 [19:26](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=1166)**  
	Nitro引擎是Nuxt 3边缘渲染能力的基础，构建时生成可在任意环境运行的分发包，提供跨平台能力与丰富API扩展层。
- **Modules模块生态 [20:32](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=1232)**  
	Nuxt 3提供官方模块市场（https://modules.nuxtjs.org），涵盖DevTools（必装）、图片优化、内容层（读取`content/`下Markdown并转换）等，构建开放插件生态，区别于Vue/React相对封闭的基础框架生态。
- **TypeScript与ES Modules支持 [23:24](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=1404)**  
	Nuxt 3全面支持ES2022+与TypeScript，提供类型推导增强、自动生成`.d.ts`类型声明文件能力，配置通过`tsconfig.json`扩展实现。
