    - **Store定义 [37–40]**  
        创建 `stores/useCounter.ts`：
        
        ts
        
        ```ts
        export const useCounter = defineStore('counter', () => {
          const count = ref(0)
          const increment = () => count.value++
          return { count, increment }
        })
        ```
        
    - **路径别名配置 [41–49]**
        - Next.js 中路径别名由 `next.config.ts` 的 `typescript` 配置项 `baseUrl` 和 `paths` 控制；
        - `tsconfig.json` 中需同步配置 `compilerOptions.paths`，如 `"~/*": ["./src/*"]`；
        - Vue 项目习惯的 `@/` 别名在 Next.js 中默认映射为 `~/`（波浪号），为兼容性保留此约定。
    - **页面中使用Store [55–59]**  
        在 `app/page.tsx`（或 `app/page.vue`）中：
        
        ts
        
        ```ts
        import { useCounter } from '~/stores/useCounter'
        const store = useCounter()
        // 模板中：{{ store.count }} <button @click="store.increment">+1</button>
        ```
        
    - **自动挂载机制 [63–65]**

- **路径规范与构建排错 [146–156]**
    
    - **构建错误原因 [145–146]**  
        项目路径含中文字符或空格（如 `C:\用户\文档\next-project`）导致 `pnpm build` 报 `ENOENT` 错误。
    - **解决方案 [147–155]**  
        将项目移至纯英文无空格路径（如 `D:\next-project`），重启 VS Code 及终端后重新执行 `pnpm build`，构建成功输出至 `.output` 目录。

    - **Composables自动加载 [180–184]**  
        创建 `composables/` 目录，其下 `.ts` 文件中的函数（如 `useUser()`）将被自动导入，无需 `import`。
