- **问题现象：PWA未成功注册 [00:01](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=1)**
    
    - 访问 `localhost:3000` 时右上角无下载按钮，Application面板可见 workbox 但 service worker 未注册。
    - 在 `index.vue` 中通过 `pwa` 对象打印 `isInstalled`、`offlineReady`、`showInstallPrompt`、`swActivated` 等属性，全部返回 `false`；`getSWRegistration()` 返回 `undefined`，`install()` 未执行。
- **根因定位：SSR上下文缺失与配置不全 [01:49](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=109)**
    
    - 官方文档仅提供 API 列表（promptOptions、events），无完整示例；需结合源码与 sample 项目分析。
    - 源码显示 `wait-pwa` 模块依赖 `periodicSyncForUpdates` 配置项触发注册逻辑，仅设 `immediate: true` 不足以保证注册成功。
- **核心修复：配置文件 `next.config.ts` 补充 [03:15](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=195)**
    
    - 添加 `workbox` 配置块，扩展缓存资源后缀：`['css', 'js', 'html', 'png', 'jpg']`。
    - 设置 `periodicSyncForUpdates: 10`（测试用，10秒轮询），替代 `index.vue` 中手动 `setInterval`；生产环境建议设为 `3600`（1小时）。
    - 此配置驱动 `swActivated` 状态更新，否则该值恒为 `false`，导致状态判断失效。
- **组件层补全：注入 manifest [05:18](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=318)**
    
    - 在根组件 `App.vue` 中添加 `<VitePWAManifest />` 组件（默认导出），确保 PWA 元数据正确注入。
- **环境清理与验证流程 [05:31](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=331)**
    
    - 清除旧 service worker：在 Application → Service Workers → `Unregister` 所有已注册项。
    - 刷新页面后验证：`offlineReady` 变为 `true`，`swActivated` 变为 `true`，证明 service worker 注册成功。
    - 启用离线模式后页面可交互（仅接口数据缺失，静态资源可用），确认离线能力生效。
- **缓存策略增强：全局通配符配置 [06:34](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=394)**
    
    - 将 `globPatterns` 设为 `['**/*']` 实现全资源缓存（含 `.json`、`.jpg` 等），避免手动枚举后缀。
    - 注意事项：大体积资源（如 `.woff`）需按需缓存；过大的 bundle 会导致缓存耗时显著增加。
- **生产环境验证：`pnpm build` + `pnpm preview`[07:55](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=475)**
    
    - 构建前注释 `mark.js` 中 development 相关代码以确保数据可访问。
    - 使用无痕模式调试：禁用 `offline` 开关后观察 Workbox 缓存详情，确认 `build/` 下所有静态资源（含 `assets/`、`downloads/`）均被完整缓存。
    - 离线状态下可正常加载页面结构与缓存资源，仅动态接口数据不可用。
- **首次加载错误说明：SSR路径匹配问题 [09:05](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=545)**
    
    - 错误源于服务端渲染时，`cache-first` 策略尝试从缓存读取 `/index.html`，但初始缓存为空，导致 fallback 失败。
    - Application → Cache Storage 中确无 `index.html` 条目，属预期行为，可安全忽略。
- **代码清理：移除冗余逻辑 [09:51](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=591)**
    
    - 删除 `index.vue` 中全部 PWA 相关代码（`usePWA`调用、`setInterval`、`install()` 调用等）。
    - `pwa` 对象声明及状态监听逻辑一并移除，统一由 `next.config.ts` 配置驱动。
