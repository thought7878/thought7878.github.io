        - 通过`$nuxtApp.$pwa.getSWRegistration()`获取`ServiceWorkerRegistration`；
        - 若存在registration，则调用`update()`（3600秒周期）；否则执行install并赋值`$nuxtApp.$pwa.registration = registration`。

- **API服务端接口规范 [12:19](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=739)**
    - 禁止在项目根目录创建与`server/api/`同名的API文件（如`api/hello.ts`），避免Next.js自动导入冲突；
    - 正确做法：在`server/api/home.get.ts`中导出`defineEventHandler`，返回`{ code: 200, data: { swipingProjects, projects, causes } }`；
    - 前端调用改用`useFetch('/api/home')`或`$fetch('/api/home')`，响应体为`{ data: ref, pending: ref, error: ref, status: ref }`。
- **SSR兼容性问题修复：document未定义 [17:41](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=1061)**
    - 错误定位：`app/layouts/default.vue`第45行`document.body`在服务端渲染时不可用；
    - 方案一（条件判断）：包裹`if (process.client) { ...document.body... }`；
    - 方案二（生命周期钩子）：将DOM操作移至`onBeforeMount(() => { ... })`，确保仅客户端执行。

    - SSR场景下禁用`onBeforeMount`发起API请求；改用`useAsyncData('home', () => $fetch('/api/home'))`，该Hook在服务端预取数据并在客户端复用。
