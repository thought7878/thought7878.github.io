### 容器启动命令详解 
[04:56](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=296)

- **完整运行命令** [05:05](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=305)  
    `$ docker run -d -p 8000:8000 -p 9443:9443 --name portainer --restart=always -v /var/run/docker.sock:/var/run/docker.sock -v portainer_data:/data portainer/portainer-ce:latest`
- **参数逐项解析** [05:10](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=310)–[06:00](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=360)
    - `-d`: 后台守护进程模式运行；
    - `-p 8000:8000 -p 9443:9443`: 端口映射（宿主机:容器），放行宿主机端口；
    - `--name portainer`: 指定容器名称；
    - `--restart=always`: Docker daemon重启时自动启动容器；
    - `-v /var/run/docker.sock:/var/run/docker.sock`: 挂载Docker守护进程套接字，赋予Portainer操作宿主机Docker权限；
    - `-v portainer_data:/data`: 挂载持久化Volume至容器`/data`目录；
    - `portainer/portainer-ce:latest`: 使用最新社区版镜像。
- **端口映射关键原则** [06:03](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=363)–[06:56](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=416)  
    冒号左侧为宿主机端口（需防火墙放行），右侧为容器内监听端口；`9443`与`8000`是Portainer服务在容器内监听的端口，不可混淆内外。
