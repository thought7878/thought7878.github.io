    - **学习入口推荐**  
        学习容器化技术最好的入门方式就是去安装docker。
    - **易用性说明**

    - **实验特性集成**  
        Docker Desktop已集成Kubernetes（KPS）及Swarm集群的便捷创建功能。
    - **实践目标明确**  
        安装环境旨在支撑后续开发：打包镜像、上传镜像、镜像管理等。
- **Docker核心价值与隔离能力 [00:37](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=37)**
    
    - **生产与测试赋能**  
        可在生产/测试环境中驱动应用部署与开发。
    - **应用为中心的设计哲学**  
        Docker是为应用而生的；应用在任意多核环境运行时，无需担忧基础环境问题。
    - **环境一致性保障**  
        镜像中封装基础环境，实现“一处书写，处处运行”，类比React口号。
    - **逻辑隔离能力**  
        提供类似虚拟机的逻辑隔离，覆盖文件系统层面与网络层面。

    - **Linux便捷安装方案 [04:03](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=243)**  
        提供自动化脚本方案，适用于虚拟机或云服务器环境，规避环境适配困扰。
- **Linux命令行安装实操 [04:32](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=272)**
    
    - **官方脚本来源**  
        访问GitHub `get.docker.com` 仓库，获取 `install.sh` 官方安装脚本。
    - **执行命令**  
        `$ curl -fsSL https://get.docker.com -o get-docker.sh` → `$ sh get-docker.sh`。
    - **国内访问替代方案 [05:36](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=336)**  
        若无法访问 `get.docker.com`，可从GitHub仓库点击 `Raw` 下载 `install.sh` 脚本，保存至本地资源文件夹或Q群共享。
    - **权限配置**  
        上传脚本至服务器后，执行 `$ chmod +x get-docker.sh` 授予执行权限。
    - **版本验证**  
        安装完成后执行 `$ docker --version`，示例输出：`Docker version 24.0.2`。
    - **服务管理**
        - 设置开机自启：`$ sudo systemctl enable docker`。
        - 查看运行状态：`$ sudo systemctl status docker`（显示 `running` 即正常）。
        - Synaptic Linux用户同理使用 `systemctl` 命令。
- **Docker基础命令体系 [07:15](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=435)**
    
    - **帮助与信息查询**  
        `$ docker help` 展示全部命令；`$ docker --version` 查版本；`$ docker info` 查系统信息。
    - **核心操作命令**
        - `$ docker run`：基于镜像启动容器（如先前演示MySQL）。
        - `$ docker exec`：在运行中容器内执行命令（用于免安装命令行工具环境）。
        - `$ docker ps`：查看当前运行中的容器。
        - `$ docker build`：构建Docker镜像。
        - `$ docker pull`：从仓库拉取镜像（类比Git clone）。
        - `$ docker push`：向仓库推送镜像（类比Git push）。
        - `$ docker login` / `$ docker logout`：登录/登出Docker仓库。
        - `$ docker search`：搜索Docker Hub上的公开镜像。
        - `$ docker images`：列出本地所有镜像。
        - `$ docker logging`：日志相关命令（原文提及但未展开）。
    - **扩展命令提示**  
        `$ docker swarm` 用于集群管理；其余扩展命令按需引入。
- **Docker核心概念图谱 [08:51](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=531)**
    
    - **镜像（Image）[09:31](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=571)**
        - 是分层式只读文件系统，存储运行程序所需的基础环境、数据及可执行脚本。
        - 从Registry（仓库）下载，以文件形式本地存储。
    - **容器（Container）[08:56](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=536)**
        - 是运行起来的镜像实例；即镜像的“激活态”。
        - 运行时产生独立存储区域，用户所有操作变更均存于此缓存区。
        - 支持生命周期管理：`start` / `stop` / `rm`。
    - **仓库（Registry）[09:41](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=581)**
        - 默认为Docker Hub（`https://hub.docker.com`），是集中托管镜像的远程服务。
        - 国内加速源：阿里云、DaoCloud等。
    - **核心操作流 [10:28](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=628)**
        - `pull`：从Registry拉取镜像（无需登录）。
        - `push`：需先`login`认证，再推送本地构建/提交的镜像至Registry。
        - `run`：执行时自动检查本地镜像，缺失则触发`pull`，随后启动容器。
        - `build`：基于Dockerfile构建新镜像。
        - `commit`：将容器修改持久化为新镜像（类比Git commit）。
    - **共享机制对比 [13:57](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=837)**  
        Docker通过Registry实现镜像共享，突破传统虚拟机因硬件与软件版本差异导致的模板不可移植瓶颈。

    - **安装方法全覆盖**  
        Windows/macOS/Linux Desktop安装条件与Linux命令行一键脚本安装。
    - **核心概念三层结构**  
        Registry（仓库）→ Image（镜像）→ Container（容器），强调其依赖与转化关系。
