### 测试覆盖率配置与执行 
[05:00](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=300)

- **coverage配置项引入**
    需在`vite.config.ts`中`test`对象内添加`coverage`配置：`provider`指定覆盖率引擎（`c8`/`v8`/`istanbul`）；`reporter`指定报告格式（`text`/`html`/`json`）；`src`指定被测源码目录（仅`c8`支持）。
- **C8引擎特性与局限** 
    `src`属性仅在`provider: 'c8'`时生效；但C8将在下一版Vitest中被V8替代。
- **覆盖率依赖自动安装**
    首次执行`npm run test:coverage`时，若缺失C8依赖，会提示并自动安装。
- **零覆盖率原因分析**
    初始测试文件未引用任何被测组件（如`card.vue`），导致无代码执行，覆盖率全为0。
- **覆盖率指标解读** 
    `stmts`为语句覆盖率（覆盖`if`/`else`/`switch`等语句）、`branches`为分支覆盖率、`functions`为函数覆盖率、`lines`为行覆盖率（如50行代码执行25行为50%）；`uncovered lines`列出未覆盖行号（如57、60–62、65、70–77、83等）。
- **未覆盖行归因**
    因测试中未传入`icon`、`imageType`、`border`、`imageClass`等props，导致对应`if`判断分支未执行。

### 覆盖率超时与进程残留问题 
[09:13](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=553)

- **timeout配置** 
    `timeout`限制覆盖率执行时长，默认值可能过短，可配置`test: { coverage: { lines: 80, ... }, timeout: 20000 }`（单位毫秒）延长至20秒或120秒。
- **进程残留警告**
    官方GitHub存在open issue，`vitest`测试后`vite`server未能正常退出；若不影响覆盖率数据生成，建议保持默认`timeout`，避免与CI/CD自动化工具的超时设置冲突。

### Vitest UI可视化工具 
[11:35](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=bb3fc3ec569540d3b6008cdc98e0ef75#?seek_t=695)

- **UI工具定位与功能**
    Vitest UI是图形化测试工具，可展示单元测试结果、模块依赖图、源码及覆盖率报告。
- **安装与脚本配置** 
    安装`vitest-ui`依赖，在`package.json`中添加`"test:ui": "vitest --ui"`脚本。
- **UI核心功能**
    启动后可查看测试用例执行状态；右侧`Report Module Graph`展示模块引用关系；`Code`面板显示源码；点击失败测试用例右上角跳转按钮可直接定位至对应测试文件。
- **Coverage按钮缺失问题溯源**
    Coverage按钮受`coverageVisible`变量控制，该变量依赖`coverageEnabled`，而`coverageEnabled`需同时满足：① `coverage`配置对象存在且`enabled: true`；② `reporter`数组包含`'html'`。
- **Coverage按钮启用配置** 
    在`vite.config.ts`的`test.coverage`中显式设置`enabled: true`，并确保`reporter`包含`'html'`（如`['text', 'html', 'json']`）。
- **UI Coverage页面验证**
    重启`test:ui`后左上角出现`Coverage`按钮；点击后加载本地HTTP Server服务；等效命令为`npx vitest preview --outDir coverage`，访问`http://localhost:4173`可查看相同HTML报告；若缓存旧PWA内容，需清空浏览器Application → Storage或使用无痕模式访问。
