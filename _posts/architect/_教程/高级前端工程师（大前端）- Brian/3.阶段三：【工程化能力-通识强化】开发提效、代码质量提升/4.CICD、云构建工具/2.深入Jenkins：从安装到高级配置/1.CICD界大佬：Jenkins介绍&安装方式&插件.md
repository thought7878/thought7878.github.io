### 直装 vs Docker：适用场景对比分析 
[01:53](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=1cd97e72b13147e4aca8217600c5deb1#?seek_t=113)

- **直装适用场景一：老旧系统兼容性** [02:03](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=1cd97e72b13147e4aca8217600c5deb1#?seek_t=123)[02:07](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=1cd97e72b13147e4aca8217600c5deb1#?seek_t=127)  
    部分旧版 Windows 系统无法安装 Docker，此时直装是唯一可行方案。
- **直装适用场景二：资源受限环境** [02:11](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=1cd97e72b13147e4aca8217600c5deb1#?seek_t=131)[02:13](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=1cd97e72b13147e4aca8217600c5deb1#?seek_t=133)[02:21](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=1cd97e72b13147e4aca8217600c5deb1#?seek_t=141)  
    Docker 自身占用系统资源（CPU、内存），在低配机器上运行 Docker + Jenkins 可能造成资源浪费及 Jenkins 性能下降。
- **直装优势总结** [02:29](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=1cd97e72b13147e4aca8217600c5deb1#?seek_t=149)  
    一条命令即可启动 Jenkins，无需容器层开销。

### 直装方式实操流程 
[02:38](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=1cd97e72b13147e4aca8217600c5deb1#?seek_t=158)

- **安装包获取** [02:38](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=1cd97e72b13147e4aca8217600c5deb1#?seek_t=158)  
    点击文档中“下载 Jenkins”，获取 `jenkins.war` 文件（视频中误称“Jackson dot y”）。
- **启动命令** [02:44](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=1cd97e72b13147e4aca8217600c5deb1#?seek_t=164)  
    执行 `$ java -jar jenkins.war` 启动 Jenkins。
- **启动说明**
    该命令仅用于启动服务；完整配置需通过浏览器访问 `http://localhost:8080` 进入 Web 界面完成。
- **生产环境部署补充说明** 
    - 实际生产中不推荐直接用 `java -jar` 启动（仅用于测试）；
    - 推荐使用系统级包管理器（如 `apt-get install jenkins`）或 systemd 守护进程实现自启、日志管理与后台运行。

### 插件三种安装方式 
[06:13](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=1cd97e72b13147e4aca8217600c5deb1#?seek_t=373)

- **方式一：Web UI 在线安装** [06:19](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=1cd97e72b13147e4aca8217600c5deb1#?seek_t=379)  
    登录 Jenkins 后台 → “系统管理” → “插件管理” → “可用插件” 中搜索并勾选安装。
- **方式二：CLI 命令行安装** [06:24]  
    使用 Jenkins CLI 工具执行 `java -jar jenkins-cli.jar install-plugin <plugin-name>`。
- **方式三：离线上传安装** [06:28]  
    下载 `.hpi` 或 `.jpi` 插件包 → 后台“高级”选项卡 → “上传插件” → 选择文件上传。
- **离线安装典型应用场景** [06:39]
    适用于政企内网、金融加密环境等无法连接外网的封闭网络，保障 CICD 流程安全可控部署。
