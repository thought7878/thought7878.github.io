## 错误解决方案：直接映射宿主机docker二进制 
[03:06](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=1cd97e72b13147e4aca8217600c5deb1#?seek_t=186)

	停止并删除原Jenkins容器，使用`docker run --name jenkins -v /usr/bin/docker:/usr/bin/docker ...`命令启动新容器，将宿主机`/usr/bin/docker`映射进容器。
- **动态库版本冲突现象** [04:06](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=1cd97e72b13147e4aca8217600c5deb1#?seek_t=246)  
	容器内执行`docker --version`或`docker info`时提示动态库版本不匹配错误。
- **LDD命令诊断原理** [04:55](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=1cd97e72b13147e4aca8217600c5deb1#?seek_t=295)  
	在宿主机执行`ldd /usr/bin/docker`显示其依赖glibc 2.35；而Jenkins容器（基于Debian 11）仅提供glibc 2.31/2.32/2.34，导致不兼容。

### 版本冲突根源分析 
[06:32](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=1cd97e72b13147e4aca8217600c5deb1#?seek_t=392)
    
- **基础镜像差异论证** [06:49](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=1cd97e72b13147e4aca8217600c5deb1#?seek_t=409)  
	Jenkins镜像源自`openjdk:8-jre-slim`，其底层为Debian 11（通过`docker run --rm openjdk:8-jre-slim cat /etc/os-release`验证），glibc版本为2.31；宿主机Ubuntu 22.04 glibc为2.35。

## 官方推荐方案：Docker-in-Docker (DinD) 
[09:51](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=1cd97e72b13147e4aca8217600c5deb1#?seek_t=591)
    
- **方案合理性转折** [09:51](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=1cd97e72b13147e4aca8217600c5deb1#?seek_t=591)  
	尽管“Docker套Docker”看似违背最佳实践，但Jenkins官方英文文档明确推荐此方案解决容器内Docker调用问题。
- **docker:dind镜像功能** [10:46](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=1cd97e72b13147e4aca8217600c5deb1#?seek_t=646)  
	`docker:dind`镜像是专为Docker-in-Docker设计的官方镜像，提供逻辑隔离的独立Docker服务。
- **DinD解决版本冲突机制** [11:35](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=1cd97e72b13147e4aca8217600c5deb1#?seek_t=695)  
	在Jenkins容器内启动`docker:dind`容器，为其安装Docker CLI；CLI通过TCP协议（默认端口2376）与DinD容器的Docker Daemon通信，规避宿主机glibc依赖。

[11:57](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=1cd97e72b13147e4aca8217600c5deb1#?seek_t=717)
    
- **自定义Jenkins镜像流程** [11:57](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=1cd97e72b13147e4aca8217600c5deb1#?seek_t=717)

- **网络配置关键参数** [12:29](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=1cd97e72b13147e4aca8217600c5deb1#?seek_t=749)  
	启动DinD容器时使用`--network alias docker`参数，将其网络别名设为`docker`；Jenkins容器通过`DOCKER_HOST=tcp://docker:2376`环境变量指向该别名。
- **DOCKER_HOST环境变量作用机制** [13:15](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=1cd97e72b13147e4aca8217600c5deb1#?seek_t=795)  
	Docker CLI默认读取`DOCKER_HOST`环境变量，若设置为`tcp://host:port`，则自动通过TCP连接指定Docker Daemon，无需本地二进制兼容。
