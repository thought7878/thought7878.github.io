        在Dockerfile中添加参数`ARG JENKINS_VERSION=2.419`，支持后续升级基础镜像版本。
    - **镜像构建命令** [00:57](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=1cd97e72b13147e4aca8217600c5deb1#?seek_t=57)  
        执行`docker build -t jenkins:2.419-docker .`，其中`-t`指定镜像标签格式为`jenkins:<version>-docker`，`.`表示使用当前目录Dockerfile。

- **docker-compose编排改造 [03:52](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=1cd97e72b13147e4aca8217600c5deb1#?seek_t=232)**

        在`docker-compose.yml`中为DinD服务设置`networks: { jenkins: { aliases: [docker] } }`，使Jenkins容器可通过`docker:2376`访问DinD。
    - **证书校验规避** [04:49](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=1cd97e72b13147e4aca8217600c5deb1#?seek_t=289)  
        若未配置`aliases: [docker]`，Jenkins访问DinD时将因TLS证书校验失败而报错。
    - **环境变量与端口映射** [05:18](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=1cd97e72b13147e4aca8217600c5deb1#?seek_t=318)  
        Jenkins服务配置`environment: { DOCKER_HOST: tcp://docker:2376 }`，端口映射`ports: ["10050:8080"]`，DinD服务不暴露2376端口（同网络内直连）。

    - **适用场景** [08:04](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=1cd97e72b13147e4aca8217600c5deb1#?seek_t=484)
