## Pipeline创建的两种模式 
[03:16](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=1cd97e72b13147e4aca8217600c5deb1#?seek_t=196)
    
- **经典UI模式（Script模式）** [03:42](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=1cd97e72b13147e4aca8217600c5deb1#?seek_t=222)  
	创建自由风格项目 → 选择“Pipeline”类型 → 在“Pipeline Script”文本框中粘贴Jenkinsfile内容 → 点击“Apply”和“Save”执行。

	通过Blue Ocean Dashboard → “New Pipeline” → 选择Git仓库 → 自动读取根目录下的 `Jenkinsfile`并生成流水线。  
	*优势：UI现代化、阶段粒度清晰、支持单步重试、构建历史与代码变更强关联*。

## Jenkinsfile编写方法论 
[06:34](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=1cd97e72b13147e4aca8217600c5deb1#?seek_t=394)
    
- **AI辅助生成法** [06:37](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=1cd97e72b13147e4aca8217600c5deb1#?seek_t=397)  
	向AI工具提问（如：“为Vue项目生成Jenkinsfile”），获取含 `checkout`、`sh 'npm install'`、`sh 'npm run build'`、`sh 'ls -la dist'` 的基础结构。
- **Pipeline Syntax生成器法** [08:28](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=1cd97e72b13147e4aca8217600c5deb1#?seek_t=508)  
	进入任意Pipeline任务配置页 → 左下角“Pipeline Syntax” → 使用“Snippet Generator”交互生成代码：
	- `git` 步骤：输入仓库URL、分支（如 `main`）、凭据ID；
	- `nodejs` 步骤：选择Node.js版本（如 `nodejs18`），生成带环境封装的 `tool 'nodejs18'` 代码块；
	- 全局生效：将 `tools { nodejs 'nodejs18' }` 置于 `pipeline` 或 `agent` 块内。

## 多分支流水线触发机制 
[15:56](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=1cd97e72b13147e4aca8217600c5deb1#?seek_t=956)
    
- **三种官方触发方式** [15:56](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=1cd97e72b13147e4aca8217600c5deb1#?seek_t=956)
	- `Timer trigger`：定时触发（如 `H/5 * * * *`）；
	- `Poll SCM`：*Jenkins主动轮询Git仓库变更（推荐，规避防火墙拦截webhook）*；
	- `Upstream`：由其他Job成功后触发，支持跨项目联动。
- **Poll SCM配置路径** [16:21](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=1cd97e72b13147e4aca8217600c5deb1#?seek_t=981)  
	传统配置页 → “Scan Multibranch Pipeline Triggers” → 勾选“Periodically if not otherwise triggered” → 设置轮询频率（如 `* * * * *` 每分钟）。
