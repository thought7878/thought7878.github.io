- **提交失败原因：未配置用户身份 [07:38](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=379ebc04bb6e45bbbc27b6cc8d806f80#?seek_t=458)**
    
    - 首次提交报错提示：`please tell me who you are`；
    - 原因：Git要求每次提交必须携带 `user.name` 和 `user.email` 以标识作者。
- **全局用户配置警告与最佳实践 [08:06](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=379ebc04bb6e45bbbc27b6cc8d806f80#?seek_t=486)**
    
    - `git config --global user.name "xxx"` 和 `git config --global user.email "xxx@xxx.com"` 会覆盖所有仓库；
    - 若不同平台（如公司GitLab vs 个人GitHub）使用不同邮箱/用户名，**不建议使用 `--global`**；
    - *推荐做法：在每个仓库根目录下执行本地配置*：  
        `$ git config user.name "Your Name"`  
        `$ git config user.email "your-email@domain.com"`
- **本地用户配置与二次提交成功 [08:59](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=379ebc04bb6e45bbbc27b6cc8d806f80#?seek_t=539)**
    
    - 终端执行：  
        `$ git config user.name "John Doe"`  
        `$ git config user.email "john@example.com"`
    - 再次点击VS Code提交按钮，成功生成commit。
