### 工作区与本地版本库的概念引入 
[00:01](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=379ebc04bb6e45bbbc27b6cc8d806f80#?seek_t=1)

- **核心概念定义**
    - **工作区**：本地项目文件夹，如示例中的 `lan gate`文件夹
    - **本地版本库**：通过 `git init` 初始化后生成的 `.git` 隐藏文件夹，其中存储版本控制所需全部元数据
    - **暂存区（Staging Area）**：介于工作区与版本库之间的中间层，用于暂存待提交的更改

### 暂存区状态查看与文件生命周期操作 
[00:44](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=379ebc04bb6e45bbbc27b6cc8d806f80#?seek_t=44)

- **查看暂存区状态**  
    使用 `git status` 命令显示当前工作区与暂存区差异；初始状态下显示“暂无提交” 
- **文件创建与未跟踪状态**  
    在工作区新建文件（如 `hello git`）后，`git status` 显示该文件为红色，表示“未跟踪（untracked）”
- **添加文件至暂存区**  
    执行 `git add <file>` 后，文件状态变为绿色，表示已进入暂存区（staged）
- **提交至本地版本库**  
    执行 `git commit -m "添加hello git"` 将暂存区内容永久保存至本地版本库，生成唯一 SHA-1 哈希值作为提交标识
- **版本库追踪机制**  
    提交后的文件受 Git 全面追踪，支持任意历史版本回退与切换。

### 本地版本库内部结构
[01:55](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=379ebc04bb6e45bbbc27b6cc8d806f80#?seek_t=115)

- **`.git` 目录内容概览**  
    使用 `ls -la` 查看 `.git` 目录，包含 `logs/`、`objects/`、`hooks/`、`description`、`config` 等子目录与文件
- **`objects` 目录作用**  
    *存储所有提交快照（snapshots）及对象数据，每个快照对应一个 SHA-1 哈希值*
- **哈希值的生成与验证**  
    `git commit` 输出的完整哈希值（如 `b8f4a...`）与 `git log` 显示的前7位字符一致，用于唯一标识提交

### 远端版本库的引入与平台实践 
[03:11](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=379ebc04bb6e45bbbc27b6cc8d806f80#?seek_t=191)

- **远端仓库必要性**  
    为实现代码共享与协作，需借助远端版本库平台
- **主流平台举例**  
    国内平台：Gitee；国际平台：GitHub
- **创建远程仓库流程**  
    在 GitHub 界面点击 `+ → New repository`，命名（如 `VR`），选择 `Private`，点击 `Create repository` 完成创建
- **远程仓库连接协议**
    - **HTTPS 协议**：基于账号密码认证
    - **SSH 协议**：基于密钥对认证；若未配置密钥，仍可使用 HTTPS 方式交互

### Git 分布式特性总结 
[04:33](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=379ebc04bb6e45bbbc27b6cc8d806f80#?seek_t=273)

- **本地全流程闭环能力**  
    工作区 → 暂存区 → 本地版本库 的全部操作（`init`、`add`、`commit`、`log`、`checkout`）均可离线完成
- **分布式优势体现**  
    每个克隆副本都含完整历史与版本库，无需依赖中央服务器即可进行分支、合并、日志查询与版本切换
- **远端协作模式**  
    通过 `push`/`pull` 命令在本地与远端仓库间同步代码与提交记录；远端平台提供可视化界面简化仓库管理与链接分发
