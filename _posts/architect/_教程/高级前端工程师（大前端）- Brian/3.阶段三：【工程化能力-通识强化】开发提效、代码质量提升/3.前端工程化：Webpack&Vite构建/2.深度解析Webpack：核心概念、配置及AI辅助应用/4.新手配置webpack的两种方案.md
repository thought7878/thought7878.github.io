- **Rollup与Webpack的历史渊源及选型原因 [00:02](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=e69d94cce7aa40dfa0e9d161533d80b0#?seek_t=2)**
    
    - Vue 1.0起核心库打包由Rollup替代原Webpack；Yarn Pack（应为Webpack）在部分场景下性能不及Rollup。
    - Vue 2/3延续历史技术栈：UCR 4.x仍用Webpack，而Vue CLI（Vite）采用Rollup；核心库始终使用Rollup。
- **Rollup的核心定位与标准化优势 [00:52](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=e69d94cce7aa40dfa0e9d161533d80b0#?seek_t=52)**
    
    - Rollup是JavaScript模块打包器，将小块代码编译为大块可复用代码；作用与Webpack类似，但默认采用ES6标准化模块格式（非CommonJS或AMD）。
    - ES模块为官方标准与语言发展方向，支持静态分析，实现Tree Shaking（清除无用代码）、循环引用、动态绑定等高级功能；CommonJS为传统非标准格式。
- **Tree Shaking机制详解 [02:41](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=e69d94cce7aa40dfa0e9d161533d80b0#?seek_t=161)**
    
    - Tree Shaking即按需加载/按需引用，核心作用是清除未被引用的无用代码。
- **Rollup核心概念与配置体系 [03:15](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=e69d94cce7aa40dfa0e9d161533d80b0#?seek_t=195)**
    
    - 核心概念与Webpack高度一致：**入口（input）**、**输出（output）**、**插件（plugins）**、**配置文件**（`rollup.config.js`）及**加载器（loaders）**。
    - 配置文件必须包含`output`（required），支持`external`、`plugins`、`treeshake`、`experimental`等选项；推荐优先研读官方文档而非中文社区。
- **Rollup安装与项目初始化规范 [04:52](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=e69d94cce7aa40dfa0e9d161533d80b0#?seek_t=292)**
    
    - 两种使用方式：项目本地安装（推荐）或全局安装CLI（`npm install -g rollup`）。
    - 初始化NPM项目时禁止将目录命名为`rollup`，否则`package.json`中`name: "rollup"`会导致`npm install rollup`失败；应使用`rollup-demo`等命名。
    - 本地安装命令：`npm install -D rollup`；执行命令：`npx rollup`或在`package.json`中配置脚本。
- **CLI命令详解与基础打包实践 [06:22](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=e69d94cce7aa40dfa0e9d161533d80b0#?seek_t=382)**
    
    - `npx rollup --help`显示全部参数：`--config`（指定配置文件）、`--file`（输出文件名）、`--input`（入口文件）、`--format`（输出格式）、`--watch`（监听模式）。
    - 输出格式支持：`amd`、`cjs`（CommonJS）、`es`（ES模块）、`iife`、`umd`、`system`，覆盖全环境需求。
    - 实践步骤：
        - 创建`src/main.js`（ES6语法），`src/cjs.js`（`export default function say(name) { console.log(name); }`）；
        - `main.js`中`import say from './cjs.js'`；
        - 执行`npx rollup src/main.js --format cjs --file dist/bundle.js`；
        - 输出`dist/bundle.js`第三行为`Object.defineProperty(...)`，标识已编译ES模块；
        - 新建`dist/test.js`，以CommonJS方式`const say = require('./bundle.js'); say('hello rollup')`；
        - 终端执行`node dist/test.js`，输出`hello rollup`。
- **多格式并行输出配置 [11:26](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=e69d94cce7aa40dfa0e9d161533d80b0#?seek_t=686)**
    
    - 创建`rollup.config.js`，导出数组形式配置：
        
        js
        
        ```js
        export default [  
          {  
            input: 'src/main.js',  
            output: { file: 'dist/bundle.cjs', format: 'cjs' }  
          },  
          {  
            input: 'src/main.js',  
            output: { file: 'dist/bundle.es.js', format: 'es' }  
          }  
        ];
        ```
        
    - `package.json`脚本设为`"build-all": "rollup -c"`，执行`npm run build-all`生成双格式文件。
- **插件系统（Plugins）集成 [13:04](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=e69d94cce7aa40dfa0e9d161533d80b0#?seek_t=784)**
    
    - 插件生态地址：awesome-rollup；常用插件包括`@rollup/plugin-commonjs`、`@rollup/plugin-json`、`rollup-plugin-terser`（代码压缩）。
    - 插件引入方式：
        - 安装：`npm install -D rollup-plugin-terser`；
        - `rollup.config.js`中`import terser from 'rollup-plugin-terser'`；
        - `plugins: [terser()]`（顺序引用、顺序执行，与Webpack逆序执行相反）。
    - 压缩验证：执行后`bundle.cjs`与`bundle.es.js`均压缩为单行代码。
- **高级配置技巧与Vue源码参考 [14:30](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=e69d94cce7aa40dfa0e9d161533d80b0#?seek_t=870)**
    
    - 共享配置：`export default { input: 'src/main.js', output: [{ file: 'dist/bundle.cjs', format: 'cjs' }, { file: 'dist/bundle.es.js', format: 'es' }] };`
    - 调试配置文件：在VS Code中对`rollup.config.js`打调试断点，直接运行`npx rollup -c`进行断点调试。
    - Vue Next仓库（`vue-next`）中`rollup.config.ts`为真实工业级范例：支持多格式（cjs/esm/umd）、`external`排除依赖、`resolve`路径解析、`plugins`链式调用，体现Rollup打包核心库的最佳实践。
- **核心总结与学习路径 [15:33](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=e69d94cce7aa40dfa0e9d161533d80b0#?seek_t=933)**
