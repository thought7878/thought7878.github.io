### 第一大应用场景：for...of 循环 
[00:47](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=a7d94a717c6448d4b2706a9e235c2be4#?seek_t=47)

- **适用范围定义 [00:47](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=a7d94a717c6448d4b2706a9e235c2be4#?seek_t=47)**  
    任何具有迭代器模式的有序结构（即拥有`Symbol.iterator`方法的对象）均可使用`for...of`。
- **for...of 与 for...in 的区分 [00:58](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=a7d94a717c6448d4b2706a9e235c2be4#?seek_t=58)**  
    本节仅聚焦`for...of`，`for...in`留待后续专题讲解，二者本质不同、无可比性。
- **数组遍历示例 [01:27](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=a7d94a717c6448d4b2706a9e235c2be4#?seek_t=87)**  
    `let arr = [1, 2, 3]; for (const n of arr) { console.log(n); }` 输出 `1`, `2`, `3`。
- **Set 遍历示例 [02:02](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=a7d94a717c6448d4b2706a9e235c2be4#?seek_t=122)**  
    `let set = new Set([100, 200, 300]); for (const n of set) { console.log(n); }` 输出 `100`, `200`, `300`。
- **DOM NodeList 遍历示例 [02:29](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=a7d94a717c6448d4b2706a9e235c2be4#?seek_t=149)**  
    `let pList = document.querySelectorAll('p'); for (const p of pList) { console.log(p); }`可直接遍历。
- **自定义迭代器对象遍历示例 [03:04](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=a7d94a717c6448d4b2706a9e235c2be4#?seek_t=184)**  
    自定义对象若实现`[Symbol.iterator]()`方法，且该方法返回对象含`next()`，且`next()`返回`{ value, done }`格式，则可被`for...of`遍历。
- **底层机制揭示 [04:15](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=a7d94a717c6448d4b2706a9e235c2be4#?seek_t=255)**  
    `for...of`本质调用目标对象的`Symbol.iterator`方法获取迭代器，再反复调用其`next()`方法，依据`done`属性判断循环终止。
- **与传统for循环对比 [04:32](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=a7d94a717c6448d4b2706a9e235c2be4#?seek_t=272)**  
    传统`for`循环需显式依赖索引（如`i`）和长度（如`arr.length`），并手动通过`arr[i]`取值；而`for...of`完全屏蔽这些细节，仅关注数据本身，体现迭代器模式的抽象性与解耦性。
- **编程范式升华 [05:34](https://b.quark.cn/apps/5AZ7aRopS/routes/mofb35Rkb?debug=0&fid=a7d94a717c6448d4b2706a9e235c2be4#?seek_t=334)**  
    传统`for`循环违背高内聚低耦合原则；`for...of`是迭代器模式的典型实践，使代码更健壮、更易维护。
