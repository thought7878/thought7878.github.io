>Docker 镜像管理详解（初学者友好版）

镜像（Image）是 Docker 的核心，理解镜像管理是学好 Docker 的第一步。下面按**使用场景**分类讲解，每个命令配一句话解释 + 实用示例。

---

## 一、基础操作命令

| 命令 | 作用 | 示例 |
|------|------|------|
| `docker pull` | 从仓库下载镜像 | `docker pull nginx:latest` |
| `docker images` | 列出本地所有镜像 | `docker images -a`（含中间层） |
| `docker rmi` | 删除本地镜像 | `docker rmi nginx:latest` |
| `docker tag` | 给镜像打新标签 | `docker tag nginx myrepo/nginx:v1` |
| `docker history` | 查看镜像构建历史 | `docker history nginx` |
| `docker inspect` | 查看镜像详细信息（JSON） | `docker inspect nginx` |

> 💡 小贴士：镜像名格式为 `仓库/镜像名:标签`，如 `library/nginx:1.24`，`latest` 是默认标签但不代表最新版！

---

## 二、镜像来源与仓库

### 🔹 公共仓库
- **Docker Hub**（hub.docker.com）：官方主仓库，镜像最全
- **阿里云/腾讯云镜像加速器**：国内访问更快，配置 `/etc/docker/daemon.json`
- **GitHub Container Registry**：与代码仓库联动，适合开源项目

### 🔹 私有仓库
```bash
# 搭建简易私有仓库（官方 registry 镜像）
docker run -d -p 5000:5000 --name registry registry:2

# 推送镜像到私有仓库
docker tag myapp localhost:5000/myapp:v1
docker push localhost:5000/myapp:v1

# 从私有仓库拉取
docker pull localhost:5000/myapp:v1
```

> ⚠️ 注意：非 HTTPS 私有仓库需在 `daemon.json` 添加 `"insecure-registries": ["localhost:5000"]`

## 三、镜像构建与优化

### 🔹 使用 Dockerfile 构建
```dockerfile
# 示例：构建 Python 应用镜像
FROM python:3.11-slim          # 1. 选精简基础镜像
WORKDIR /app                   # 2. 设置工作目录
COPY requirements.txt .        # 3. 先复制依赖文件（利用缓存）
RUN pip install --no-cache-dir -r requirements.txt  # 4. 安装依赖
COPY . .                       # 5. 复制应用代码
EXPOSE 8000                    # 6. 声明端口（文档作用）
CMD ["python", "app.py"]       # 7. 启动命令
```
```bash
# 构建命令
docker build -t myapp:v1 .     # 当前目录构建，指定标签
docker build --no-cache -t myapp:v1 .  # 禁用缓存（调试用）
```

### 🔹 镜像瘦身技巧
| 方法 | 效果 | 示例 |
|------|------|------|
| 选精简基础镜像 | 减少 50%~80% 体积 | `python:3.11-slim` 替代 `python:3.11` |
| 多阶段构建 | 仅保留编译结果 | 见下方示例 |
| 合并 RUN 指令 | 减少镜像层数 | `RUN apt update && apt install -y xxx && rm -rf /var/lib/apt/lists/*` |
| 清理缓存 | 避免残留文件 | `npm cache clean --force` / `apt clean` |
| 使用 .dockerignore | 避免复制无用文件 | 类似 `.gitignore`，排除 `node_modules`、`.git` 等 |

#### ✅ 多阶段构建示例（Go 应用）
```dockerfile
# 第一阶段：编译
FROM golang:1.21 AS builder
WORKDIR /app
COPY . .
RUN go build -o main .

# 第二阶段：运行（最终镜像仅含二进制文件）
FROM alpine:latest
COPY --from=builder /app/main /main
CMD ["/main"]
# 最终镜像从 800MB+ 降至 10MB 左右！
```

## 四、镜像导出与迁移

| 场景 | 命令 | 说明 |
|------|------|------|
| **导出为文件** | `docker save -o nginx.tar nginx:latest` | 生成 `.tar` 包，含所有层 |
| **导入镜像** | `docker load -i nginx.tar` | 从文件恢复镜像到本地 |
| **仅导出容器快照** | `docker export -o container.tar <容器ID>` | 导出容器文件系统（非镜像，无构建历史） |
| **跨机器迁移** | `save` + `scp` + `load` 组合 | 适合无网络环境或内网部署 |

> 🔄 `save/load` vs `export/import`：  
> - `save/load` 操作**镜像**，保留分层和元数据，推荐用于备份/迁移  
> - `export/import` 操作**容器**，扁平化为单层，适合快速快照

## 五、镜像标签与版本管理

### 🔹 标签规范建议
```bash
# ✅ 推荐：语义化版本 + 环境标识
docker tag myapp myrepo/myapp:1.2.3
docker tag myapp myrepo/myapp:1.2.3-prod

# ❌ 避免：生产环境用 latest（不可追溯）
docker tag myapp myrepo/myapp:latest  # 仅用于开发测试

# ✅ 同时打多个标签（一次构建，多环境复用）
docker build -t myapp:1.2.3 -t myapp:latest .
```

### 🔹 查看与清理
```bash
# 查看镜像大小与创建时间
docker images --format "table {{.Repository}}\t{{.Tag}}\t{{.Size}}\t{{.CreatedAt}}"

# 删除悬空镜像（无标签的中间镜像）
docker image prune

# 删除所有未被容器使用的镜像
docker image prune -a

# ⚠️ 谨慎：强制删除（即使被容器引用）
docker rmi -f nginx:latest
```

## 六、镜像安全与可信验证

| 措施 | 命令/工具 | 作用 |
|------|-----------|------|
| **扫描漏洞** | `docker scan nginx:latest`（集成 Snyk） | 检查基础镜像已知 CVE 漏洞 |
| **验证签名** | `docker trust inspect --pretty nginx` | 查看 Docker Content Trust 签名 |
| **启用内容信任** | `export DOCKER_CONTENT_TRUST=1` | 强制只拉取签名镜像（生产推荐） |
| **使用固定 Digest** | `nginx@sha256:abc123...` | 避免标签被篡改，确保镜像内容一致 |

> 🔐 最佳实践：  
> 1. 定期 `docker pull` 更新基础镜像（修复安全漏洞）  
> 2. 生产环境避免使用 `:latest`，改用固定版本或 Digest  
> 3. 敏感应用使用私有仓库 + 镜像签名 + 漏洞扫描流水线

## 🎯 初学者实操小任务（建议动手试试）

```bash
# 1️⃣ 拉取并查看 nginx 镜像
docker pull nginx:alpine
docker images | grep nginx
docker history nginx:alpine

# 2️⃣ 重命名并导出
docker tag nginx:alpine mynginx:test
docker save -o mynginx.tar mynginx:test

# 3️⃣ 删除后重新导入验证
docker rmi mynginx:test
docker load -i mynginx.tar
docker images | grep mynginx

# 4️⃣ 尝试构建一个自定义镜像（新建 Dockerfile）
echo -e "FROM alpine\nRUN echo 'Hello Docker' > /hello.txt" > Dockerfile
docker build -t hello:v1 .
docker run --rm hello:v1 cat /hello.txt  # 输出: Hello Docker
```

---

✅ **镜像管理核心口诀**：  
🔹 拉取用 `pull`，查看用 `images`，删除用 `rmi`  
🔹 构建写 `Dockerfile`，优化靠「精简 + 多阶段」  
🔹 迁移用 `save/load`，安全靠「签名 + 扫描」  
🔹 标签要规范，生产不用 `latest`
